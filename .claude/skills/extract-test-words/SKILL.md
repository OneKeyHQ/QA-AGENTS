# 提取测试单词

## Overview

从用户发来的图片中提取带序号的单词，按序号升序排列后，**只输出单词本身**（不带序号、不带多余解释）。

## 执行步骤

1. **读取本条消息附带的所有图片**。没有图片时，提示用户补发图片，不要凭空输出。
2. **逐个识别「序号 + 单词」对**。注意：
   - 排序**以图片上标注的序号为准**，不是视觉位置顺序。助记词卡片常见多列布局，序号可能是「竖排」（第一列 1-6，第二列 7-12）或「横排」交错，必须看数字标签。
   - 多张图片时，把所有序号+单词对合并后统一按序号排序。
3. **完整性校验**：序号应从 1 连续到 N（常见 12 / 18 / 24 个）。
4. **输出**。

## 输出格式（严格）

只输出一行按序号排序的单词，空格分隔，不带序号、不带引号、不带其他说明文字：

```
# TEST_ONLY 占位示例（BIP39 词表字母序前 12 词，非真实助记词）
abandon ability able about above absent absorb abstract absurd abuse access accident
```

> ⚠️ 提取出的真实测试单词只发到对话里，**严禁写入任何仓库文件 / 日志 / 报告**。

## 异常处理（不允许猜测）

| 情况 | 处理 |
|------|------|
| 某个序号缺失或重复 | 输出已识别的单词，并单独一行指出「序号 X 缺失/重复」 |
| 某个单词模糊无法确认 | 该位置用 `?` 占位，并单独指出「序号 X 无法识别，疑似 xxx」，不要硬填 |
| 两个相似候选词拿不准（如 `then/them`） | 列出候选让用户确认，不要默认选一个 |
| 图片里没有序号 | 询问用户按什么顺序排（不要自作主张按视觉顺序输出） |

## 禁止行为

- 不要在单词前后附加序号、编号列表、markdown 列表符号
- 不要输出「以下是提取结果：」之类的前缀废话（异常说明除外）
- 不要把识别不清的单词凭印象补全
