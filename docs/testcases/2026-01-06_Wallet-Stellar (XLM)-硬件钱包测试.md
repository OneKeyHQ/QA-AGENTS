# Wallet - Stellar (XLM) 硬件钱包测试

## 测试概览

**测试目标**：验证 Stellar 链在硬件钱包下的主币转账、代币转账和 dApp 连接功能，重点验证硬件钱包签名确认流程及设备上签名详情的显示。

**测试范围**：
- XLM 原生转账（硬件钱包签名）
- Stellar Asset Token 转账（硬件钱包签名）
- dApp 连接与交易（硬件钱包签名）

**不支持功能**（仅备注，不输出详细用例）：
- ❌ Contract Token 转账：不支持
- ❌ Swap：不支持
- ❌ Soroban Token：不支持

**测试环境**：Mainnet

**硬件钱包型号**：所有支持的硬件钱包型号（需回归测试）

**签名验证说明**：Stellar 为非盲签链，需验证硬件设备上显示的签名详情（From/To 地址、转账金额、链名称、Gas 费用、代币名称）

**Memo 规则**：Stellar 链 Memo 最大长度为 28 字节

1. **硬件钱包支持判断**：Contract Token 和 Swap 功能在硬件钱包下应被禁用
2. **签名详情解析**：验证硬件钱包签名详情解析逻辑（From/To 地址、金额、链名称、Gas 费用、代币名称）
3. **Memo 长度验证**：验证 Memo 最大长度限制（Stellar 链为 28 字节）

### 端到端（E2E）重点
1. **Mock 数据**：
   - 硬件钱包连接状态
   - 硬件钱包签名响应（确认/拒绝/超时）
   - 硬件钱包签名详情显示（From/To 地址、金额、链名称、Gas 费用、代币名称、Memo）
   - Horizon API 响应（账户信息、余额、交易记录）
   - 链上交易状态（Pending → Success/Fail）
2. **关键拦截点**：
   - 硬件钱包连接请求
   - 硬件钱包签名请求
   - 硬件钱包签名详情显示验证
3. **断言验证**：
   - 硬件钱包签名确认流程完整性
   - 硬件设备上签名详情显示正确性（非盲签链验证）
   - 签名详情与交易参数一致性
   - Memo 长度和格式验证
   - 交易状态流转（Pending → Success）
