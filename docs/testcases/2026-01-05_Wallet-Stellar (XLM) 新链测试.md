# Wallet - Stellar (XLM) 新链测试

## 测试概览
测试目标：Stellar 链的基础转账、Token 管理、dApp 连接及硬件钱包支持。<br>
重点关注：账户最低余额限制、Trustline（信任线）激活机制、不同 Token 类型的处理逻辑、交易所（CEX）充提。

## 自动化实施方案

### Unit 测试重点
1. **Memo 类型推断逻辑**：纯数字 → MEMO_ID，字符串 → MEMO_TEXT
2. **账户余额计算**：可用余额 = 总余额 - 锁定金额（0.5 XLM × Token 数量）- 1 XLM（基础保留）
3. **Token 类型识别**：Code:Issuer 格式识别、C 开头地址判断
4. **精度处理**：XLM/Asset Token 固定 7 位，Contract Token 动态查询

### E2E/API 测试重点
1. **Mock 数据**：
   - Horizon API 响应（账户信息、余额、交易记录）
   - RPC simulateTransaction 响应（Contract Token 余额、精度）
   - WebSocket 连接状态（dApp 连接）
2. **关键拦截点**：
   - createAccount 交易构建
   - createTrustline 交易构建
   - payment 交易构建
   - Memo 类型设置
   - Gas 费用计算
3. **状态验证**：
   - 交易状态流转（Pending → Success/Fail）
   - 余额更新时机
   - Trustline 激活状态
