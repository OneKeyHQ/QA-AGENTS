# Market - Token 收藏取消收藏 冒烟测试 Checklist

> 生成时间：2026-01-04  
> 来源用例：`docs/testcases/cases/2026-01-04_Market-Token收藏取消收藏.md`
> 
> **执行模式说明**：
> - `smoke` / `smoke-fast`：只执行 [P0] 标记的核心场景（约 20%）
> - `smoke-full`：执行所有场景（P0/P1/P2）

## 前置条件
- [ ] 浏览器已打开，可访问 Market 模块
- [ ] Market 模块已加载完成，用户无需登录
- [ ] 网络连接正常
- [ ] 观察列表初始状态：部分 Token 已收藏，部分未收藏（用于测试）

## 操作步骤

### 场景1：观察列表收藏/取消收藏测试
- [P0] (a) 进入观察列表，如果列表为空则显示空状态提示和引导添加收藏的提示或按钮
- [P0] (b) 在观察列表中勾选部分未收藏的 Token 添加收藏，验证勾选的 Token 显示已收藏状态（图标/标识变化），观察列表数据增加正确（收藏数量+勾选数量）
- [P1] (c) 切换其他分类后返回观察列表，验证相应 Token 已标识收藏状态，其他分类列表中对应 Token 显示已收藏标识
- [P2] (d) 在观察列表中取消所有已收藏 Token，然后添加所有推荐 Token 到收藏，验证所有已收藏 Token 取消成功（标识消失），所有推荐 Token 添加成功（标识出现），观察列表数据更新正确（数量=推荐 Token 数量），观察列表仅显示已收藏的 Token
- [P2] (e) 如果观察列表有大量收藏 Token（>50），滚动列表到底部，验证列表支持滚动加载，所有收藏 Token 正确展示，滚动流畅无卡顿（FPS ≥ 30）

### 场景2：分类列表收藏/取消收藏测试
- [P0] (a) 切换到其他分类列表（如"热门"、"涨幅榜"），点击某个未收藏 Token 的收藏按钮，验证 Token 收藏成功（按钮状态变化，显示已收藏标识），观察列表数据增加正确（数量+1，新增 Token 出现在列表）
- [P0] (b) 切换其他分类核对，验证相应 Token 已标识收藏状态，收藏按钮状态在所有分类中保持一致
- [P1] (c) 在分类列表中点击已收藏 Token 的取消收藏按钮，验证取消收藏成功（按钮状态变化，收藏标识消失），观察列表数据删除正确（数量-1，Token 从列表移除）
- [P1] (d) 切换其他分类核对，验证相应 Token 标识取消（显示未收藏状态），取消收藏按钮状态在所有分类中保持一致
- [P2] (e) 在分类列表中快速连续点击同一 Token 的收藏按钮 3 次，等待操作完成，验证防抖生效（仅执行最后一次操作），最终状态与最后一次点击一致，观察列表数据与最终状态一致，无重复请求或状态错乱
- [P2] (f) 如果分类列表支持筛选/排序，在筛选后的列表中收藏 Token，清除筛选条件后核对观察列表，验证收藏操作不受筛选条件影响，观察列表正确显示已收藏 Token，筛选条件清除后 Token 收藏状态保持不变

### 场景3：Token 详情页收藏/取消收藏测试
- [P0] (a) 选择一个未收藏的 Token 进入详情页，点击收藏按钮，验证收藏成功（按钮状态变化，显示已收藏标识），观察列表数据增加正确（数量+1，新增 Token 出现在列表）
- [P1] (b) 切换其他分类核对，验证相应 Token 已标识收藏状态，详情页收藏按钮状态与列表页一致
- [P1] (c) 进入已收藏 Token 的详情页，点击取消收藏按钮，验证取消收藏成功（按钮状态变化，收藏标识消失），观察列表数据删除正确（数量-1，Token 从列表移除）
- [P1] (d) 切换其他分类核对，验证相应 Token 标识取消（显示未收藏状态），详情页取消收藏按钮状态与列表页一致
- [P2] (e) 在详情页点击收藏后返回列表页，再次进入该 Token 详情页，验证详情页显示已收藏状态，列表页显示已收藏标识，再次进入详情页时状态保持一致
- [P2] (f) 如果详情页支持分享/复制链接，在详情页收藏 Token 后通过分享链接打开详情页，验证通过分享链接打开的详情页显示正确的收藏状态，收藏状态与用户账户绑定，不随链接变化

### 场景4：搜索列表收藏/取消收藏测试
- [P0] (a) 搜索"USDT"，在搜索列表点击某个未收藏的 USDT Token 的收藏按钮，验证收藏成功（按钮状态变化，显示已收藏标识），观察列表数据增加正确（数量+1，新增 Token 出现在列表）
- [P1] (b) 切换其他分类核对，验证相应 Token 已标识收藏状态，搜索列表中的收藏状态与其他入口一致
- [P1] (c) 继续搜索"USDT"，在搜索列表点击已收藏 Token 的取消收藏按钮，验证取消收藏成功（按钮状态变化，收藏标识消失），观察列表数据删除正确（数量-1，Token 从列表移除）
- [P1] (d) 切换其他分类核对，验证相应 Token 标识取消（显示未收藏状态），搜索列表中的取消收藏状态与其他入口一致
- [P0] (e) 搜索主币（SOL/SUI/BTC/ETH），在搜索列表点击收藏按钮，验证主币收藏成功（按钮状态变化，显示已收藏标识），观察列表数据增加正确（数量+1，主币出现在列表），主币在分类列表中显示已收藏标识
- [P1] (f) 搜索主币并进入详情页，点击取消收藏按钮，验证取消收藏成功（按钮状态变化，收藏标识消失），观察列表数据删除正确（数量-1，主币从列表移除），切换其他分类后主币标识取消（显示未收藏状态），详情页与列表页状态一致
- [P2] (g) 搜索不存在的 Token 名称，验证显示"未找到相关结果"或空状态，无收藏按钮或收藏按钮不可用
- [P2] (h) 搜索"USDT"，分别收藏不同链的 USDT Token，核对观察列表，验证每个链的 USDT Token 可独立收藏，观察列表显示所有已收藏的 USDT Token（含链标识），不同链的 Token 收藏状态独立管理

### 场景5：数据一致性与状态同步测试
- [P0] (a) 在观察列表收藏 Token A，切换到分类列表核对 Token A 的收藏状态，进入 Token A 详情页核对收藏状态，验证观察列表显示已收藏，分类列表显示已收藏标识，详情页显示已收藏状态，三个入口的收藏状态完全一致
- [P1] (b) 在分类列表收藏 Token B，切换到观察列表核对 Token B 是否出现，搜索 Token B 核对搜索列表中的收藏状态，验证观察列表立即显示 Token B（或刷新后显示），Token B 出现在观察列表正确位置，搜索列表中 Token B 显示已收藏状态，所有入口状态同步（延迟 ≤ 1 秒）
- [P2] (c) 在任意入口收藏 Token，抓取收藏 API 请求，验证 API 请求包含正确的 Token 标识（symbol/address/chainId），响应状态码=200，返回收藏成功标识，响应数据包含收藏时间戳，请求幂等性：重复收藏返回相同结果
- [P2] (d) 在端 A 收藏 Token，在端 B 刷新 Market 页面，核对端 B 的收藏状态，验证端 B 的观察列表显示端 A 收藏的 Token，端 B 的分类列表显示已收藏标识，跨端数据同步（延迟 ≤ 5 秒）
- [P2] (e) 如果收藏列表有大量 Token（>100），进入观察列表滚动到底部，执行收藏/取消收藏操作，验证列表滚动流畅（FPS ≥ 30），收藏/取消收藏操作响应时间 ≤ 500ms，操作后列表更新正确，无闪烁或跳动

### 场景6：网络异常与容错测试
- [P1] (a) 点击收藏按钮后立即断开网络，观察 UI 状态，验证显示网络错误提示（如"网络异常，请稍后重试"），收藏状态回滚到操作前状态，提供重试按钮或自动重试机制
- [P2] (b) 断开网络后点击收藏按钮，验证显示网络错误提示，收藏操作不执行（按钮状态不变），恢复网络后可正常收藏
- [P2] (c) Mock 收藏 API 返回 500 错误，执行收藏操作，验证显示错误提示（如"服务异常，请稍后重试"），收藏状态不改变，提供重试入口
- [P2] (d) 模拟弱网环境（延迟 > 3s），执行收藏操作，观察加载状态，验证显示加载状态（按钮显示 loading 或禁用），操作完成后状态正确更新，无重复请求

### 场景7：多端适配测试
- [P1] (a) 在 Android 端进入 Market 模块，执行收藏/取消收藏操作，验证收藏按钮位置和大小适配移动端，操作响应流畅（点击反馈 ≤ 100ms），观察列表布局适配移动端（列表项高度、间距合理）
- [P1] (b) 在 iOS 端进入 Market 模块，执行收藏/取消收藏操作，验证收藏按钮符合 iOS 设计规范，操作响应流畅，支持 iOS 手势（如滑动删除收藏，如适用）
- [P2] (c) 在 Desktop 端进入 Market 模块，执行收藏/取消收藏操作，验证收藏按钮支持鼠标悬停效果，支持键盘快捷键（如适用），列表布局适配桌面端（可显示更多列信息）
- [P2] (d) 在 Extension 端进入 Market 模块，执行收藏/取消收藏操作，验证收藏功能在扩展窗口正常使用，窗口大小变化时布局自适应，与主应用数据同步

### 场景8：边界与异常场景测试
- [P1] (a) 尝试收藏第 101 个 Token（假设收藏数量上限为 100），验证显示上限提示（如"收藏数量已达上限"），收藏操作不执行，已收藏的 Token 仍可正常取消收藏
- [P2] (b) 如果观察列表为空，尝试取消收藏，验证无取消收藏按钮或按钮不可用，显示空状态提示
- [P2] (c) 在分类 A 收藏 Token，快速切换到分类 B、C、D，返回分类 A 核对状态，验证收藏状态在所有分类中正确显示，无状态丢失或错乱，切换流畅无卡顿
- [P2] (d) 执行收藏操作后立即切换应用到后台，5 秒后返回应用，验证收藏操作在后台完成，返回应用后状态正确，观察列表数据已更新
