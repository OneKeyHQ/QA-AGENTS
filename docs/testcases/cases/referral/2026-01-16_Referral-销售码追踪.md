## 自动化实施方案

### 1. 单元测试（Unit）重点
- 兑换码格式校验逻辑（Trim、特殊字符过滤）
- 按钮状态切换逻辑（空输入置灰、有输入可点击）
- 错误码映射逻辑（API 错误码 → 用户提示文案）

### 2. API 测试重点
- 兑换接口参数校验
- 各类错误码返回场景（invalid_code、error_old_account、error_not_eligible、error_already_reedem）
- 兑换成功后账户等级变更

### 3. 端到端（E2E）重点
- Mock 数据：
  - 有效/无效兑换码
  - 不同账户状态（新账户、有返佣记录、高级别、已使用过销售码）
  - 网络超时场景
- 关键拦截点：
  - `/api/redemption/redeem` - 兑换请求
  - `/api/referral/level` - 返佣等级查询
  - `/api/redemption/history` - 兑换历史查询
