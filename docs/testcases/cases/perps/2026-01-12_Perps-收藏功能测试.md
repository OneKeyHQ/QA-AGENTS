## 自动化实施方案（P0/P1）

### 1. 单元测试（Unit）重点
- **收藏状态管理逻辑**：验证收藏/取消收藏的状态切换、状态持久化
- **数据同步逻辑**：验证多入口收藏状态的一致性检查机制
- **防抖/节流逻辑**：验证快速连续点击的防抖处理
- **显示模式切换逻辑**：验证百分比/涨跌幅切换的状态管理
- **数据校验**：验证代币标识（symbol/pairId）的格式校验

### 2. 端到端（E2E/API）重点
- **需要 Mock**：
  - 收藏/取消收藏 API（成功/失败/超时场景）
  - 默认推荐代币列表 API
  - 自选列表查询 API
  - 行情数据 API（价格、涨跌幅）
  - WebSocket 实时数据更新（如适用）
- **关键拦截点**：
  - 收藏 API 请求参数（代币标识、用户 ID、时间戳）
  - 收藏 API 响应数据（成功标识、收藏列表更新）
  - 自选列表数据查询接口
  - 显示模式切换的本地存储（localStorage）
- **UI 断言点**：
  - 推荐列表复选框状态（勾选/未勾选）
  - 收藏按钮状态（已收藏/未收藏图标变化）
  - 自选列表数据数量变化（+1/-1）
  - 行情页面顶部列表更新（新增/移除代币）
  - 显示模式切换（「$」/「%」按钮激活状态、价格变化显示格式）
  - 多入口收藏状态一致性（推荐列表/自选列表/搜索列表/分类列表/行情页顶部）
  - 网络异常时的错误提示和状态回滚
