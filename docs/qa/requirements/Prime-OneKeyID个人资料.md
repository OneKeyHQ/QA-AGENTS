# Prime - OneKey ID 个人资料

> 模块：Prime  
> App 版本：待定  
> 日期：2026-03-20  
> 测试端：iOS / Android / Desktop / Extension / Web

---

## 1. 需求背景

OneKey ID 个人资料用于在 Prime 体系下承载用户头像、昵称、邮箱与 Prime 状态展示，并支持跨端查看与编辑。个人资料既是 OneKey ID 入口的核心展示区，也是 Prime 状态的主要承载位置之一。

---

## 2. 功能描述

### 2.1 菜单与入口展示

| 项目 | 说明 |
|-----|------|
| 未登录态 | 菜单顶部显示 OneKey ID 注册/登录入口与默认头像 |
| 已登录态 | 显示头像、昵称、邮箱、当前登录客户端对应的 Prime 状态图标 |
| Prime 状态图标 | 正常权益显示绿标；设备超限显示黄标；无 Prime 不显示 |
| 默认昵称 | 未设置昵称时显示 `OneKey_单词+数字`，超长按 UI 规则省略 |
| 邮箱脱敏 | 按首字母 3 位 + `***` + 后缀显示 |

### 2.2 点击行为

| 点击区域 | 结果 |
|---------|------|
| Prime 状态图标 | 打开 Prime 弹窗 |
| 头像 | 打开个人资料编辑弹窗 |
| 其余 OneKey ID 区域 | 打开 OneKey ID 弹窗 |

### 2.3 个人资料编辑

| 项目 | 说明 |
|-----|------|
| 昵称字符 | 不限制字符类型 |
| 昵称长度 | 最多 20 字符，超过无法继续输入 |
| 清空昵称 | 触发不能为空校验 |
| 保存按钮 | 校验通过后可提交；校验报错时不可提交 |
| 网络异常 | 断网状态下提交修改需提示失败 |

### 2.4 头像选择

| 项目 | 说明 |
|-----|------|
| 非 iOS | 打开系统图片选择器 |
| iOS | 首次需申请照片权限 |
| 选择图片 | 支持进入裁剪流程后更新头像预览 |

### 2.5 跨端刷新

| 项目 | 说明 |
|-----|------|
| 资料同步 | 修改成功后，最新头像与昵称同步到其他已登录设备 |
| 状态展示 | 各端按各自当前客户端状态展示 Prime 状态图标；颜色仅表示当前登录客户端自身状态 |

---

## 3. 业务规则

| 规则项 | 规则描述 |
|-------|---------|
| 登录态区分 | 菜单顶部需区分未登录态与已登录态展示 |
| 图标语义 | Prime 状态图标仅表示当前登录客户端自身状态 |
| 默认昵称 | 未设置昵称时展示系统生成昵称 |
| 邮箱展示 | 邮箱需按脱敏规则展示 |
| 昵称输入 | 支持多字符类型，最大长度 20 |
| 保存限制 | 昵称校验失败时不可提交 |
| 权限申请 | iOS 修改头像需先处理照片权限 |
| 提交异常 | 网络异常时不得误提示修改成功 |
| 跨端同步 | 头像与昵称修改后需同步到其他登录设备 |

---

## 4. 已知风险

- 特殊字符、Emoji、多语言字符在不同端的显示宽度与截断策略待统一
- iOS 照片权限拒绝后二次申请路径需与系统版本兼容
- 跨端刷新存在缓存时的显示延迟待确认
- Prime 状态图标与设备超限状态的联动依赖 4.5 设备限制规则
