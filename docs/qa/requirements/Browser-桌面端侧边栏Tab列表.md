# Browser - 桌面端侧边栏 Tab 列表需求文档

> **模块**：Browser（DApp）
> **功能名称**：桌面端侧边栏 Tab 列表
> **版本**：App-6.3.0
> **测试端**：Desktop

---

## 1. 需求背景

当前 Browser 桌面端采用竖向侧边栏承载 Tab 列表，涉及标签展示、分区、激活态、拖拽排序、关闭快捷键、关闭 / Unpin 结果以及新建 Tab 的插入偏好等一系列交互。为保证桌面端标签列表行为可预测、可验证，需要以“侧边栏 Tab 列表”作为完整功能模块描述，而不是仅以单一设置项描述。

---

## 2. 前置条件

- 已在 Desktop 端进入 Browser 页面，并可看到竖向侧边栏
- 侧边栏存在至少 2 个未固定（unpinned）Tab，便于观察新增与重排效果
- 可在 Browser 中打开新标签页、访问网站、Pin / Unpin Tab、拖拽调整 Tab 顺序

---

## 3. 功能概述与业务规则

### 3.1 标签列表基础展示与切换

业务规则：

- 侧边栏展开态下显示标签列表，并展示多个 Tab 项
- 当前打开的 Tab 在列表中显示选中态，且与当前页面内容保持一致
- 点击非激活 Tab 后，页面内容切换到对应标签页，激活态同步更新
- 同时存在 Pinned 与 unpinned 标签时，Pinned 区位于 unpinned 区上方

---

### 3.2 偏好项与默认值

业务规则：

- 新增持久化偏好 `newBrowserTabPosition: 'top' | 'bottom'`
- 默认值为 `bottom`，老用户升级后保持原有新增 Tab 追加到底部的行为
- 偏好项切换后，对后续新增 Tab 的插入位置立即生效

---

### 3.3 设置入口与菜单结构

业务规则：

- 仅在侧边栏展开态时，侧边栏底部右下角显示 Settings 图标
- 侧边栏收起态不显示 Settings 图标
- 点击 Settings 后弹出 ActionList
- ActionList 中提供 `新标签位置` 二级选项，包含 `顶部` 与 `底部`

---

### 3.4 顶部 / 底部模式下的新建 Tab 行为

业务规则：

- 选择 `顶部` 后，新建 Tab 插入到 unpinned 区顶部
- `顶部` 模式下连续新建多个 Tab，最新创建的 Tab 继续叠在最顶端
- 选择 `底部` 后，新建 Tab 追加到 unpinned 区底部
- 偏好切换仅影响新增 Tab 的插入位置，不主动重排已有 unpinned Tab

---

### 3.5 自动滚动与导航保持

业务规则：

- auto-scroll 跟随偏好：`top` 模式滚到顶，`bottom` 模式滚到底
- `top` 模式下新建的 Tab 在页内导航、URL 更新后，Tab 保持创建时位置，不因为 `timestamp` bump 跳到底部
- `bottom` 模式下 URL 变化行为保持旧逻辑

---

### 3.6 交互保护与弹出层行为

业务规则：

- Settings 的 ActionList 打开期间，通过 `reportPopoverOpen` 阻止侧边栏因 hover 区域变化而被误收起
- 鼠标移入弹出层时，侧边栏保持展开
- 关闭弹出层后，若鼠标已离开侧边栏与弹出层区域，侧边栏再按原逻辑收起

---

### 3.7 与 Pinned / unpinned / 拖拽 / 关闭 / Unpin 的边界关系

业务规则：

- Pinned Tabs 始终位于 unpinned 区之上，切换偏好不会打乱 Pinned 区顺序
- Pinned 与 unpinned 标签可分别在各自分区内拖拽排序，但保持分区隔离，不能拖动到另一分区
- 用户拖拽产生的现有排序不因偏好切换被覆盖
- unpinned 标签在列表展开时显示关闭按钮；关闭后从列表中移除；如果关闭的是当前打开的 unpinned 标签，窗口自动切换到列表中上方一个 Tab
- Pinned 标签在列表展开时显示 Unpin 按钮
- unpin 一个 Tab 后，该 Tab 固定回到 unpinned 区顶部，与 `新标签位置` 设置无关
- Pinned 标签不能通过列表内关闭按钮关闭，只能通过关闭快捷键执行关闭
- 关闭 Pinned 标签后，当前窗口切换到 Browser 首页，且该标签保留在列表中并变为非激活状态

---

### 3.8 持久化

业务规则：

- 重启 App 后仍保留上次选择的 `新标签位置` 偏好
- 重启后再次新建 Tab，插入位置与滚动方向继续遵循持久化后的偏好

---

## 4. 异常与鲁棒性规则

- Settings 弹出层打开与关闭过程中，侧边栏不出现误收起或无法收起
- 在 `top` 与 `bottom` 间反复切换偏好后，新建 Tab 的位置与自动滚动方向保持一致
- 当前激活 Tab 的选中态与实际页面内容保持一致，不出现选中错位
- 已有 Pinned / unpinned / 拖拽排序状态在切换偏好后保持可观测一致性
- Pinned 与 unpinned 之间的跨分区拖拽不产生越区排序
- 关闭当前打开的 unpinned 标签后，窗口自动切换到列表中上方一个 Tab
- 关闭 Pinned 标签后，当前窗口切换到 Browser 首页，且该标签保留在列表中并变为非激活状态
