# Referral - 硬件销售奖励（Hardware Sales Reward）

> 版本：待确认
> 测试端：iOS / Android / Extension / Desktop / Web
> 创建日期：2026-02-09

---

## 需求背景

硬件销售奖励是返佣模块的核心功能之一，用户通过分享邀请码/链接，被邀请人购买硬件产品后，邀请人可获得销售返佣。

---

## 功能模块

### 1. Overview 页面

#### 1.1 奖励概览

| 指标 | 说明 |
|------|------|
| Total earned | 已赚取总额，包含 Distributed + Undistributed |
| Distributed | 已发放金额 |
| Undistributed | 待发放金额 |
| Receiving address | 奖励接收地址，可编辑 |
| Next distribution | 下次发放时间 |

#### 1.2 等级体系

| 等级 | 返佣比例 | 销售门槛 |
|------|---------|---------|
| 铜牌 Bronze | 10% | 注册即获得 |
| 银牌 Silver | 15% | $300 |
| 金牌 Gold | 18% | $1,000 |
| 钻石 Diamond | 20% | $8,000 |

**等级规则**：
- 默认等级：铜牌（Bronze）
- 升级：达到销售门槛后立即生效
- 降级：每月10号考核用户销售金额，手动降级
- 后台手动修改：管理员可手动调整等级

#### 1.3 邀请方式

| 项目 | 说明 |
|------|------|
| Referral code | 用户的邀请码，可复制 |
| Referral link | 邀请链接（onekey.so/r/{CODE}），可复制 |

### 2. Hardware Sales 详情页

#### 2.1 详情页 Header（通用组件）

与链上奖励共用统一的筛选 Header，包括：
- 时间选择器
- 筛选按钮（Filter，按邀请码筛选）
- 导出按钮（Export）

#### 2.2 统计卡片

| 指标 | 说明 |
|------|------|
| Total reward | 筛选范围内的奖励总额 |
| Undistributed | 待发放金额 |
| Pending | 待入账金额（30天确认期内），下方显示确认天数 |

#### 2.3 Details 列表

**桌面端字段**：

| 字段 | 格式 | 排序 | 说明 |
|------|------|------|------|
| TIME | YYYY/MM/DD HH:mm | 可排序 | 订单时间 |
| ORDER ID | #XXXXXXXXX | 不可排序 | 订单编号 |
| STATUS | Completed / Pending / Refunded | 可排序 | 订单状态 |
| REFERRAL CODE | 标签样式 | 不可排序 | 绑定的邀请码 |
| AMOUNT | $X.XX | 可排序 | 订单金额 |
| REWARD | +$X.XX（绿色）/ -$X.XX（红色） | 可排序 | 返佣金额 |

**移动端列表**：
- 收起状态：订单号 + 状态标签 + 返佣金额
- 展开状态：显示完整订单信息

#### 2.4 订单详情弹窗

点击列表项展开详情，显示：

| 字段 | 说明 |
|------|------|
| Time | 订单时间 |
| Order ID | 订单编号 |
| Status | 订单状态（带颜色标识） |
| Rewards | 返佣金额 |
| Referral Code | 邀请码 |
| Order history | 订单状态时间线（倒序） |

#### 2.5 订单状态流转

| 订单状态 | 显示 | 返佣处理 |
|---------|------|---------|
| 未支付 | 不显示 | 不计入 |
| 下单成功 / 收货成功 | Pending | 待入账 |
| 下单 30 天后 | Completed | 待入账金额 → 待分发金额 |
| 退款 | Refunded | 新增退款记录，-销售金额，待入账金额减少 |

#### 2.6 订单状态时间线（Order History）

| 状态节点 | 说明 |
|---------|------|
| Order placed | 下单时间，显示 "Reward pending confirmation (≤ 30 days)" |
| Reward confirmed | 确认时间，显示 "Waiting for next distribution (expected by July 10)" |
| Refunded | 退款时间，显示 "Refunded by the buyer, reward revoked" |
| Reward distributed | 发放时间 |

### 3. 奖励发放规则

| 项目 | 规则 |
|------|------|
| 确认期 | 下单后 30 天确认 |
| 发放周期 | 每月 1 号 0 点，对当前待分配奖励金额进行快照 |
| 发放方式 | 发放到用户填写的奖励接收地址 |
| 未填地址 | 进入下个结算周期，待分发奖励金额不变 |
| 币种 | USDC |

### 4. 邀请下单流程

| 步骤 | 说明 |
|------|------|
| 1 | 被邀请人通过邀请链接/码进入商店 |
| 2 | 选择商品后下单，邀请码自动带入折扣券（折扣与返佣比例一致） |
| 3 | 下单成功后，邀请人端显示订单记录 |

---

## 空状态

| 场景 | 处理 |
|------|------|
| 无奖励 | 显示空状态文案 |
| 有奖励 | 显示待发放（USDC）+ 待入账（USDC），保留两位小数 |
