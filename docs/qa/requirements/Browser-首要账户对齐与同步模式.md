# Browser - 首要账户对齐与同步模式需求文档

> **模块**：Browser（DApp）
> **功能名称**：首要账户对齐与同步模式（对齐至钱包 / 独立模式 / 始终使用钱包账户）
> **版本**：待定
> **测试端**：全端（iOS / Android / Extension / Desktop / Web）

---

## 1. 需求背景

在 Browser（DApp）模块中，用户可在不同 DApp 标签页连接不同钱包账户与网络。为避免跨钱包/跨网络切换导致连接状态混乱，需要提供「对齐首要账户」配置项，并保证三种模式下的钱包账户与 DApp 账户/网络同步逻辑可预测、可验证。

---

## 2. 前置条件

- 已创建至少两个钱包账户（A 钱包、B 钱包）
- 分别拥有不同的账户（账户1、账户2）
- 当前 Solana 跟 EVM 路径为 BIP44

---

## 3. 功能概述与业务规则

### 3.1 对齐至钱包模式（将 dApp 账户对齐至钱包 / 默认）

配置项：`设置 - 对齐首要账户 - 默认选择「将 dApp 账户对齐至钱包」`

业务规则：

- 当某个 DApp 标签页仅连接单一钱包账户与单一网络时，钱包显示的账户与网络需要与该 DApp 连接状态保持一致。
- 当某个 DApp 标签页同时连接多个钱包账户与网络时，钱包侧不需要同步 DApp 的多连接状态。
- 不同 DApp 之间的切换不应互相影响：当在某个 DApp 标签页切换其连接账户/网络后，其他已连接 DApp 标签页的网络不受影响。
- 在 `设置 - 账户派生路径` 中切换 Solana 与 EVM 路径（BIP44 <-> Ledger Live）后：
  - 断开并重新连接 DApp 后，DApp 侧显示的地址需要随当前派生路径更新。
  - 在构建交易进入交易确认页时，交易确认页的地址应与当前派生路径下的地址一致。

---

### 3.2 独立模式

配置项：`设置 - 对齐首要账户 - 选择「独立模式」`

业务规则：

- 钱包侧与 DApp 标签页侧的账户与网络应完全独立：钱包首页的账户/网络切换不应改变 DApp 标签页的连接账户/网络。
- DApp 标签页内切换连接账户/网络后，钱包侧的账户/网络不应同步改变。

---

### 3.3 始终使用钱包账户

配置项：
- `设置 - 账户派生路径`：支持切换 Solana 与 EVM 路径（BIP44 <-> Ledger Live）
- `设置 - 对齐首要账户 - 选择「始终使用钱包账户」`

业务规则：

- 在该模式下，DApp 标签页连接的「账户」需要随钱包首页的账户切换而更新，但「网络」不随钱包网络自动同步。
- 当钱包切换到某账户但该账户未创建对应地址时：
  - DApp 的连接账户不需要同步到该未创建地址的账户。
  - 若出现创建地址引导，创建成功后新的账户应能够连接到对应 DApp。
- 私钥账户与观察地址之间：
  - 当切换到“同系列”的私钥账户或观察地址时，DApp 的账户切换与连接应生效。
  - 切换到“不同系列”的私钥账户或观察地址时，不应触发账户切换联动。
- 当在该模式下查看 DApp 连接管理：
  - DApp 侧不应出现网络与账户的切换入口（网络/账户由钱包侧驱动，且网络保持独立）。

---

## 4. 异常与鲁棒性规则

- 钱包或 DApp 账户断开后重新连接，需恢复到对应模式的同步逻辑。
- 在切换账户或网络过程中若发生中断操作（如关闭钱包、切换至其他应用）：
  - 不应产生错乱的账户/网络状态。
- 快速切换账户与网络时：
  - 钱包与 DApp 的响应应不出现阻塞或同步混乱。
- 在三种模式之间进行任意切换：
  - 不应出现跨模式错乱交互，需符合当前启用模式的逻辑。
