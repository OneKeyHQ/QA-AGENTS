# Prime - OneKey Cloud

> 模块：Prime  
> App 版本：待定  
> 日期：2026-03-20  
> 测试端：iOS / Android / Desktop / Extension / Web

---

## 1. 需求背景

OneKey Cloud 是 Prime 会员专属的数据同步能力，用于在同一 OneKey ID 下跨端同步账户名称、自定义配置、地址簿、书签等信息，并通过 OneKey ID 密码保障云备份安全。

---

## 2. 功能描述

### 2.1 开启条件

| 项目 | 说明 |
|-----|------|
| 自动锁定限制 | 当前为「永不」或「离开 4 小时」时不可直接开启 |
| 自动调整 | 用户确认继续后，自动锁定需调整为 4 小时以下并继续验证 APP 密码 |
| 首次开启 | 需勾选确认项并设置 OneKey ID 密码 |
| 非首次开启 | 需验证 OneKey ID 密码；错误密码可重试 |

### 2.2 OneKey ID 密码规则

| 项目 | 说明 |
|-----|------|
| 检查条件 | 12 字符、数字、字母、英文特殊字符 |
| 复杂度分级 | 弱 / 良 / 强 |
| 提交条件 | 满足全部检查条件、复杂度不为弱、两次输入一致 |
| 输入框行为 | 支持可见 / 不可见切换；可见状态下允许复制 |

### 2.3 同步范围

| 同步对象 | 说明 |
|---------|------|
| HD / HW / QR / 隐藏钱包账户名 | 修改后同步到其他端，旧名称进入设备历史 |
| 私钥 / 公钥 / 观察钱包账户名 | 仅同步对应账户 |
| 空 passphrase 隐藏钱包与标准钱包 | 名称分别同步 |
| 自定义代币 / 网络 / RPC | 支持新增与修改同步 |
| 浏览器书签 | 支持新增、修改、排序同步 |
| 市场观察列表 | 支持新增与修改同步 |
| 地址簿 | 支持新增与修改同步 |
| 自定义 RPC 开关 | 开关状态同步到其他端 |

### 2.4 同步触发与冲突处理

| 项目 | 说明 |
|-----|------|
| 异步触发 | 打开账户选择器、地址簿、管理 Token、网络选择器、浏览器首页、Market 首页等入口时触发 |
| 异步间隔 | 上述异步同步默认 5 分钟间隔 |
| 立即同步 | 云同步开关、断线重连、重新激活 Prime、手动同步可立即触发 |
| 冲突原则 | 同一数据多端修改时按最后一次修改结果生效 |
| 冲突示例 | 关闭同步期间多端改名，重新开启后按最后修改结果覆盖 |

### 2.5 密码变更与重置

| 项目 | 说明 |
|-----|------|
| 修改密码 | 需先验证 APP 密码，再验证 OneKey ID 密码 |
| 其他端表现 | 其他登录端提示密码已被修改或重置，并自动关闭云同步开关 |
| 忘记密码 | 需勾选删除云备份确认项后才可删除 |
| 删除结果 | 删除云备份后自动关闭云同步开关，并隐藏「更改 OneKey ID 密码」入口 |

## 3. 业务规则

| 规则项 | 规则描述 |
|-------|---------|
| 会员限制 | OneKey Cloud 为 Prime 专属功能 |
| 开启前置 | 自动锁定不满足条件时需先调整锁定时长 |
| 首次启用 | 首次开启必须设置 OneKey ID 密码 |
| 密码校验 | 密码需满足长度、字符类型、复杂度与确认一致性要求 |
| 同步范围 | 账户名、自定义代币/网络/RPC、书签、观察列表、地址簿等支持跨端同步 |
| 排序同步 | 浏览器书签排序需跨端同步 |
| 开关同步 | 自定义 RPC 开关状态需跨端同步 |
| 冲突解决 | 多端冲突统一按最后一次修改结果生效 |
| 密码变更传播 | 密码变更或重置后其他端需感知并关闭云同步 |
| 忘记密码处理 | 删除云备份后需清除原有云同步密码管理入口 |

---

## 4. 已知风险

- 异步同步 5 分钟间隔内的用户感知需与产品预期保持一致
- 断网重连、重新激活 Prime、手动同步的优先级和去重策略待验证
- 多端同时修改同一字段时的冲突时间基准待确认
- OneKey ID 密码复杂度计算规则需与客户端实现保持一致
