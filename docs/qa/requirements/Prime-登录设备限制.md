# Prime - 登录设备限制

> 模块：Prime  
> App 版本：待定  
> 日期：2026-03-20  
> 测试端：Desktop / Extension / iOS / Android / Web

---

## 1. 需求背景

Prime 会员资格与 OneKey ID 绑定。为控制会员权益在多设备间的同时生效范围，需要对同一 OneKey ID 的登录设备进行有效设备与超限设备判定，并在当前客户端展示自身对应状态。

---

## 2. 功能描述

### 2.1 有效设备上限

| 项目 | 说明 |
|-----|------|
| 生效前提 | 同一 OneKey ID 已开通 Prime，且登录到多台设备 |
| 有效设备数量 | 仅 5 台设备可同时生效 Prime 权益 |
| 超限判定 | 设备数未满 5 台时，新登录设备直接生效；满 5 台后新增设备为超限设备 |
| 状态稳定 | 仅刷新最后活跃时间不会改写当前生效设备集合 |

### 2.2 设备状态展示

| 项目 | 说明 |
|-----|------|
| 有效设备标签 | 当前登录客户端处于前 5 台有效设备内时显示绿色标签 |
| 超限设备标签 | 当前登录客户端处于超限状态显示黄色标签 |
| 显示范围 | 标志颜色仅在当前登录客户端对应的 Prime 状态区域展示，不直接展示其他客户端的有效/超限状态 |

### 2.3 设备列表与活跃时间

| 项目 | 说明 |
|-----|------|
| 排序规则 | 设备列表按最后活跃时间排序，供展示与补位使用 |
| 活跃刷新 | 访问 OneKey ID 相关功能会刷新最后活跃时间；仅影响排序，不直接改写当前生效设备集合 |

### 2.4 设备移除与登出

| 操作 | 结果 |
|-----|------|
| 移除超限设备 | 不影响当前生效设备集合 |
| 移除未超限设备 | 生效设备减少后，由最近活跃的超限设备补位 |
| 未超限设备登出 | 生效设备减少后，由最近活跃的超限设备补位 |

---

## 3. 业务规则

| 规则项 | 规则描述 |
|-------|---------|
| 权益并发上限 | 同一 OneKey ID 同时最多 5 台设备生效 Prime 权益 |
| 设备占位 | 设备数未满 5 台时，新登录设备直接占位；满 5 台后新增设备为超限设备 |
| 标签语义 | 绿标表示当前客户端为有效设备；黄标表示当前客户端为超限设备 |
| 当前端展示 | 仅展示当前登录客户端自身状态，不代表其他客户端的有效/超限状态 |
| 活跃时间 | 最后活跃时间用于排序与补位，不直接改写当前生效设备集合 |
| 移除逻辑 | 移除超限设备不影响当前状态；移除生效设备后由最近活跃的超限设备补位 |
| 登出补位 | 生效设备主动登出后，由最近活跃的超限设备补位 |

---

## 4. 已知风险

- 多端设备状态同步存在短暂延迟时的展示策略待确认
- 同一物理设备的不同端形态是否独立计入设备数待明确
- Web 端是否计入设备限制范围需与实际产品策略持续对齐
