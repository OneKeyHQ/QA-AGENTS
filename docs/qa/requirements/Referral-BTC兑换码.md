# Referral - BTC 兑换码（OneKey BTC 奖励兑换）

> 版本：6.3.0
> 测试端：iOS / Android / Extension / Desktop / Web
> 创建日期：2026-04-30

---

## 需求背景

从 2026 年中大促开始，OneKey 促销不再打折，改为赠送等额 BTC 奖励：
- 用户购买硬件钱包后，包装内附「BTC 兑换卡」（类似礼品卡），印刷有 16 位兑换码
- 用户在 OneKey App 内通过兑换码领取等额 cbBTC 奖励
- 30 天退货期 + 每月 10 日统一发放
- 兑换码后台分类：**OneKey BTC Reward**

兑换码格式：`QA99-7V1H-YSYW-T6MW`
- 4 段 × 4 字符
- 字符集：大写字母 + 数字
- 段间用 `-` 分隔
- 总长度 19 字符（含 3 个分隔符）

## 业务全链路（运营 + 客户端）

```
[运营 Dashboard]      [线下印刷发货]       [客户端兑换]         [后端等待期]          [手动发放]
  创建活动 →     →     印刷兑换卡 →    →    输入兑换码 →    →    30 天退货期 →    →   生成快照
  分批生成码           塞入产品包装           输入凭证码                                导出 CSV
  导出 TXT                                  验证订单                                  链上打款 BTC
                                            选地址确认                                上传发放 CSV
                                                                                     推送到账通知
```

**码状态流转：**
```
未使用 → 待发放 → 发放中 → 已发放
            ↓ 人工拒绝
          已拒绝
未使用 → 作废 → 已作废
```

| 状态 | 含义 | 用户端展示 |
|------|------|-----------|
| 未使用 | 兑换码已生成但用户未输入 | （用户不可见） |
| 待发放 | 用户已提交兑换；30 天退货期内显示「等待中（30 天）」，期满后显示「待发放」 | 等待中（30 天）/ 待发放 |
| 发放中 | 已纳入快照，运营导出 CSV 准备链上打款 | 发放中 |
| 已发放 | CSV 上传确认，链上打款完成 | 已发放 |
| 已拒绝 | 待发放状态下被运营人工拒绝 | 已拒绝 |
| 已作废 | 未使用状态下被运营作废 | （用户不可见） |

> **作废与拒绝的边界**：
> - **作废仅限未使用状态**：只有未被绑定订单的码可被作废；作废后用户输入该码 → 走销售码兜底流程，不出现在历史记录
> - **已使用码无法作废**：一旦码已绑定订单（进入待发放及之后状态），运营只能选择「人工拒绝」，**不能改为作废**
> - 作废与拒绝路径不可逆向变更
>
> **状态强制覆盖**：运营在后台 Dashboard 可直接修改状态（如跳过 30 天退货期），但需遵守「作废仅限未使用 / 已使用只能拒绝」的边界约束。
> 所有码均需关联订单。活动赠码等场景由后端预关联固定订单，用户无需手动输入凭证码。

### 操作按钮
| 按钮 | 颜色 | 行为 |
|------|------|------|
| 人工拒绝 | 红色 | 将状态置为「已拒绝」，奖励不发放（待发放状态下可用） |
| 立即可发放（测试用） | 蓝色 | 跳过 30 天退货期，直接将状态推进到「待发放（已可发）」（仅测试环境） |

### 用户端联动
- 运营修改状态后，**提交该兑换的设备**历史记录 / 奖励详情实时同步新状态（其他设备因无本地记录不受影响）
- 「人工拒绝」后该设备状态徽章变为「已拒绝」，状态描述更新为拒绝相关文案

## 功能模块

### 1. 入口
- 路径：菜单 → 更多 → 兑换（票券图标）
- 与销售码共用同一入口（兑换中心 Modal）
- **无需登录 OneKey ID 即可进入并输入码**

### 2. 兑换中心 Modal（输入兑换码）

#### 2.1 UI 元素
| 元素 | 文案 / 行为 |
|------|------------|
| 标题 | 兑换中心 |
| 副标题 | 输入兑换码以领取专属奖励 |
| 历史记录入口 | 顶部「历史记录」链接 / 票券图标 |
| 输入框 | 默认 placeholder 为空；自动大写、连字符自动补齐（每 4 字符 `-`） |
| 兑换按钮 | 输入为空时置灰；输入完整 16 位字符后可点击 |

#### 2.2 输入识别与路由
- 命中 BTC 格式（`/^[A-Z0-9]{4}-[A-Z0-9]{4}-[A-Z0-9]{4}-[A-Z0-9]{4}$/`）且后端校验**存在且未使用** → 进入 **验证凭证页**
- 以下情形**全部走销售码兜底流程**（弹 OneKey ID 登录对话框，不显示任何 BTC 流程提示）：
  - 不命中 BTC 格式
  - 命中 BTC 格式但后端不存在
  - 命中 BTC 格式但**已被使用过**（已绑定订单）
  - 命中 BTC 格式但**已被后台作废**
- **关键规则**：兑换码与设备 1:1 绑定，单个兑换码**仅可使用一次**，不存在「多次码」概念。订单含 N 个 BTC Card → 生成 N 个独立兑换码。

### 3. 验证凭证页（输入凭证码）

#### 3.1 UI 元素
| 元素 | 文案 / 行为 |
|------|------------|
| 标题 | 验证凭证 |
| 描述 | 输入您的凭证码，将其与兑换码关联。 |
| 凭证码输入框 | placeholder：`例如 #1001` |
| 帮助提示 | 在哪里找到您的凭证码？<br>· Shopify 订单：确认邮件或 OneKey Shop 的订单详情<br>· 线下活动或经销商：BTC 卡上的凭证贴纸 |
| 验证并继续按钮 | 输入为空时置灰；有内容时可点击 |

#### 3.2 凭证码校验
| 场景 | 处理方式 |
|------|---------|
| 凭证码无效 / 不存在 | 输入框红色边框 + 红字提示「订单无效或可兑换数量已用完」 |
| 凭证码已用尽（超过最大兑换次数） | 同上 |
| 凭证码合法但订单**不包含 BTC Card 商品** | 同上（绑定失败） |
| 凭证码合法且订单包含 BTC Card 商品且仍有可兑换次数 | 进入选择收款地址页 |

> **订单商品校验**：除凭证码本身合法之外，还会校验订单 line items 中是否包含 BTC Card 类商品；若整张订单仅包含其他商品（如硬件钱包、配件），即使订单号正确也会绑定失败。
> **凭证还剩 X 次兑换机会**：兑换码与设备 1:1 绑定（每张 BTC Card 对应一个独立兑换码）；订单下所有未被绑定的 BTC Card 数量即为该凭证的剩余兑换数。每次成功兑换后，该订单的剩余数 -1。订单下全部 BTC Card 兑换完毕后，再输入该订单号提示「订单无效或可兑换数量已用完」。
> 活动赠码等场景由后端预关联订单，用户无需输入凭证码（直接跳过此页面）。

### 4. 选择收款地址页

#### 4.1 UI 元素
| 元素 | 文案 / 行为 |
|------|------------|
| 标题 | 选择收款地址 |
| 网络 | **固定为 Base 链**（cbBTC 在 Base 上发放，无网络切换入口） |
| 地址列表 | 仅显示有私钥的钱包账户（HD 钱包 / HW 钱包 / 私钥钱包 / Keyless Wallet）显示 EVM 地址 |
| 当前选中行 | 头像 + 钱包名/账户名（如 `ran / Account #1`）+ 缩略地址（如 `0x2DB700...E73A42`） |
| 警示框（黄色） | 提交后无法更改地址<br>仅显示有私钥的钱包。继续前请仔细确认。 |
| 剩余次数提示（左下） | 此凭证还剩 X 次兑换机会（X = 订单中 BTC Card 数量 - 已兑换次数） |
| 下一步按钮 | 选中地址后可点击 |

#### 4.2 账户筛选规则
| 钱包类型 | 是否显示 |
|---------|--------|
| HD 钱包（已备份） | ✅ |
| HW 钱包 | ✅ |
| 私钥钱包 | ✅ |
| Keyless Wallet | ✅ |
| 观察账户 | ❌ |
| 外部账户 | ❌ |

#### 4.3 空状态（无可用钱包）
当用户仅有观察账户 / 外部账户、不存在任何有私钥钱包时：
- 顶部钱包选择器显示空状态：钱包图标 + 占位文字「选择钱包」+ 右侧下拉箭头
- 黄色警示框照常显示
- 左下角剩余兑换次数照常显示
- 「下一步」按钮置灰不可点击
- 点击钱包选择器：弹出列表为空

## 边缘情况与产品逻辑

### 1️⃣ 兑换码格式与校验
| 场景 | 处理方式 |
|------|---------|
| 输入空 | 兑换按钮置灰 |
| 输入纯数字 | 不满足 BTC 正则 → 走销售码兜底流程（弹登录对话框） |
| 输入小写字母 | 自动转大写后参与校验 |
| 含空格 | Trim 后校验 |
| 截断少于 16 位 | 兑换按钮置灰 |
| BTC 格式但**不存在** | 走销售码兜底流程，**无 BTC 流程提示** |
| BTC 格式但**已被使用过** | 走销售码兜底流程，**无 BTC 流程提示** |
| BTC 格式但**已作废** | 走销售码兜底流程，**无 BTC 流程提示** |

### 2️⃣ 凭证码校验
| 场景 | 处理方式 |
|------|---------|
| 凭证码空 | 「验证并继续」按钮置灰 |
| 凭证码格式错误 / 不存在 | 输入框红框 + 提示「订单无效或可兑换数量已用完」 |
| 凭证码可兑换次数已用尽 | 同上 |
| 凭证码对应订单不含 BTC Card 商品 | 同上（绑定失败，文案统一） |
| 同一凭证码可兑换多次 | 验证通过，下一页显示「此凭证还剩 X 次兑换机会」 |

### 3️⃣ 收款地址校验
| 场景 | 处理方式 |
|------|---------|
| 仅观察账户 / 外部账户 | 列表为空，无法继续 |
| 多账户 | 默认选中第一个有私钥的账户，可下拉切换 |
| 切换账户 | 仅在有私钥账户（HD / HW / 私钥 / Keyless）范围内切换 |

### 4️⃣ 提交后限制
| 场景 | 处理方式 |
|------|---------|
| 已提交记录 | 不能再次更改收款地址 |
| 等待中状态被运营拒绝 | 状态变为已拒绝，无奖励发放 |
| 退货期内退货导致订单失效 | 状态可能变为已拒绝（按运营规则） |

### 5️⃣ 历史记录
| 场景 | 处理方式 |
|------|---------|
| 无记录 | 显示空状态 |
| 查看详情 | 点击列表项进入奖励详情页 |
| 存储位置 | **仅本地存储，不跨端同步**；同一 OneKey ID 在不同设备上各自维护独立历史 |
| 状态更新 | 提交该兑换的设备可从后端拉取最新状态；其他设备因无记录无需同步 |

### 6️⃣ 锁价机制
- 提交瞬间按 OneKey 后端的实时 BTC 价格锁定 cbBTC 数量
- 锁定价格在详情页持续展示（不随时间波动）
