# Market - 法币切换需求文档

> **模块**：Market（行情）
> **功能名称**：法币切换（统一 USD 计价 + 次级法币）
> **版本**：
> **测试端**：iOS / Android / Extension / Desktop / Web

---

## 1. 需求背景

当前 Market 模块中，代币价格会跟随用户在系统设置中选择的偏好法币（默认货币）进行转换显示。链上 99% 的资产天然以美元计价，将价格转换为其他法币后存在：

- **行业惯例不符**：OKX、Binance、GMGN 等在 Dex/Market 模块均采用 USD 固定计价
- **模块对齐**：Perps 模块已采用固定美元计价，Market 应与之对齐

---

## 2. 核心改动：统一美元计价

Market 整个页面及所有子模块中，所有价格展示统一固定为美元（USD）计价，不再跟随用户的默认货币设置进行法币转换。

### 2.1 涉及范围

| 范围 | 说明 |
|-----|------|
| 代币列表价格 | 列表页所有代币价格 |
| Token Details 主价格 | 详情页核心价格 |
| K 线图表价格 | K 线 Y 轴及十字线价格 |
| 交易量（Volume） | 所有 Volume 展示 |
| 搜索 | 搜索结果中的价格 |
| 交易记录 | Market 模块内交易记录的价格 |
| 其他 | Market 模块内所有价格数据 |

### 2.2 前端改动

- Market 模块内所有价格展示组件，硬编码使用 USD 计价，不再读取全局货币偏好设置
- K 线图表保持 USD 数据源不变（修复因货币切换导致的显示 bug）

### 2.3 后端改动

- 若接口返回的价格已基于用户货币偏好转换，需改为统一返回 USD 价格

---

## 3. 次级价格展示（Token Details）

> 仅在 Token Details 页面（移动端、桌面端）增加次级价格，其他 Market 页面无任何法币转换。

### 3.1 规则

| 规则项 | 规则描述 |
|-------|---------|
| 展示条件 | 仅当用户系统设置中的默认货币为非美元时，才展示次级价格 |
| 展示内容 | 当前代币 USD 价格按实时汇率换算为用户偏好法币价格 |
| 展示位置 | 紧贴主价格下方，字号和视觉权重低于主价格 |
| 展示格式 | 法币符号 + 金额，如用户偏好人民币时显示 ¥7.28 |
| 默认状态 | 用户设置为 USD 时，不展示次级价格行，避免重复 |

### 3.2 交互设计

| 端 | 说明 |
|----|------|
| 移动端 | 主价格区域展示 USD；主价格下方新增一行次级文案，展示用户偏好法币等值价格；次级字号较小、颜色较浅 |
| 桌面端 | 同移动端逻辑 |

### 3.3 汇率换算

- 可在前端完成（使用现有汇率数据），或由后端新增字段返回

---

## 4. Pro Mode 特殊情况

| 规则项 | 规则描述 |
|-------|---------|
| 我的仓位 | 字段与其他模块共用，保留现有逻辑（跟随用户货币偏好） |

---

## 5. 边缘情况

| 场景 | 规则 |
|-----|------|
| 汇率数据缺失 | 次级价格区域不展示（静默隐藏），不影响主价格 |
| 极端汇率精度 | 如越南盾等，需合理处理数字格式化 |
| 用户切换货币设置 | 修改默认货币后，返回 Token Details 时次级价格应实时更新 |
| 价格为 0 或无数据 | 代币 USD 价格为 0 或无数据时，次级价格不展示 |
