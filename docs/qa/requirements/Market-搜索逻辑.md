# Market - 搜索逻辑需求文档

> **模块**：Market（行情）
> **功能名称**：搜索逻辑（表格样式 + 热门代币）
> **版本**：
> **测试端**：iOS / Android / Extension / Desktop / Web

---

## 1. 需求概述

优化 Market 搜索功能的展示样式，将搜索结果和热门代币列表改为表格（Table）样式，提升信息密度和可读性；同时调整热门代币的数据源和点击交互逻辑。

---

## 2. Market 搜索结果 - 表格样式

### 2.1 表格字段（从左到右）

| 字段 | 说明 | 备注 |
|-----|------|------|
| 收藏 | 星标收藏按钮 | 放在最前面，点击可收藏/取消收藏 |
| 名称 | 代币图标 + 名称 + 验证标识 + 合约地址 | 图标右下角显示网络标识，地址可复制 |
| 价格 | 当前价格 + 24h涨跌幅 | 涨跌幅绿涨红跌 |
| 流动性 | Liquidity 数值 | |
| 24h成交额 | 24小时交易量 | |
| 市值 | Market Cap | |
| 数量 | 在 All 分类下展示前三即可 | |

### 2.2 UI 交互

| 规则项 | 规则描述 |
|-------|---------|
| 表头固定 | 表头在 Market Tab 固定，内容区域纵向滚动 |

## 3. 热门代币列表（Trending）

### 3.1 数据源更换

| 规则项 | 规则描述 |
|-------|---------|
| 数据源 | 从当前源更换为 Market 数据源 |
| 展示样式 | 与 Market 搜索结果保持一致（表格样式） |
| 字段 | 收藏、名称、价格/涨跌幅、流动性、24h成交额、市值 |

### 3.2 点击交互优化

| 规则项 | 规则描述 |
|-------|---------|
| 点击代币行 | 直接跳转到 Market 交易页面 |
| 跳过中间页 | 跳过当前 App 中的代币详情中间页 |

### 3.3 列表生成逻辑（后端提供）

| 规则项 | 规则描述 |
|-------|---------|
| 获取范围 | 从接口获取前 30 的热门代币 |
| 网络确定 | 根据最近 24 小时的交易量，确定某个代币的主要所在网络 |
| 去重规则 | 每个代币只取其当前所在的主要网络作为一条数据 |
| 展示数量 | 最终在列表里面展示最新 15 条（线上 15 条） |
| BTC 特殊 | 对于 BTC 代币需要特殊处理 |

---

## 4. 边缘 Case 与异常处理

### 4.1 数据相关

| 场景 | 处理方案 |
|-----|---------|
| 代币价格/成交额等字段为空或为 0 | 显示「--」或「$0」，不隐藏该行 |
| 代币图标加载失败 | 显示默认占位图标 |
| 数值过长（如价格小数位过多） | 统一格式化规则：价格保留有效数字，大数值用 K/M/B/T 缩写 |

### 4.2 UI 适配相关

| 场景 | 处理方案 |
|-----|---------|
| 代币名称过长 | 截断显示，末尾加省略号 |
| 合约地址显示 | 固定格式：0x1234...abcd（前6位+后4位） |
