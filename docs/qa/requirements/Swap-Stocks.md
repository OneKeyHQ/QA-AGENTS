# Swap Stocks

> 模块：Swap
> 功能：Trade 下新增股票（Stocks）交易分类
> 版本：
> 测试端：Android / iOS / Desktop / Extension / Web
> 关联资源：用户提供的移动端 Stocks 交易页、股票代币选择页、桌面端 Stocks 页面截图

---

## 需求背景

Trade 页面需要新增股票（Stocks）分类，用于承接受监管股票映射代币的买卖场景。本期优先级高于 Swap 与 Bridge 分类合并，因此先在现有 Trade 结构中直接新增 Stocks 分类，后续再处理分类合并。

---

## 功能描述

### 1. Tab 与布局

- Trade 页面新增股票（Stocks）分类，和现有 `Swap & Bridge`、`Pro` 等分类并列。
- 小屏移动端分类栏超出一屏时，使用可左右滑动的选择器：
  - 调整 Tab 间距
  - 右侧 3 个选项保持可见
  - 其余选项通过横向滑动查看
- 大屏范围内保持全部分类直接展示。

### 2. 股票交易资产与渠道

- Stocks 模式使用维护的股票交易代币列表。
- 本期仅开放 BNB Chain 维护列表，后续再扩展 ETH Ondo、xStock、Solana 列表与网络切换能力。
- 交易渠道固定为 1inch Fusion，后续可扩展 Near。
- 可买入 / 卖出的稳定币列表由 Dashboard 配置，本期默认配置为 `USDC / USDT`。

### 3. 交易面板

- 当前股票代币展示区显示：
  - Ticker
  - 公司名称
  - 发行商
  - 交易状态
  - 价格
- 上述展示数据从 Market 模块获取。
- K 线按钮点击行为参考 Pro，跳转到 Market Token details，而不是复用 Swap & Bridge 的简化 K 线弹层。

### 4. 设置项

- Stocks 交易统一走 REQ 模式。
- 设置面板中不显示滑点设置。
- 除滑点外，其余与 Swap 共用的设置项保留。

### 5. Buy

- 默认支付币种只能是稳定币，不支持主币。
- 可选支付币种来源于 Dashboard 配置。
- 默认支付币种选择规则：
  - 比较用户在 BSC 上 `USDC` 与 `USDT` 余额
  - 余额较多者作为默认币种
  - 若两者余额都为 0，则使用 Dashboard 返回列表中的第一个
- 用户主动切换过支付币种后，客户端记录该选择，后续沿用最后一次切换结果。
- 预计收到（Estimated received）、汇率（Rate）、渠道商展开信息复用 Swap 逻辑。
- 汇率刷新沿用 Swap 自动刷新与最大刷新次数逻辑。

### 6. Sell

- Sell 与 Buy 主流程一致。
- 卖出币种固定为当前 Ticker 位置的股票代币，不在支付币种选择器中切换。
- 切换顶部股票代币后，Sell 的卖出币种随之变化。
- 预计收到默认是稳定币，默认稳定币选择逻辑与 Buy 一致。

### 7. My position

- Stocks Tab 下的持仓列表仅统计维护的股票代币持仓，不包含其他普通代币持仓。
- 其余持仓数据获取与展示方式沿用 Swap 现有逻辑。

### 8. 股票代币切换

- 股票代币选择器展示维护的股票代币列表。
- 本期仅展示 BNB Chain 列表，不提供网络切换入口。
- 搜索支持关键词和合约地址。
- 搜索范围仅限维护列表中的股票代币。

### 9. Review Order 与历史

- Stocks 模式沿用 Swap 的询价、构建订单、确认、签名与提交流程。
- 提交后的订单新增到交易历史。

### 10. 常见报错与交易控制

- 报错由后端下发，包含：
  - 错误信息
  - 交易控制字段
- 前端根据错误信息展示对应文案。
- 前端根据交易控制字段决定是否禁用交易按钮。
- 需要覆盖的错误类型包括：
  - 未知错误
  - 最小金额限制：`20 USDC`
  - 最大金额限制：`100K`
  - 地区限制
  - 休市
- 休市场景细分：
  - 若 Perps 存在同标的 Ticker，提示前往 Perps
  - 若无同标的 Ticker，展示开市倒计时
  - 若无法获取倒计时，提示等待开市

### 11. Desktop 专项

- Desktop 交易流程与移动端一致。
- 页面额外展示 K 线与 Market Data 区域。
- K 线直接使用 TradingView，默认选中 `1h`。
- Market Data 数据与 tooltip 均从 Market 模块获取。

## 业务规则

| 规则 | 描述 |
|------|------|
| 分类入口 | Trade 下新增 `Stocks` 分类 |
| 当前网络范围 | 本期仅开放 BNB Chain 维护列表 |
| 当前交易渠道 | 固定 `1inch Fusion` |
| 默认稳定币 | `USDC / USDT`，按余额较多者优先，均为 0 时取 Dashboard 第一个 |
| 稳定币记忆 | 用户手动切换后沿用最后一次选择 |
| 设置页差异 | Stocks 模式不显示滑点设置 |
| 持仓范围 | 仅统计维护列表内的股票代币 |
| 搜索范围 | 仅搜索维护列表中的股票代币 |
| K 线入口 | 点击后进入 Market Token details |
| Desktop 图表 | TradingView，默认 `1h` |

## 已知风险

- 股票代币列表、支持链、交易渠道、稳定币列表都属于可配置项，后续扩容时需要同步回归搜索范围、默认币种、持仓过滤与历史记录展示。
- Market 数据、交易状态、倒计时、Perps 同标的判断分别依赖不同后端数据源，联调时需要覆盖多接口返回组合。
- 小屏横向滑动 Tab 与桌面端大屏完整展示属于双布局路径，需要分别回归。
