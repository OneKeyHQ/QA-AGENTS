## 1. 需求背景

将 Perps 新下单入口从内嵌 TradingView 图表内**移出**，改为 App 原生渲染的下单面板，由图表的 `+` / 右键菜单在指定画线点位上唤起。既保证下单交互与主面板一致的风控/确认门禁，又让用户可以直接在图表关注价位上快速挂限价单或为持仓设置止盈/止损。

## 2. 功能范围

### 2.1 入口与平台 gating
- 图表 `+`（或右键）菜单，菜单项含当前画线水平线点位价格。
- 仅在**大屏非原生端**显示下单菜单（`gtMd && !platformEnv.isNative`）；移动端 / native 不显示。
- 受 `enablePerpsTradingUi` 控制；老版本 chart bundle 不显示菜单（version skew 安全）。部署顺序：先 chart bundle，后 App。

### 2.2 菜单意图（intent）
- 「交易 <SYMBOL> @ <price> 限价」→ `limitEntry`（限价下单面板）。
- 「交易 <SYMBOL> @ <price> 止盈」→ `positionTpSl`（仓位止盈/止损面板，**仅有持仓时显示**）。
- 「绘制水平线点位：<price>」→ 仅画线，不下单。

### 2.3 限价下单面板（LimitOrderForm）
- 标题「限价单 · <SYMBOL>」+ 订单类型 info 图标。
- 可用余额 / 当前持仓摘要卡。
- 价格（预填画线价格 + `Mid` 标签）、最优价（BBO）、数量、百分比 slider、只减仓、止盈/止损、TIF GTC、买入做多 / 卖出做空、成本、预估强平价。
- 独立本地 state，不写全局 `tradingFormAtom`。
- 提交走 `canTrade` 门禁 + 订单确认弹窗。

### 2.4 仓位止盈/止损面板（SetTpslModal）
- 标题「仓位止盈/止损」+ 副标题。
- 资产 / 仓位 / 开仓价格 / 标记价格。
- 止盈价格（预填画线价格）+ 百分比 + 预期利润；止损价格 + 亏损%；部分仓位；确认订单。
- 方向随持仓方向匹配。

### 2.5 边界与回归
- 无可用保证金 + 选 slider 百分比 → 提示「保证金不足 / 余额不足」（OK-56739）。
- 未连接外部钱包 → 显示「连接钱包 / 创建地址」，非「启用交易」（OK-56723）。
- 账户 index 在刷新后保留。

## 3. 已知风险点（用于测试聚焦）
1. 平台 / 版本 gating 错误（移动端误显示、老 bundle 误显示）。
2. 面板价格预填与图表画线点位不一致。
3. 面板 state 泄漏到主下单面板（`tradingFormAtom`）。
4. 只减仓 / TP/SL 语义未正确透传到 SDK。
5. 无持仓时误显示「止盈」菜单项。
6. 零余额 / 无钱包时的提示文案错误。
