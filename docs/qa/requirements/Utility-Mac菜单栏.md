# Utility - Mac 菜单栏（Menu Bar / Tray）

> 模块：Utility（通用）
> App 版本：待定
> 日期：2026-04-27
> 测试端：Desktop（macOS 主，Windows / Linux 跨平台隔离验证）
> 关联 PR / 代码：`https://github.com/OneKeyHQ/app-monorepo/tree/x`（参考实现，可能含 bug）

---

## 1. 需求背景

macOS 客户端在系统菜单栏（Menu Bar / Tray）新增 OneKey 图标，提供常驻的"快速一瞥"入口：用户无需打开主窗口即可查看当前钱包账户、投资组合资产、自选行情和待处理交易。点击菜单栏图标弹出 360x480 悬浮面板，面板支持快速跳转主窗口对应页面，以及通过原生系统通知提醒交易状态变化。

Windows / Linux 暂不支持，确保不出现 Tray 入口、不报错、不影响现有功能。

---

## 2. 功能描述

### 2.1 应用启动与常驻行为（macOS）

| 项目 | 说明 |
|-----|------|
| 启动后图标显示 | 客户端启动后，系统菜单栏出现 OneKey 图标 |
| 面板尺寸 | 360 × 480（点击图标弹出） |
| 失焦自动隐藏 | 面板失焦（点击外部）自动收起 |
| Esc 隐藏 | 面板内按 Esc 自动收起 |
| 唯一实例 | 同一时刻只允许一个面板实例 |

### 2.2 设置页 Preferences（macOS）

| 项目 | 说明 |
|-----|------|
| 入口位置 | `Settings > Preferences > Menu Bar` 开关 |
| 开 → 关 | 立即销毁菜单栏图标，面板若打开同步关闭 |
| 关 → 开 | 立即创建菜单栏图标 |
| 持久化 | 重启后保持用户上次选择 |
| 升级默认 | 旧版本升级且无该字段历史值时，默认开启 |

### 2.3 Tray 面板内容（macOS）

| 区块 | 内容 |
|-----|------|
| 顶部账户区 | 钱包名称 / 头像（emoji） / Account #N / 总资产 USD 值 |
| 涨跌区 | 24h 投资组合涨跌（百分比 + 金额，正绿负红） |
| 自选行情列表 | 用户在主窗口收藏的 Token / Perps 合约（图标、名称、链、价格、24h 涨跌） |
| 待处理交易 | 当前未完成交易（pending），含交易类型、金额、状态徽标 |

**空态：**

| 状态 | 文案/UI |
|-----|--------|
| 无钱包 | 引导文案 + 「打开主窗口创建/导入钱包」按钮 |
| 应用锁定 | 显示 "App is Locked" 文案，停止有效轮询 |
| 无自选 / 无待处理交易 | 区域内显示对应空态文案（如 "暂无自选" / "暂无待处理交易"） |
| 网络离线 | 显示离线提示，数据保留上次缓存 |

### 2.4 Tray 面板点击跳转（macOS）

> 自选条目按数据源走两条独立路径；待处理交易点击不跳转独立页面，而是在主窗口内打开对应历史记录的弹窗。

| 点击位置 | 数据源 | 行为 |
|---------|-------|-----|
| 顶部账户区（资产头部） | 当前账户 | 唤起主窗口 → 跳转到首页 / 当前账户概览 |
| 自选现货条目 | Market | 唤起主窗口 → 跳转到 Market 下的该代币详情页 |
| 自选合约条目 | Perps | 唤起主窗口 → 跳转到 Perps 下的该合约详情页 |
| 待处理交易条目 | 当前账户的 pending | 唤起主窗口 → 在主窗口内**打开对应历史记录的弹窗**并定位该笔交易（不跳转独立页面） |

### 2.4.1 账户隔离规则（核心）

Tray 面板涉及账户数据的两个区域均严格按**当前选中钱包账户**隔离，**不跨账户串显**：

| 区域 | 隔离规则 |
|-----|---------|
| 待处理交易 | 只显示当前账户的 pending 交易；切换账户后立即刷新为新账户的 pending；新账户无 pending 时显示空态 |
| 顶部钱包账户与余额 | 钱包名称 / 头像 / Account #N / 总资产 / 24h 涨跌均按当前账户展示；切换账户后顶部信息整体刷新 |

切换账户后必须保证：
- 不出现"前账户余额 + 新账户交易"的混合渲染
- 切换瞬间的中间态最终收敛到新账户
- 整个面板无白屏 / 报错 / 卡死
- 切换钱包（HD ↔ HW 等）同样适用

### 2.5 交易状态通知（macOS）

| 项目 | 说明 |
|-----|------|
| 触发条件 | 待处理交易状态变化为 `confirmed` 或 `failed` |
| 通知形式 | 原生系统通知（macOS Notification Center） |
| 通知点击 | 打开主窗口 → 定位到该交易 |
| 误报防护 | 锁屏期间不上报"已确认"通知；解锁后差量计算，不补发误报 |

### 2.6 锁屏与轮询行为（macOS）

| 状态 | Tray 行为 |
|-----|---------|
| 应用锁定 | 显示 "App is Locked"，停止行情/交易轮询 |
| 应用解锁 | 数据恢复刷新，不产生误报通知 |

### 2.7 主题与本地化（macOS）

| 切换项 | 行为 |
|-------|-----|
| 系统深 / 浅色切换 | Tray 面板实时跟随系统主题切换 |
| 应用语言切换 | Tray 面板文案实时切换为新语言 |

### 2.8 跨平台隔离

| 平台 | 行为 |
|-----|------|
| macOS | 完整支持本节所有功能 |
| Windows | 不显示 Tray 入口、Settings > Preferences 不出现「Menu Bar」开关、启动不报错 |
| Linux | 同 Windows |

### 2.9 主窗口安全边界（IPC 调用来源控制）

| 来源 | 行为 |
|-----|------|
| 主窗口 | OAuth / HttpServer / Tray 相关 IPC 调用按预期响应 |
| 非主窗口（含 Tray、其他 webview） | 拦截非授权调用，不破坏主流程 |

### 2.10 资产总值计算口径（涉及 Home / 账户详情）

| 项目 | 说明 |
|-----|------|
| 计算公式 | 总值 = token 总值 + DeFi 总值 |
| 覆盖分支 | 特定网络 / 全网络（All Networks） / merge derive（聚合派生路径） |
| 一致性 | Tray 顶部账户区总值与主窗口 Home / 账户详情显示完全一致 |

## 3. 业务规则

| 规则项 | 规则描述 |
|-------|---------|
| 平台支持 | macOS 全功能；Windows / Linux 隔离不出现入口 |
| 默认开关 | 升级无历史字段时默认开启；用户切换后持久化 |
| 面板尺寸 | 360 × 480 固定 |
| 隐藏触发 | 失焦 / Esc |
| 锁屏轮询 | 锁定时停止有效轮询 |
| 误报防护 | 锁屏期间不发"已确认"通知；解锁后差量校准 |
| IPC 控制 | 仅主窗口可发起受控 IPC，其他来源拦截 |
| 资产总值 | token + DeFi 全分支叠加，与主窗口一致 |

## 4. 已知风险

- 失焦判定：在外部应用激活、System Preferences 弹出、屏保激活等场景下的 blur 事件不一致
- 轮询：锁屏 / 解锁切换时的轮询启停时机错配可能产生误报通知
- 通知：macOS Notification Center 权限被拒、勿扰模式、通知中心折叠
- 主题：系统主题切换时 Tray 面板的实时刷新（vs 仅打开时刷新）
- 资产总值：merge derive / 全网络 / DeFi 多分支叠加在 Tray 与主窗口之间可能因不同数据源时序差异出现瞬时不一致
- 多显示器：菜单栏在副屏的位置 / 弹出位置
- macOS 版本兼容：menubar API 在不同 macOS 版本（Sequoia / Sonoma / Ventura）的差异
- 通知 Deep Link 在主窗口未启动 / 已退出时的处理路径
