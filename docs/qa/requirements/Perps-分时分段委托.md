# Perps - 分时委托（TWAP）与分段委托（Scaled Orders）

## 1. 需求背景

- 模块：Perps（合约交易）
- 功能：在「市价单 / 限价单 / 市价止盈止损 / 限价止盈止损」之外，新增 2 种**高级下单模式**：
  - **分时委托（TWAP）**：把一笔大单按时间均分为多笔市价子单，避免一次性冲击盘口。
  - **分段委托（Scaled Orders）**：在指定价格区间内，以等距挂出 N 笔限价单，构造网格式入场/出场。
- 数据源 & API：1:1 对齐 Hyperliquid。OneKey 仅做表单 + 状态展示 + 调 HL 接口。
- 参考：https://app.hyperliquid.xyz/trade/xyz:PURRDAT （HL 同款样式 / 同款数据）
- 源码（仅作辅助参考，**不作真值**）：`OneKeyHQ/app-monorepo` 分支 `x`
  - `packages/kit/src/views/Perp/components/TradingPanel/panels/PerpTradingForm.tsx`
  - `packages/kit/src/states/jotai/contexts/hyperliquid/atoms.ts`

## 2. 功能范围

### 2.1 入口与下单模式
- 下单类型切换区域共 **3 个一级 tab**：`市价单` / `限价单` / `[当前选中的高级模式] ▼`（第三个 tab 是下拉聚合）。
- **永续合约（Perps）下拉 4 项**：`市价止盈止损` / `限价止盈止损` / `分段委托` / `分时委托`。
- **现货（Spot）下拉 2 项**：`分段委托` / `分时委托`（HL 现货不支持 TP/SL 条件单，所以下拉里没有这两项）。
- 选中后，表单字段联动更新；切回 `市价单 / 限价单` 时切换字段无残留。
- 分时委托 / 分段委托 在永续与现货下核心功能字段一致；现货（Spot）专属差异如下（源码 `PerpTradingForm.tsx` 已核对）：
  - **无杠杆**（强制 `leverage = 1`）、**无保证金模式**（逐仓/全仓）、**无「当前持仓」行** → 改为 `可用余额` + `最大可买/最大可卖`（随方向变化：买入看 USDC、卖出看持有代币）。
  - **分时委托隐藏「只减仓」复选框**（保留「随机执行」）；**分段委托隐藏「只减仓」**（整段 reduceOnly 区不渲染）。提交时强制 `twapReduceOnly = false` / `scaleReduceOnly = false`。
  - 第三个 tab 下拉仅 2 项；从条件单模式切到现货时第三个 tab 默认**回退为「分段委托」**。
  - 方向文案 `买入(Buy)` / `卖出(Sell)`，`isBuy=true → side='long'`；`assetId ≥ 10000`（`SPOT_ASSET_ID_OFFSET`）；价格精度 `MAX_DECIMALS_SPOT = 8`（最大小数位 `8 - szDecimals`）。
  - 现货不支持条件单：服务层 `orderMode='trigger'` 兜底 Toast `Trigger orders are not supported in spot mode`。

### 2.2 分时委托（TWAP）字段
| 字段 | 默认 | 限制 | 说明 |
|------|------|------|------|
| 可用余额 / 当前持仓 | - | 只读 | 持仓显示 `数量 + 资产符号`（多仓绿色，空仓红色） |
| 数量 | 0%（百分比滑块 0/20/40/60/80/100，单位 USD 或 coin 二选一） | 滑块 5 档 + 百分比 + 单位切换 | 单位与「市价单 / 限价单」共享 |
| 执行时长 | `10` 分钟 | 5min ~ 24h（5 ≤ minutes ≤ 1440） | 由「小时」「分钟」两个输入框组成；时长 = `h*60 + m` |
| 时长快捷按钮 | - | `1h / 6h / 12h / 24h` | 点击 = 一键填入对应 `h / m` |
| 只减仓 | 否（不勾） | 0~1 | 勾选时，下单仅减少当前仓位，方向被强制反向 |
| 随机执行 | 是（勾选） | 0~1 | 对应 HL `twap.randomize`，子单时间 / 数量做 ±随机偏移 |
| 卖出/买入 按钮 | 红色卖 / 绿色买 | - | 余额不足时左侧灰色「保证金不足」按钮置灰显示估值 ≈ `$XXX` |

### 2.3 分段委托（Scaled）字段
| 字段 | 默认 | 限制 | 说明 |
|------|------|------|------|
| 下限 | 空 | 数值 > 0；不可与上限相等 | 价格区间下限 |
| 上限 | 空 | 数值 > 0；不可与下限相等 | 价格区间上限 |
| 订单数 | `5` | 整数 2 ≤ count ≤ 100 | 平均挂在 `[下限, 上限]` 区间内的限价单数 |
| 数量 | 0% | 滑块 5 档 + 百分比 + USD/coin | 同 TWAP |
| TIF | `GTC` | `GTC / IOC / ALO` 三选项 | 限价单 Time-in-Force（OneKey UI tooltip 文案见下） |
| 数量分配 | `固定` | `固定 / 递增` 二选一 | `固定` = 每笔委托数量相同；`递增` = **价格越优的委托数量越大**（买入价格更低、卖出价格更高），即 HL `sizeSkew` 朝价格更优端倾斜 |
| 只减仓 | 否（不勾） | 0~1 | 同 TWAP |
| 卖出/买入 按钮 | 同上 | - | 同上 |

### 2.4 校验规则（按 HL 1:1 对齐）

#### 通用
- **数量百分比 100% 的语义**：
  - 未勾「只减仓」→ 100% = 当前**可下单最大数量**（受可用余额 + 杠杆 + 风控）。
  - 勾「只减仓」→ 100% = 当前**仓位数量**。
  - 仓位为 0 且勾「只减仓」→ 100% × 0 = 0，**走通用 $10 最小金额校验**，提示「下单金额需要 $10」；按钮置灰。无独立「无可减仓位」文案。
- **最小下单金额**：每笔（TWAP 每片 / Scaled 每挂单）notional ≥ `$10`。
  - TWAP：总金额 / 估算切片数 < $10 → 提示「每片金额过小，请增加金额或缩短时长」。
  - Scaled：总金额 / 订单数 < $10 → 提示「每笔金额 ≥ $10，请增加金额或减少订单数」。
- **保证金不足**：可用余额不足以下该单时，按钮置灰显示「保证金不足」，但仍显示估值 ≈ `$XXX`。

#### 分时委托
- 执行时长为空 → 提示「请输入执行时长」。
- 执行时长 < 5min 或 > 24h → 提示「执行时长需 5min~24h」。
- **子单间隔 = 30 秒一笔（OneKey 与 HL 一致）**，因此 5min 时长 = 10 个子单，1h = 120 个，24h = 2880 个。每片金额校验按此估算（每片 ≥ $10）。

#### 分段委托 - TIF
- 字段标签：`TIF`
- 选项与定义（与 HL 网页 tooltip 完全一致）：
  - **GTC** (Good Til Cancel): Order will rest until filled or canceled.
  - **IOC** (Immediate Or Cancel): Any portion that is not immediately filled will be canceled.
  - **ALO** (Add Liquidity Only): Order will exist only as a limit order on the book. Also known as post-only.

#### 分段委托 - 数量分配（tooltip 文案）
- 字段标签：`数量分配`
- tooltip 原文：`固定：每笔委托数量相同；递增：价格越优的委托数量越大，买入价格更低、卖出价格更高。`
- 解释：
  - `固定`：每笔挂单数量严格等分（总数量 / 订单数）。
  - `递增`：按 HL `sizeSkew` 朝**价格更优端**等差倾斜。
    - 买入（做多 / 平空）：价格越低越优 → **下限端**挂单数量越大、**上限端**挂单数量越小。
    - 卖出（做空 / 平多）：价格越高越优 → **上限端**挂单数量越大、**下限端**挂单数量越小。

#### 分段委托
- 上限 / 下限任一为空 → 提示「请输入上下限价格」。
- 上限 == 下限 → 显式错误提示：「**分段委托下限和上限价格不能相同**」（红色文案、紧贴上限输入框下方）。
- 上限 < 下限 → **显式拒绝**：红色错误文案（如「上限需大于下限」），按钮置灰，不允许提交。
- 订单数非整数 / < 2 / > 100 → 提示「订单数需为 2~100 的整数」。
- 上下限超出阈值（偏离 mark price 过大）→ OneKey 会显式报错提示「价格偏离过大」/ 与现有限价单一致的阈值文案（HL 95% 偏离规则）。

### 2.5 提交后行为

#### 分时委托
- 提交成功 → 顶部 toast / 通知；下单面板不清空，保留输入便于追单。
- 不进入「当前委托(N)」普通限价单 tab；进入「分时委托 → 当前委托」专属 tab。
- 提交期间 HL 后端按预估间隔（≈30s）下发子市价单，每笔成交进入「分时委托 → 成交历史」并贡献到主「历史成交」「账户历史」。
- 完成（全部成交 / 时长到期） → 从「当前委托」移到「历史委托」，状态 `已完成`。
- 用户在「当前委托」点击「终止」可提前停止；剩余未成交不再下子单，状态变为 `已终止`。

#### 分段委托
- 提交成功 → 一次性挂出 N 笔限价单。
- 这些限价单**显示在「分时委托 → 当前委托」子 tab**（与 TWAP 共用同一面板，**不进入**底部主「当前委托(N)」普通 tab）。
- 单笔成交按主「成交历史」流程记录，同时在「分时委托 → 成交历史」也可见。
- 单笔可独立撤单（与普通限价单一致）。
- 全部成交 / 全部撤单 / 全部被拒绝 → 移到「分时委托 → 历史委托」并按对应状态归类。

## 3. 已知风险点（用于测试聚焦）

1. **每笔最小金额 vs 滑块**：用户拉到极小百分比 + 大订单数时，前端校验 vs 服务端校验是否一致。
2. **TIF=ALO（post-only）**：单笔挂单若立刻可成交，会被 HL 拒绝；分段下多笔挂单的部分拒绝行为如何展示。
3. **TIF=IOC**：Scaled 模式下 IOC 的语义（任一笔不能立成的部分立即撤销）与 Scaled 网格挂单的产品价值冲突，需观察 HL 是否实际允许该组合。
4. **价格剧烈波动**：TWAP 子单可能因偏离 95% 被 HL 拒绝；OneKey 列表如何展示「已拒绝」子单。
5. **断网 / 杀进程**：已挂出的 TWAP / Scaled 是否能在重连后继续显示并允许终止。
6. **杠杆与逐仓 / 全仓切换**：在 TWAP / Scaled 模式下切杠杆，是否影响估值数字。
