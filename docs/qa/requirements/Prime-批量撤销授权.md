# Prime - 批量撤销授权

> 模块：Prime  
> App 版本：待定  
> 日期：2026-03-20  
> 测试端：iOS / Android / Desktop / Extension / Web

---

## 1. 需求背景

批量撤销授权是 Prime 会员专属功能，用于在授权管理场景中对多个 Token 授权进行批量处理，减少逐笔撤销的操作成本。该能力需要兼顾软件钱包自动流程、硬件钱包逐笔签名、过程状态可视化与中断恢复。

---

## 2. 功能描述

### 2.1 支持范围

| 支持 | 不支持 |
|-----|--------|
| 软件钱包、无私钥钱包、私钥账户、外部账户、硬件钱包、QRWallet | 观察账户 |

### 2.2 撤销模式

| 模式 | 说明 |
|-----|------|
| 逐一取消 | 多笔时按顺序逐个签名提交，可取消当前笔并继续下一笔 |
| 批量撤销 | Prime 专属；软件钱包自动批量；硬件钱包需逐一签名 |
| 非 Prime | 未开通或已过期时使用批量撤销，跳转 Prime 权益页 |

### 2.3 支持网络

| 网络 |
|------|
| ethereum |
| bsc |
| polygon |
| arbitrum |
| avalanche |
| optimism |
| base |

### 2.4 批量进度状态

| 状态 | 说明 |
|-----|------|
| 待处理 | 尚未执行 |
| 进行中 | 当前正在撤销 |
| 成功 | 交易已提交并完成 |
| 跳过 | 因预估 Gas Fee 过高等原因跳过，可查看原因 |
| 失败 | 因 Gas 不足、断网、硬件拒绝、通信中断、4XX/5XX 等失败，可查看原因 |
| 已暂停 | 批量进程被用户或系统暂停 |

### 2.5 中断规则

| 操作 | 结果 |
|-----|------|
| 关闭弹窗 / 取消 | 停止未执行交易；已成功交易保留在交易历史 |
| 暂停 | 当前进程变为已暂停，可继续 |
| 继续 | 重新发起当前交易进程 |
| 退出页面 | 暂停当前批量进程 |
| 锁定 App | 自动暂停批量撤销 |
| 解锁 App | 保留当前进程，可继续恢复 |
| 退出 App / 杀进程 | 中断批量撤销；已成功交易保留在交易历史 |

### 2.6 结果校验

| 项目 | 说明 |
|-----|------|
| 前台约束 | 批量处理中需保持页面激活；退出页面会暂停进程 |
| 进度汇总 | 已执行数、成功数、跳过数需与实际一致 |
| 总 Gas Fee | 完成后显示总 Gas Fee 与法币换算 |
| 详情信息 | 成功项支持查看合约地址、交易哈希、复制与区块浏览器跳转 |
| 历史记录 | 成功撤销的交易需出现在交易历史 |

## 3. 业务规则

| 规则项 | 规则描述 |
|-------|---------|
| 会员限制 | 批量撤销为 Prime 专属功能 |
| 钱包支持 | 观察账户不支持该功能 |
| 模式差异 | 软件钱包支持自动批量处理；硬件钱包需要逐笔签名 |
| 网络范围 | 当前仅支持 ethereum、bsc、polygon、arbitrum、avalanche、optimism、base |
| 过程状态 | 批量撤销需暴露待处理、进行中、成功、跳过、失败、已暂停等状态 |
| 跳过规则 | 预估 Gas Fee 过高时允许跳过并保留原因 |
| 中断处理 | 关闭、取消、退出页面、锁定、退出等中断场景不得丢失已成功交易；退出页面需暂停当前进程 |
| 历史追溯 | 成功交易需可在交易历史追溯 |
| 结果核对 | 总 Gas Fee 与法币金额需可核对 |

---

## 4. 已知风险

- Gas 预估波动可能导致跳过/失败状态与用户预期不一致
- 大批量处理中切后台、锁屏、杀进程后的恢复体验待确认
- 硬件钱包多笔签名过程中通信中断的恢复策略需与设备侧保持一致
- 4XX / 5XX 错误的用户可见文案与重试策略待统一
