# Utility - 自定义网络需求文档

> **模块**：Utility（通用）
> **功能名称**：自定义网络（列表页 + 网络详情）
> **版本**：待定
> **测试端**：Desktop / Extension / Web（如移动端后续支持需补充）

---

## 1. 需求背景

原“添加自定义网络”入口流程调整为两级结构：

- 入口点击后先进入「自定义网络列表页」
- 原添加页在列表页右上角以「手动添加」按钮进入
- 原“添加自定义网络”页面命名调整为「网络详情」

本次调整目标是统一“自定义网络发现 / 搜索 / 选择 / 预填”路径，并提升从服务端链表快速补全配置的体验。

---

## 2. 页面与入口变更

### 2.1 入口行为

- 从以下入口点击「添加自定义网络」后：
  - 网络选择器 -> 添加自定义网络
  - 设置 -> 网络 -> 添加自定义网络
  - 不再直接进入表单页
  - 先进入「自定义网络列表页」

### 2.2 页面命名

- 原页面名称「添加自定义网络」改为「网络详情」
- 列表页右上角提供「手动添加」按钮，点击进入「网络详情」

---

### 2.3 入口覆盖范围

- 网络选择器路径：网络选择器 -> 添加自定义网络
- 设置路径：设置 -> 网络 -> 添加自定义网络
- dApp 快捷添加路径（如 chainlist / wallet_addEthereumChain）：
  - 若目标网络已存在：直接切换到该网络
  - 若目标网络不存在：进入「网络详情」并自动带入网络信息，用户确认后保存

---

## 3. 数据源与请求规则

### 3.1 接口来源

- 列表页数据源：`/wallet/v1/network/chainlist`

### 3.2 拉取策略

- 列表页**不设缓存**
- 每次打开列表页必须重新请求 `/wallet/v1/network/chainlist`

### 3.3 异常处理

- 请求失败 / 网络异常时，页面需给出对应错误提示
- 错误提示应可感知；当前不支持页面内直接重试，需退出页面后再次进入触发重试

---

## 4. 列表页规则

### 4.1 分页加载

- 每页返回 50 条
- 下滑到页面底部触发翻页加载（无限滚动）

### 4.2 搜索能力

- 支持模糊匹配
- 搜索字段范围：
  - 网络名称（Name）
  - 网络 ID（Chain ID）
  - 符号（Symbol）

---

## 5. 网络选择与详情预填

### 5.1 跳转行为

- 在列表页选择任一网络后，跳转到「网络详情」页

### 5.2 自动填入字段

- 跳转后自动填充以下信息：
  - 网络名称
  - RPC
  - ID（Chain ID）
  - 符号
  - 区块浏览器 URL

---

## 6. RPC 选择策略

### 6.1 多 RPC 返回

- 当服务端返回多个 RPC 时：
  - 前端在 loading 中对 RPC 进行测速
  - 自动选择测速最快的 RPC 进行填入

### 6.2 全部不可用兜底

- 若所有返回 RPC 都无法通信：
  - 默认填入返回数组中的第一个 RPC

### 6.3 空 RPC 过滤

- 前端需过滤掉 RPC 为空的数据
- RPC 为空的数据不应出现在可选或预填结果中

---

## 7. 手动添加与详情操作规则

### 7.1 手动添加字段与校验

- 必填字段：网络名称、RPC URL、链 ID（可由 RPC 解析后回填）、符号
- 非必填字段：区块浏览器 URL
- 校验规则：
  - 必填缺失时禁止保存并提示
  - RPC URL 非法或不支持时提示错误
  - 可识别 RPC 时自动回填链 ID，并提示常用符号

### 7.2 添加后结果

- 保存成功后显示添加成功反馈
- 自动切换到新添加网络
- 新网络在网络选择器可检索
- 新网络默认 icon 为首字母样式（若无服务端下发 icon）
- 区块浏览器 URL 已配置时，可从钱包首页跳转到当前链浏览器地址页
- 区块浏览器 URL 未配置时，「在浏览器中查看」置灰

### 7.3 添加后的可用性

- 网络可正常发起转账并获取手续费
- 交易加速/取消能力正常（按链能力）
- 历史记录展示正确
- 可在该网络下添加代币并完成后续转账
- 在 All networks 下该网络的主币/代币角标展示正确

### 7.4 网络详情编辑与移除

- 自定义网络应提供编辑入口，可进入「网络详情」修改并保存
- 删除操作需二次确认弹窗：
  - 取消：关闭弹窗且不变更
  - 确认移除：删除成功，网络选择器中不再展示该网络

### 7.5 服务端下发同链覆盖

- 当服务端下发与自定义网络同 chainId 的内置网络后：
  - 网络状态由自定义转为内置
  - 不再展示编辑入口
  - 名称/icon/代币信息以服务端下发为准
  - 原有自定义 RPC 保留在自定义 RPC 列表，默认关闭
  - 后续交易 RPC 走 app 配置的链路，转账与浏览器跳转正常
