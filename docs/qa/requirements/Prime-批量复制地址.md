# Prime - 批量复制地址

> 模块：Prime  
> App 版本：待定  
> 日期：2026-03-20  
> 测试端：Desktop / Extension / Mobile

---

## 1. 需求背景

批量复制地址是 Prime 会员专属功能，用于在软件钱包、硬件钱包与隐藏钱包中批量导出地址，支持按账户范围或派生范围生成地址，并支持复制或导出 CSV 文件。

---

## 2. 功能描述

### 2.1 支持范围

| 支持 | 不支持 |
|-----|--------|
| 软件钱包、无私钥钱包、硬件钱包、隐藏钱包 | 二维码钱包、私钥钱包、观察钱包、外部账户、未备份软件钱包、已重置硬件钱包 |

### 2.2 功能入口

| 入口路径 | 说明 |
|---------|------|
| 首页更多 | 打开批量复制地址弹窗，默认选择当前钱包 |
| Prime 权益页 | 打开批量复制地址弹窗，默认选择当前钱包 |
| 账户管理器 | 仅软件 / 硬件 / 隐藏钱包提供入口 |
| 非 Prime | 未开通或已过期时点击入口，唤起 Prime 介绍弹窗 |

### 2.3 弹窗结构

| 项目 | 说明 |
|-----|------|
| 钱包列表 | 展示支持的钱包图标与钱包名 |
| 网络列表 | 与网络选择器一致；不展示 All networks 和 Lightning network |
| 我的账户 Tab | 无账户显示空状态；有账户按派生路径展示账户数量 |
| 设置范围 Tab | 支持设置派生路径、起始索引、生成数量 |

### 2.4 参数规则

| 项目 | 说明 |
|-----|------|
| 网络默认值 | 默认使用首页当前网络；首页在 All networks 时默认 Bitcoin |
| 派生路径默认值 | 默认与设置页一致；TON 使用 v4R2；其他网络默认 BIP44 |
| 起始索引 | 默认 1，最大 2147483548，仅允许输入正整数 |
| 生成数量 | 默认 10，提供 1 / 10 / 100 快捷项，最大 100，仅允许输入正整数 |
| 上限处理 | 超过上限时按钮置灰并提示 |

### 2.5 导出行为

| 项目 | 说明 |
|-----|------|
| 复制 | 支持按当前结果顺序复制地址 |
| 导出 CSV | 文件名为 `钱包名称_网络名称_addresses_时间戳.csv` |
| CSV 列 | `Account name`、`<Network> address`、`Derivation path` |
| 导出取消 | 获取地址过程中取消导出，停止流程并提示已取消 |
| 硬件无设备导出 | 需二次确认风险后允许导出，并显示「地址未验证」标签 |

## 3. 业务规则

| 规则项 | 规则描述 |
|-------|---------|
| 会员限制 | 批量复制地址为 Prime 专属功能 |
| 钱包范围 | 仅支持软件钱包、无私钥钱包、硬件钱包、隐藏钱包 |
| 入口默认值 | 从首页或 Prime 权益页进入时默认选择当前钱包 |
| 网络选择 | 网络列表与网络选择器保持一致，不展示 All networks 与 Lightning network |
| Tab 分工 | 我的账户 Tab 面向已有账户；设置范围 Tab 面向派生批量生成 |
| 索引与数量 | 「从」和「生成数量」仅允许正整数 |
| 数量上限 | 单次最多生成 100 个地址 |
| CSV 格式 | 导出 CSV 的文件名与列结构需固定 |
| 取消流程 | 导出过程中取消后不得继续生成剩余地址 |
| 风险确认 | 硬件无设备导出前必须二次确认风险 |

---

## 4. 已知风险

- 大批量地址生成时的性能与等待反馈机制需确认
- 多链派生路径规则与设置页的一致性依赖链配置
- CSV 文件在不同端的编码、下载路径与打开方式存在差异
- 硬件钱包无设备导出可能引发用户对地址真实性的误解，需保持风险提示清晰
