# Swap - Panora 渠道同链测试

> 生成时间：2026-07-16
> 规则文档：`docs/qa/rules/swap-rules.md`、`docs/qa/rules/swap-network-features.md`
> 需求文档：依据 `docs/qa/rules/swap-rules.md` 中 Panora 渠道规则与网络特性表编写
> 测试端：iOS / Android / Desktop / Extension / Web
> 关联文档 / 参考模板：`docs/qa/testcases/cases/swap/2026-07-16_Swap-1inch渠道同链测试.md`

## 前置条件

- 已登录 HD 或 HW 钱包；当前账户可切换 `Aptos` 网络，且 `APT` 余额足够支付网络费。
- 已准备 `APT / USDC / USDT` 余额样本，建议优先使用规则文档中已验证的金额基线：`APT -> USDC = 1`、`USDC -> APT = 1`、`USDC -> USDT = 100`。
- Aptos 账户地址、`USDC / USDT` 合约地址、主币标识以 `docs/qa/rules/swap-network-features.md` 为准。
- 若需联调渠道链路，Panora 询价走 `/swap/v1/quote/events`，构建阶段必须复用询价返回的 `quoteResultCtx.panoraQuoteResultCtx`。
- Aptos 场景无 EVM `Approve` 流程；执行时需重点关注报价、构建态、签名提交、历史记录与费用展示。

## 测试范围说明

**Panora 同链支持网络**：Aptos

**同链兑换类型覆盖**：
- 主币→代币（`APT -> USDC`）
- 代币→主币（`USDC -> APT`）
- 代币→代币（`USDC -> USDT`，禁止同币对）

**测试覆盖要求**：
- 仅覆盖 `Aptos` 同链，不扩展到其他网络。
- **金额基线**：优先覆盖 `1 / 1 / 100` 三组已验证成功金额；如报价不稳定，先按同方向金额阶梯复探，再执行完整流程。
- **测试路线**：报价测试、构建订单测试、费用测试、历史记录测试。
- **渠道标识**：询价页、确认页、历史记录中的渠道商名称为 `Panora` 或产品定义的等价文案。
- **链路依赖**：每条 Panora 订单必须按 `quote/events -> quoteResultCtx.panoraQuoteResultCtx -> build` 流程执行；缺少 `quoteResultCtx` 不得进入有效构建态。
- **Aptos 特性**：无 `Approve`；`APT` 主币路径需正确识别为 `0x1::aptos_coin::AptosCoin`。
