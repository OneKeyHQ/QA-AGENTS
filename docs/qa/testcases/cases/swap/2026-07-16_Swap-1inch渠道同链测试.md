# Swap - 1inch 渠道同链测试

> 生成时间：2026-07-16
> 规则文档：`docs/qa/rules/swap-rules.md`、`docs/qa/rules/swap-network-features.md`
> 需求文档：依据 `docs/qa/rules/swap-rules.md` 中 1inch 渠道规则与网络特性表编写
> 测试端：iOS / Android / Desktop / Extension / Web
> 关联文档 / 参考模板：`docs/qa/testcases/cases/swap/2026-03-30_Swap-0x渠道同链支持网络测试.md`

## 前置条件

- 已登录 HD 或 HW 钱包；各测试网络主币余额与 Gas 余额充足。
- 可切换 `Ethereum / Arbitrum / Optimism / BSC / Polygon / Avalanche / Fantom / zkSync Era / Base`。
- EVM 网络的 `USDC / USDT` 合约地址、账户地址以 `swap-network-features.md` 为准。
- 若覆盖 ERC20 作为源币场景，需准备未授权状态与已授权状态两类账户样本。

## 测试范围说明

**1inch 同链支持网络**：Ethereum、Arbitrum、Optimism、BSC、Polygon、Avalanche、Fantom、zkSync Era、Base

**同链兑换类型覆盖**：
- 主币→代币（Native → Token）
- 代币→主币（Token → Native）
- 代币→代币（Token → Token，禁止同币对）

**测试覆盖要求**：
- 每个支持网络至少覆盖 1 条同链兑换。
- **金额**：最小可识别精度、中间值、`Max`（扣除 Gas）并入场景步骤，不单列边界章节；`Ethereum` 作为最小 + 中间 + `Max` 的完整标杆。
- **授权**：ERC20 作为源币时，覆盖 `Approve+Swap` 捆绑提交与 `Approve / Swap` 单独提交。
- **测试路线**：报价测试、构建订单测试、手续费测试、历史记录测试。
- **渠道标识**：询价页、确认页、历史记录中的渠道商名称为 `1inch`，不误显示为 `1inch Fusion`。
- **费用口径**：1inch 渠道返佣比例为 `0.25%`，返佣地址为 `0xeb373e57f59aaaf4e2957bc9920a20255b9aa694`。

| ❗️❗️P0❗️❗️ | 1. 当前网络=`Ethereum`<br>2. 源=`USDT`<br>3. 目标=`ETH`<br>4. `USDT` 为未授权或历史授权状态 | 1. 输入中间值<br>2. 继续授权与兑换流程 | 1. 满足 `Ethereum USDT -> ETH` 二次授权规则时，按产品定义展示重置授权 / 再授权流程<br>2. 二次授权完成后才进入兑换提交流程 |
| ❗️❗️P0❗️❗️ | 1. 当前网络=`Ethereum`<br>2. 源=`USDC`<br>3. 目标=`USDT` | 1. 输入中间值询价<br>2. 进入确认页 | 1. 返回同链 `Token -> Token` 报价<br>2. 渠道商显示 `1inch`<br>3. 不误显示为 `1inch Fusion` |
| P1 | 1. 报价带刷新倒计时或过期机制 | 1. 等待报价失效后点击 `Swap` | 1. 触发重新询价或展示报价失效提示<br>2. 更新后的报价仍可继续进入确认页 |
| P1 | 1. 页面网络异常或聚合器异常 | 1. 触发询价 | 1. 显示错误提示或重试入口<br>2. 不残留上一笔错误报价 |
