# Swap - HiFi 渠道跨链测试

> 生成时间：2026-05-15
> 规则文档：`docs/qa/rules/swap-rules.md`、`docs/qa/rules/swap-network-features.md`
> 需求文档：用户口述「Swap 新渠道 HiFi」跨链网络支持清单与 BTC 专项规则
> 测试端：iOS / Android / Desktop / Extension / Web
> 关联文档：同链场景见 `2026-05-15_Swap-HiFi渠道同链测试.md`
> 本文档 **仅保留 P0** 用例。

## 前置条件

1. 已登录 HD 或 HW 钱包（§2 为 HW）
2. 源链主币余额充足（覆盖跨链兑换金额 + 源链交易费 + 协议费）
3. HiFiSwap 按中心化换汇服务渠道处理，EVM / Tron 网络不覆盖 ERC20 / TRC20 授权逻辑；BTC / Solana 按各自链签名规则执行
4. 账户地址与代币合约地址**统一以** `swap-network-features.md` 为基线，禁止临时手填
5. BTC 钱包已具备可花费 UTXO；测试前记录当前 BTC 新鲜地址与历史已用地址，便于核对接收地址不复用
6. 后端已接入 HiFi 跨链渠道；provider 常量以接入实现为准，确认后回填 `swap-rules.md` 映射表

## 测试范围说明

**HiFi 跨链支持网络**：BTC、Ethereum、BSC、Polygon、Arbitrum、Base、Solana、Tron

**测试覆盖要求**：
- 兑换类型：在上述跨链网络组合中覆盖主币→主币、主币→代币、代币→主币、代币→代币（4 种类型）
- **金额**：最小可识别精度、中间值、**Max**（扣源链交易费）并入 §1 各表步骤与预期，不另立边界章节；§1.1「Ethereum → BSC」为最小+中间+Max 完整示例
- **BTC 专项**：支持 BTC 网络的 HiFi 场景必须覆盖 BTC 目标链新鲜地址与 BTC 源链找零地址
- 账户：软件钱包、硬件钱包
- 测试路线：报价测试、构建订单测试、手续费测试、历史记录测试
- **渠道标识**：询价路由、交易确认页、历史记录中渠道商名称为 **HiFi**（或与产品文案一致的官方名称）
- **授权口径**：HiFiSwap 为中心化换汇服务渠道，EVM / Tron 网络不测试授权入口；预期为不展示授权按钮，直接进入渠道订单确认 / 换汇确认流程
