# Swap - OKX 渠道同链测试

> 生成时间：2026-07-16
> 规则文档：`docs/qa/rules/swap-rules.md`、`docs/qa/rules/swap-network-features.md`
> 需求文档：依据 `docs/qa/rules/swap-rules.md` 中 OKX 渠道规则与网络特性表编写
> 测试端：iOS / Android / Desktop / Extension / Web
> 关联文档 / 参考模板：`docs/qa/testcases/cases/swap/2026-03-30_Swap-0x渠道同链支持网络测试.md`

## 前置条件

- 已登录 HD 或 HW 钱包；各测试网络主币余额与网络费余额充足。
- 已准备 OKX 当前支持的同链网络样本：`Ethereum / Arbitrum / Optimism / BSC / Polygon / Avalanche / Fantom / zkSync Era / Base / Solana / TON / SUI / Sonic / Scroll / Blast / Linea / X Layer`。
- 账户地址、`USDC / USDT` 合约地址、主币标识以 `docs/qa/rules/swap-network-features.md` 为准。
- EVM 网络需准备未授权状态与已授权状态两类账户样本；`Solana / TON / SUI` 不需要授权。
- 特殊基线资产需提前准备：
  - `Fantom`：稳定币对优先使用 `USDC -> fUSDT`
  - `Blast`：稳定币基线使用 `USDB`
  - `X Layer`：主币为 `OKB`，稳定币基线使用规则文档中的 `USDC` 特殊地址

## 测试范围说明

**OKX 同链支持网络**：Ethereum、Arbitrum、Optimism、BSC、Polygon、Avalanche、Fantom、zkSync Era、Base、Solana、TON、SUI、Sonic、Scroll、Blast、Linea、X Layer

**同链兑换类型覆盖**：
- 主币→代币（Native → Token）
- 代币→主币（Token → Native）
- 代币→代币（Token → Token，特殊网络按已验证基线选择可用币对）

**测试覆盖要求**：
- `OKX` 仅覆盖**同链**，不覆盖跨链；源网络与目标网络不一致时，不应命中 `OKX` 路由。
- 每个支持网络至少覆盖 1 条同链兑换；整体需覆盖 `Native -> Token`、`Token -> Native`、`Token -> Token` 三类方向。
- **金额**：稳定币默认以 `100` 为基线；主币金额按规则文档中约 `100 USD` 的已验证口径执行；`Ethereum` 作为最小 + 中间 + `Max` 的完整标杆。
- **授权**：EVM 网络 ERC20 作为源币时，覆盖 `Approve+Swap` 捆绑提交与 `Approve / Swap` 单独提交；`Solana / TON / SUI` 不覆盖授权流。
- **测试路线**：报价测试、构建订单测试、费用测试、历史记录测试。
- **渠道标识**：询价页、确认页、历史记录中的渠道商名称为 `OKX` 或产品定义的等价文案。
- **链路依赖**：OKX 构建不依赖 `quoteResultCtx`；刷新报价或修改金额后，只要当前报价有效，即可进入可提交构建态。
- **特殊链路**：`SUI` 同链路由当前可能在提交阶段返回流动性兜底结果，需校验前端展示业务层错误，而不是把渠道误判为不支持。

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 1. 已登录 HD 钱包<br>2. 当前网络=`Ethereum`<br>3. 账户有 `ETH` 余额 | 1. 进入 Swap<br>2. 源=`ETH`，目标=`USDC`（Ethereum）<br>3. 输入**最小可识别精度**询价 | 1. 返回同链报价<br>2. `Est Received` 显示 `USDC` 数量，精度正确<br>3. `Network Fee` 显示 `ETH` 与法币值<br>4. 路由 / 渠道标识包含 `OKX` |
| ❗️❗️P0❗️❗️ | 1. 同上，已获取有效报价 | 1. 输入**中间值**重新询价<br>2. 点击 `Max` | 1. 报价随输入刷新<br>2. `Max` 填充值为扣除 Gas 后的可用上限<br>3. 不出现余额透支提示 |

| ❗️❗️P0❗️❗️ | 1. 当前网络=`Ethereum`<br>2. 源=`USDT`<br>3. 目标=`ETH`<br>4. `USDT` 为未授权或历史授权状态 | 1. 输入中间值<br>2. 继续授权与兑换流程 | 1. 满足 `Ethereum USDT -> ETH` 二次授权规则时，按产品定义展示重置授权 / 再授权流程<br>2. 二次授权完成后才进入兑换提交流程 |
| ❗️❗️P0❗️❗️ | 1. 当前网络=`Ethereum`<br>2. 源=`USDC`<br>3. 目标=`USDT` | 1. 输入中间值询价<br>2. 进入确认页 | 1. 返回同链 `Token -> Token` 报价<br>2. 渠道商显示 `OKX`<br>3. 两侧代币地址不相同 |
| P1 | 1. 报价带刷新倒计时或过期机制 | 1. 等待报价失效后点击 `Swap` | 1. 触发重新询价或展示报价失效提示<br>2. 更新后的报价仍可继续进入确认页 |
| P1 | 1. 页面网络异常或聚合器异常 | 1. 触发询价 | 1. 显示错误提示或重试入口<br>2. 不残留上一笔错误报价 |
