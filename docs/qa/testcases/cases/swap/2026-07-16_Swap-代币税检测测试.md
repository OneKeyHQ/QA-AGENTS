# Swap - 代币税检测测试

> 生成时间：2026-07-16
> 规则文档：`docs/qa/rules/swap-rules.md`
> 需求文档：`docs/qa/requirements/Swap-代币税检测.md`
> 测试端：Web / iOS / Android / Desktop / Extension

## 测试范围说明

**功能范围**：Swap 代币税双数据源检测、税率提示展示、默认滑点自动叠加、用户手动滑点优先、切代币 / 切方向重算、OKX 异常降级
**测试重点**：覆盖 GoPlus / OKX 单边命中、双源税率不一致、from / to 双向有税、晚到结果不覆盖用户手动值

---

## 前置条件

- 已登录可用钱包；测试网络与 Swap 服务可用。
- 准备可控联调环境或 Mock 数据，可分别控制 GoPlus / OKX 的税率返回、超时、失败、无数据。
- 准备以下基础场景数据：
  - 无税 Token 对
  - 仅 GoPlus 命中代币税的 Token 对
  - 仅 OKX 命中代币税的 Token 对
  - GoPlus / OKX 税率不一致的 Token 对
  - 仅 from 侧有税、仅 to 侧有税、from / to 双向有税的 Token 对
  - 高税率（如 30%+）Token 对
- 用户默认滑点预设为 1%，并具备手动修改滑点入口。
