# Swap - K 线测试

> 生成时间：2026-05-28
> 规则文档：`docs/qa/rules/swap-rules.md`、`docs/qa/rules/market-rules.md`、`docs/qa/qa-rules.md`
> 需求文档：`docs/qa/requirements/Swap-K线.md`
> 测试端：Android / iOS / Desktop / Extension / Web
> 关联资源：设计稿 Swap K-line；K 线数据与组件复用 Market 模块

## 测试范围说明

**功能范围**：Swap 页面 K 线入口状态、K 线视图打开与关闭、默认展示代币判定、From / To 切换、周期 fallback、数据加载、空状态、加载失败与多端布局。

**覆盖重点**：
- From / To 均未选择时入口禁用，任一已选择时入口可点击
- 默认展示代币按稳定币判定规则选择
- K 线周期按 `1H -> 1D -> 1W` 顺序 fallback
- K 线视图仅展示主图与 Volume，不展示 MA / MACD / RSI 等冗余指标

---

## 前置条件

1. 已登录 HD 钱包 / HW 钱包，Swap 页面可打开
2. 已准备非稳定币 A、非稳定币 B、稳定币 C、稳定币 D 四组测试 Token
3. 已准备一组 1H 有数据 Token、一组仅 1D 有数据 Token、一组仅 1W 有数据 Token、一组无 K 线数据 Token
4. 可通过 Mock / 代理 / 测试环境模拟 K 线接口 loading、空数据与失败响应

## 3. K 线周期与数据展示

| 优先级 | 场景 | 操作步骤 | 预期结果 |
|---|---|---|---|
| ❗️❗️P0❗️❗️ | 1. 当前 Token 的 1H K 线有数据 | 1. 打开 K 线视图<br>2. 等待数据加载 | 1. 默认请求 1H 数据<br>2. 1H 周期状态=选中<br>3. 显示 K 线主图与 Volume |
| ❗️❗️P0❗️❗️ | 1. 当前 Token 的 1H 无数据<br>2. 1D 有数据 | 1. 打开 K 线视图<br>2. 等待 fallback 完成 | 1. 先请求 1H 数据<br>2. 1H 无数据后请求 1D 数据<br>3. 最终选中 1D 并显示 K 线主图与 Volume |
| ❗️❗️P0❗️❗️ | 1. 当前 Token 的 1H / 1D 无数据<br>2. 1W 有数据 | 1. 打开 K 线视图<br>2. 等待 fallback 完成 | 1. 请求顺序为 1H -> 1D -> 1W<br>2. 最终选中 1W<br>3. 显示 K 线主图与 Volume |
| P1 | 1. K 线视图已显示数据 | 1. 点击 1m / 15m / 30m / 4h / 1d / 1w 任一周期 | 1. 被点击周期状态=选中<br>2. K 线主图按所选周期刷新<br>3. Volume 按所选周期刷新 |
| P1 | 1. K 线视图已显示数据 | 1. 观察图表底部与指标区域 | 1. 显示 Volume<br>2. 不显示 MA<br>3. 不显示 MACD<br>4. 不显示 RSI |
| P1 | 1. 当前 Token 有 K 线数据<br>2. 当前 Swap 页面显示 Token 价格 | 1. 打开 K 线视图<br>2. 对比 K 线最新价格与 Swap Token 价格 | 1. K 线最新价格存在<br>2. Swap Token 价格存在<br>3. 两处价格差异在行情刷新容忍范围内 |

## 4. 数据来源与网络参数

| 优先级 | 场景 | 操作步骤 | 预期结果 |
|---|---|---|---|
| P1 | 1. From Token 与 To Token 属于不同网络<br>2. K 线视图已打开 | 1. 切换 From / To Token 选项<br>2. 查看请求参数或接口日志 | 1. 请求参数 network 与当前选中 Token 网络一致<br>2. 请求参数 contract address 与当前选中 Token 合约地址一致<br>3. 不使用另一个 Token 的网络或合约地址 |
| P1 | 1. 当前 Token 为主币<br>2. K 线视图已打开 | 1. 查看请求参数或接口日志 | 1. 请求参数使用 Market 模块主币行情标识<br>2. K 线视图显示主币名称、图标、网络 |
| P1 | 1. 当前 Token 为代币<br>2. K 线视图已打开 | 1. 查看请求参数或接口日志 | 1. 请求参数包含代币 contract address<br>2. K 线视图显示代币名称、图标、网络 |

| 优先级 | 场景 | 操作步骤 | 预期结果 |
|---|---|---|---|
| ❗️❗️P0❗️❗️ | 1. 当前 Token 的 1H / 1D / 1W 均无数据 | 1. 打开 K 线视图<br>2. 等待 fallback 完成 | 1. 显示设计稿空状态<br>2. 不显示空白图表区域<br>3. 周期选项仍可见 |
| P1 | 1. K 线接口返回超时或 5xx | 1. 打开 K 线视图<br>2. 等待请求失败 | 1. 显示加载失败提示<br>2. 显示重试入口<br>3. 不关闭 K 线视图 |

| P1 | 1. K 线数据请求中 | 1. 观察 K 线视图内容区域 | 1. 显示 loading 状态<br>2. 不显示空白内容区域<br>3. 显示 Token 切换项或加载态 |
| P1 | 1. K 线数据请求中<br>2. 用户切换 From / To Token | 1. 点击另一个 Token 切换项 | 1. 取消或忽略旧 Token 请求结果<br>2. 显示新 Token loading 状态<br>3. 最终展示新 Token K 线数据 |
