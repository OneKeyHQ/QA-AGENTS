# Swap - LiFi 渠道同链测试

> 生成时间：2026-07-16
> 规则文档：`docs/qa/rules/swap-rules.md`、`docs/qa/rules/swap-network-features.md`
> 需求文档：依据 `docs/qa/rules/swap-rules.md` 中 LiFi 渠道规则与网络特性表编写
> 测试端：iOS / Android / Desktop / Extension / Web
> 关联文档 / 参考模板：`docs/qa/testcases/cases/swap/2026-07-16_Swap-OKX渠道同链测试.md`

## 前置条件

- 已登录 HD 或 HW 钱包；各测试网络主币余额与网络费余额充足。
- 已准备 LiFi 当前支持的同链网络样本：`Ethereum / BSC / Polygon / Arbitrum / Avalanche / zkSync / Sonic / Scroll / Mantle / Blast / Base / Optimism / Fantom / Solana / HyperEVM / Robinhood`。
- 账户地址、`USDC / USDT` 合约地址、主币标识以 `docs/qa/rules/swap-network-features.md` 为准。
- LiFi 构建阶段必须复用询价返回的 `quoteResultCtx`，不允许直接构建。
- Arbitrum / Optimism / Mantle / HyperEVM 的 `USDC / USDT` 地址执行时保持全小写，避免出现 `Provider error` 或 `quoteResultCtx` 缺失。
- 特殊基线说明：
  - `Blast` 优先使用 `USDB` 作为稳定币基线。
  - `Robinhood` 当前仅新增到支持矩阵；若执行前仓库仍未补齐地址 / token 基线，则该网络记录为前置阻塞，不硬编固定交易对。

## 测试范围说明

**LiFi 同链支持网络**：Ethereum、BSC、Polygon、Arbitrum、Avalanche、zkSync、Sonic、Scroll、Mantle、Blast、Base、Optimism、Fantom、Solana、HyperEVM、Robinhood

**同链兑换类型覆盖**：
- 主币→代币（Native → Token）
- 代币→主币（Token → Native）
- 代币→代币（Token → Token，特殊网络按已验证基线选择可用币对）

**测试覆盖要求**：
- 每个支持网络至少覆盖 1 条同链兑换；整体需覆盖 `Native -> Token`、`Token -> Native`、`Token -> Token` 三类方向。
- `Ethereum` 作为最小 + 中间 + `Max` 的完整标杆。
- **金额**：稳定币默认以 `100` 为基线；主币金额按规则文档中约 `100 USD` 的已验证口径执行，默认采用 `ETH 系 0.053`、`BNB 0.156`、`POL 1070`、`AVAX 12.1`、`SOL 1.34`、`MNT 164`、`S 2635`、`HYPE 1.38`。
- **授权**：EVM 网络 ERC20 作为源币时，覆盖 `Approve+Swap` 捆绑提交与 `Approve / Swap` 单独提交；`Solana` 不覆盖授权流。
- **测试路线**：报价测试、构建订单测试、费用测试、历史记录测试。
- **渠道标识**：询价页、确认页、历史记录中的渠道商名称为 `LiFi` 或产品定义的等价文案。
- **链路依赖**：每条 LiFi 订单必须按 `quote -> quoteResultCtx -> build` 流程执行；缺少 `quoteResultCtx` 不得进入有效构建态。
- **特殊链路**：
  - `Blast` 走 `USDB` 基线，不沿用通用 `USDC / USDT` 占位。
  - `Robinhood` 若未完成地址基线回填，则该网络作为阻塞项单独记录，不混入已执行成功集。

| 优先级 | 场景 | 操作步骤 | 预期结果 |

| ❗️❗️P0❗️❗️ | 1. 当前网络=`Ethereum`<br>2. 源=`USDC`<br>3. 目标=`USDT` | 1. 输入中间值询价<br>2. 进入确认页 | 1. 返回同链 `Token -> Token` 报价<br>2. 渠道商显示 `LiFi`<br>3. 两侧代币地址不相同 |
