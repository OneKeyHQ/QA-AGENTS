> 生成时间：2026-07-16

> 需求文档：依据 `docs/qa/rules/swap-rules.md` 中 LiFi 渠道规则与网络特性表编写
> 测试端：iOS / Android / Desktop / Extension / Web
> 关联文档：同链场景见 `docs/qa/testcases/cases/swap/2026-06-03_Swap-LiFi渠道同链测试.md`
> 参考模板：`docs/qa/testcases/cases/swap/2026-04-20_Swap-RocketX渠道跨链测试.md`

## 前置条件

- 已登录 HD 或 HW 钱包；源链主币余额充足，覆盖跨链兑换金额、源链 Gas、协议费与桥费用。
- 已准备 LiFi 当前支持的跨链网络样本：`Ethereum / BSC / Polygon / Arbitrum / Avalanche / zkSync / Sonic / Scroll / Mantle / Blast / Base / Optimism / Fantom / Solana / HyperEVM`。
- 账户地址、`USDC / USDT` 合约地址、主币标识以 `docs/qa/rules/swap-network-features.md` 为准。
- LiFi 构建阶段必须复用询价返回的 `quoteResultCtx`，不允许直接构建。
- Arbitrum / Optimism / Mantle / HyperEVM 的 `USDC / USDT` 地址执行时保持全小写，避免出现 `Provider error` 或 `quoteResultCtx` 缺失。
- 特殊基线说明：
  - `Blast` 跨链路由优先以 `ETH` 或 `USDB` 作为源 / 目标资产，不沿用通用 `USDC / USDT` 占位。
  - `Fantom` 当前新增到跨链支持矩阵，执行固定路由前需确认地址基线与币对可用性。

## 测试范围说明

**LiFi 跨链支持网络**：Ethereum、BSC、Polygon、Arbitrum、Avalanche、zkSync、Sonic、Scroll、Mantle、Blast、Base、Optimism、Fantom、Solana、HyperEVM

**测试覆盖要求**：
- 在上述跨链网络组合中覆盖 `主币->主币`、`主币->代币`、`代币->主币`、`代币->代币` 四种类型。
- `Ethereum -> BSC` 作为最小 + 中间 + `Max` 的完整标杆。
- **金额**：稳定币默认以 `100` 为基线；主币金额按规则文档中约 `100 USD` 的已验证口径执行，默认采用 `ETH 系 0.053`、`BNB 0.156`、`POL 1070`、`AVAX 12.1`、`SOL 1.34`、`MNT 164`、`S 2635`、`HYPE 1.38`。
- **授权**：EVM 网络 ERC20 作为源币时，覆盖 `Approve+Swap` 捆绑提交与 `Approve / Swap` 单独提交；`Solana` 不覆盖授权流。
- **测试路线**：报价测试、构建订单测试、费用测试、历史记录测试。
- **渠道标识**：询价页、确认页、历史记录中的渠道商名称为 `LiFi` 或产品定义的等价文案。
- **链路依赖**：每条 LiFi 订单必须按 `quote -> quoteResultCtx -> build` 流程执行；缺少 `quoteResultCtx` 不得进入有效构建态。

#### 场景：Ethereum → BSC（ETH → BNB）

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 1. 已登录 HD 钱包<br>2. 当前源链=`Ethereum`<br>3. 账户有 `ETH` 余额 | 1. 进入 Swap<br>2. 源=`ETH`（Ethereum），目标=`BNB`（BSC）<br>3. 输入**最小可识别精度**询价 | 1. 显示跨链报价，命中 `LiFi` 路由<br>2. `Est Received` 显示 `BNB` 数量，精度正确<br>3. `Network Fee`、`Protocol Fee`、预估时间等字段可读取<br>4. 返回有效 `quoteResultCtx` |
