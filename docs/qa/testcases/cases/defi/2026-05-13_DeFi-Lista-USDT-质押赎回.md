# DeFi - Lista USDT 质押与赎回

> **需求来源**：录制流程（2026-05-13）<br>
> **规则文档**：`docs/qa/rules/defi-rules.md`（待新增 Lista 章节）<br>
> **测试端**：桌面端（先做 Desktop，后续按需扩展到 Extension / Web）<br>
> **入口路径**：所有资产 → DeFi → 简单赚币 → USDT → Lista 渠道

## 测试范围说明

- **协议**：Lista（Pangolins 金库管理）
- **币种**：USDT（链上具体网络以 app 当前选中为准）
- **核心断言**：
  1. APY 公式：`综合 APY = (原生 APY − 业绩费) + LISTA APY`
  2. 预估年收益（USD）：
     - `USDT 部分 = (原生 APY − 业绩费) × 认购价值(USD)`
     - `LISTA 部分 = LISTA APY × 认购价值(USD)` **（LISTA 奖励不扣业绩费）**
  3. 代币显示数量 = USD 收益 / 该代币当前价格（四舍五入）
  4. 业绩费规则：利润的 10%（OneKey 5% + 金库管理者 5%），**仅对利润收费，不收本金**

## 3. 认购金额输入与预估年收益断言

### 3.1 简化版 APY 公式验证（**主验证 P0**）

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 在 Lista USDT 详情页，进入认购 tab | 输入测试金额 100 USDT（不实际提交，仅为方便计算）| 1. 输入框显示 100<br>2. 下方显示 USD 估值（如 \$99.99）<br>3. 显示「预估年收益」两行：<br>　• `0.4703 USDT ($0.47)`<br>　• `15.7526 LISTA ($1.36)`<br>4. 显示「认购余额不足」警告（钱包余额 < 100，预期）|
| ❗️❗️P0❗️❗️ | 同上 | **核心验证：sum(USD) ≈ APY% × 认购价值(USD)** | 读取页面顶部 APY 值（如 1.83%）+ 预估年收益两行的 USD 部分相加。验证：<br>`USDT 部分 USD + LISTA 部分 USD ≈ APY% × 认购价值 USD`<br>例：1.83% × \$99.99 = \$1.83 ≈ \$0.47 + \$1.36 = \$1.83 ✓<br>容差：±\$0.05 或 ±5% 相对误差 |
| ❗️❗️P0❗️❗️ | 验证完成 | 清空输入框 → 重新输入 0.001（为后续质押用例准备）| 1. 输入框值 = 0.001<br>2. 「认购余额不足」警告消失<br>3.「授权」按钮变为可点击 |

## 备注

1. **业绩费机制**：利润 10%（OneKey 5% + 金库管理者 5%），仅对利润收费，不收本金。在小金额测试场景下（0.001 USDT），利润 ≈ 0，业绩费可忽略。
2. **APY 实时变化**：综合 APY 数值会随市场波动，断言公式应基于**当前页面读取的 APY 值**而非硬编码。
3. **代币价格依赖**：USDT 与 LISTA 的 USD 单价从页面读取（USD 估值文本），不依赖外部行情 API。
4. **历史记录延迟**：交易**立即提交成功**，但出现在历史列表中通常需要 ≤ 60 秒。脚本中用轮询 + 超时机制。
5. **录制依赖**：本用例脚本基于 `shared/results/recording/steps.json` 录制结果（2026-05-13 session）。
