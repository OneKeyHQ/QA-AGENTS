# DeFi - Native 存入与赎回

> **需求来源**：PR [#11717](https://github.com/OneKeyHQ/app-monorepo/pull/11717)（OK-54973），目标版本 `release/v6.3.0`
> **规则文档**：`docs/qa/rules/defi-rules.md` §3.5 Native
> **需求文档**：`docs/qa/requirements/DeFi-Native协议.md`
> **测试端**：桌面端 / 插件端 / Web 端
> **协议**：Native（自管金库收益协议，以太坊）
> **支持币种**：ETH / WETH / USDT
> **入口**：DeFi → 资产列表 → ETH / WETH / USDT → Native 渠道

## 测试范围

- **存入**：USDT / WETH 直接存入（授权 → 存入，2 步）；ETH 走 wrap 存入 WETH 金库（wrap → 授权 → 存入，3 步）
- **赎回**：立即赎回（即时到账，收手续费）/ 排队赎回（约 9 天，免手续费）/ 取消已排队赎回
- **领取**：普通收益 / 空投奖励
- **赎回方式切换**：立即 ↔ 排队切换时，面板内到账金额、手续费按当前选项刷新

| P1 | ETH 余额不足 | 选 ETH 后输入超出余额（已扣 Gas）的金额 | 显示余额不足，主按钮禁用 |

## 备注

1. **金库支持**：WETH 金库同时支持 WETH（直存）与 ETH（先 wrap）；USDT 为独立金库；以 app 实际加载列表为准。
2. **APY 实时性**：APY 为动态数值，断言公式按页面当前 APY 计算，不硬编码（5.00% 仅为设计稿示意）。
3. **赎回方式切换**：立即 ↔ 排队切换时，面板内到账金额与手续费按当前选项刷新；切回后数值与首次一致。
4. **排队等待时长**：设计稿示意约 9 天，ETH 标注 7-9 天，以页面文案为准。
5. **小金额测试**：质押 / 赎回 0.001 USDT 等微量场景，资产变动可忽略 Gas 与手续费微小误差。
6. **交易损耗确认弹窗**：当存入金额过小、按当前 APY 估算预估收益短期难以覆盖网络费用时，点击底部主按钮触发「交易损耗」确认弹窗，主文案 `按当前预估收益率计算，大约需要 X 年，收益才能弥补损失。`。年数随金额与 APY 动态计算，越小越久（如 0.0001 ETH 估算 4 年）。赎回流程在金额过小时是否同样触发待实跑验证。
