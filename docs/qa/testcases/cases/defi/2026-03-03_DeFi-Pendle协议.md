## 7. 异常与边界

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 1. Buy 面板已打开<br>2. 支付币种余额为 0 | 1. 输入大于 0 的金额<br>2. 或点击 Max | 1. 显示余额不足类提示或 Receive 不更新<br>2. 支付金额为 0 或不可用，主按钮禁用或提示不足 |
| ❗️❗️P0❗️❗️ | 网络断开或 API 异常 | 1. 进入所有资产列表或代币详情<br>2. 触发报价或列表刷新 | 1. 显示网络错误或加载失败提示<br>2. 不展示错误数据；重试或恢复网络后可恢复 |
| P1 | 支付金额输入为 0 | 保持金额 0，查看主按钮 | Buy / Sell early 或 Approve 按钮禁用 |
| P1 | 支付金额输入负数或非法字符 | 输入 -1 或非数字 | 不接受或清空；或显示格式错误提示 |
| P1 | 支付金额超过余额 | 输入大于钱包余额的数值 | 显示余额不足或超限提示；主按钮禁用或不可提交 |
| P2 | 市场已到期 | 进入该市场详情页 | 仅展示 Redeem 或仅赎回可用，Buy 不展示或禁用 |
| P2 | 极小精度金额 | 输入 0.00000001 等极小值 | 按链/协议规则截断或提示最小单位；不崩溃 |

## 8. 刷新与询价机制

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 代币详情或列表页已加载 | 在 Buy 的 Amount 输入框输入或修改金额 | 系统实时询价；Receive 与 Fee 等随输入更新 |
| P1 | 列表页或详情页存在刷新按钮 | 1. 点击刷新按钮<br>2. 等待约 5 秒 | 1. 触发一次刷新，数据更新<br>2. 约 5 秒后再次触发一次自动刷新 |
| P1 | 列表页或详情页支持自动刷新 | 不进行任何操作，等待约 15 秒 | 系统每约 15 秒自动刷新一次，数据更新 |
| P1 | 刚执行完一次手动刷新 | 在 5 秒内再次点击刷新按钮 | 不触发刷新或显示需等待提示；5 秒后可再次触发 |
| P2 | 输入金额后正在询价 | 快速连续修改金额 | 展示的 Receive/Fee 与最近一次询价结果一致，无错乱或旧数据残留 |
