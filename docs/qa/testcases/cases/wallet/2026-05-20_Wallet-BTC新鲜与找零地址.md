## 前置条件与测试数据

1. 已导入含历史交易记录的 BTC 测试助记词：`music strategy danger page blood lumber tongue trust clump rose rely saddle`
2. 已与第三方工具（BIP39 / Trezor / Sparrow / Electrum）对齐地址生成结果（参考测试数据 wiki）
3. 老用户客户端：升级前的客户端，**默认关闭**多地址模式；新安装客户端：全新安装且未导入旧数据，**默认开启**多地址模式
4. 至少一个 BTC HD 钱包账户（已备份）+ 至少一个 BTC 观察账户 + 至少一个 BTC 私钥账户
5. 一个可用的外部 BTC 收款地址（用于发起转账，触发找零）
6. dApp：`https://dapp-example.onekeytest.com/btcUnitsat`（用于验证单地址连接限制）
7. 派生路径：Taproot（默认）；其他路径（Native SegWit / Nested SegWit / Legacy）按需切换覆盖

## 3. 多地址模式 — 转账与找零行为

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 转账后接收地址刷新 | 记录新鲜地址 A → 发起转账 → 立即回接收页 | 地址变为新的新鲜地址 B（A ≠ B），无需等待上链 |
| ❗️❗️P0❗️❗️ | 转账生成新找零地址 | 转账金额 < 账户余额（触发找零）→ 区块浏览器查输出 | 输出含外部接收地址 + 1 个全新找零地址；该找零地址进入 Address list 找零 tab |
| ❗️❗️P0❗️❗️ | 再次向历史新鲜地址转账 | 向钱包曾用过的旧接收地址 X 转账 | 入账成功；钱包历史显示该记录；总余额按金额增加；X 在已使用列表的累计接收字段累加 |
| ❗️❗️P0❗️❗️ | 总余额含前 20 个找零合计 | 钱包首页查看 BTC 总余额 | 总余额 = 接收地址 + 最多 20 个找零地址合计；超出部分需进 Address list 查看 |
| P1 | 历史记录展示 | 完成多笔转账 → 进入 BTC 历史 | 按时间倒序显示；地址按 xpub 聚合（详见 `Wallet-最近转账.md` v2.6）|

## 5. 多地址模式 — dApp / Swap 交互

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | dApp 单一地址连接拦截 | 打开 `https://dapp-example.onekeytest.com/btcUnitsat` → 连接钱包 | 弹出「仅支持单一地址」提示弹窗，含「取消」/「切换」两按钮 |
| ❗️❗️P0❗️❗️ | 点击「取消」 | 弹窗中点击「取消」 | 弹窗关闭；dApp 未连接；多地址开关保持开启 |
| ❗️❗️P0❗️❗️ | 点击「切换」 | 弹窗中点击「切换」 | 仅当前端多地址关闭；dApp 连接成功，以单一固定地址展示 |
| ❗️❗️P0❗️❗️ | 多地址设置单端独立 | 桌面端开启 / 手机端关闭（或反之） | 两端互不影响（不通过 OneKey ID / iCloud / Prime 同步）|
| P1 | 插件端无弹窗 | 插件端打开同 dApp 连接 | 仅显示提示文案，不弹窗（按产品实现允许直接切换）|
| ❗️❗️P0❗️❗️ | Swap 询价与交易 | 多地址关闭后 → Swap BTC → 询价 → 提交 | 询价返回有效报价；交易上链；Swap 完成后余额变化 |
| P1 | 重新开启多地址后再连 dApp | 打开多地址 → 再次连 btcUnitsat | 再次弹出同一弹窗 |

## 6. 关闭多地址模式 — UI 与字段恢复

> 前置：在多地址模式开启状态下完成至少一笔转账（产生历史新鲜 + 找零地址），然后将开关关闭

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 接收页恢复固定地址 | 关闭多地址 → 进入接收页 | 展示账户固定首个接收地址（不再显示多地址入口提示） |
| P1 | 代币详情页恢复「我的地址」 | 进入 BTC 代币详情页 | 恢复显示「我的地址」栏 |
| ❗️❗️P0❗️❗️ | More 菜单：仅隐藏 Address list | BTC HD 账户 → More 菜单 | 不展示 Address list；仍展示 Coins (UTXO) |
| P1 | 历史地址仍可被搜索到 | 发送页 → 搜索某历史新鲜 / 找零地址 | 仍能匹配到该账户（联想数据保留）|

## 7. 关闭多地址模式 — 转账行为

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 最小值转账后地址不变 | 关闭多地址 → 发起最小值转账 → 查看接收地址 | 交易上链；接收地址保持固定，不生成新地址 |
| ❗️❗️P0❗️❗️ | 找零回到固定地址 | 区块浏览器查同笔交易输出 | 找零输出 = 原固定地址，不生成新找零地址 |
| P1 | 多次转账找零地址一致 | 关闭多地址后连续 2~3 笔转账 | 每笔找零输出均为同一固定地址 |

## 8. 边界与异常

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| P1 | 派生路径切换同步 | 多地址开启状态下切换派生路径（Taproot → Native SegWit）| 接收页地址 + Address list + 总余额按新路径刷新 |
| P1 | 重启 App 后状态保留 | 切换多地址 → 杀掉 App → 重启 | 开关状态保留 |
| P2 | 重置钱包后默认值 | 重置应用 → 重新导入助记词 | 多地址默认开启（仅当前端）|
| P2 | 弱网下转账后地址刷新 | 弱网环境下提交转账 → 查看接收页 | 接收页本地立即刷新到下一个新鲜地址 |
