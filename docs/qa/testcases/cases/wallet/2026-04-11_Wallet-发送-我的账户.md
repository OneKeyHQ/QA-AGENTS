# 发送 - 我的账户（选择地址 - 账户 Tab）

> 规则文档：`docs/qa/rules/account-rules.md` - 第 1、2、3.4 章
> 测试端：iOS / Android / Desktop / Extension
> 变更说明：首版——覆盖发送流程「选择地址」弹窗中「账户」Tab 的账户列表展示、派生路径切换（BTC/LTC）、BTC 新鲜地址、搜索与选择代入逻辑
> 关联用例：`2026-04-03_Wallet-最近转账.md`（最近 Tab）、地址簿用例（地址簿 Tab）

---

## 前置条件与测试数据

1. 已导入 HD 软件钱包（已备份）/ 硬件钱包 / 私钥账户 / Keyless Wallet，各含至少一个账户
2. 已创建未备份软件钱包 / 观察账户 / 外部账户（用于验证隐藏规则）
3. BTC / LTC 账户已创建全部派生路径地址
4. BTC 已开启「新鲜地址」，且至少有一个已使用过的旧地址

---

## 1. 账户列表展示与过滤规则

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | HD 软件钱包（已备份） | 进入发送页 → 点击「选择地址」→ 切换到「账户」Tab | 该钱包下所有账户可见，显示账户名称和地址，条目数与钱包内账户数一致 |
| ❗️❗️P0❗️❗️ | 硬件钱包 | 同上 | 硬件钱包独立分组展示，账户名称和地址可见 |
| ❗️❗️P0❗️❗️ | 私钥账户 | 同上 | 私钥账户可见，显示账户名称和地址 |
| ❗️❗️P0❗️❗️ | Keyless Wallet | 同上 | Keyless Wallet 可见，支持点击选择 |
| ❗️❗️P0❗️❗️ | 观察账户 | 同上 | **不显示**观察账户（无转账权限） |
| ❗️❗️P0❗️❗️ | 外部账户 | 同上 | **不显示**外部账户 |
| ❗️❗️P0❗️❗️ | 未备份软件钱包 | 同上 | **不显示**未备份钱包的账户 |
| P1 | 不同链下地址格式 | 分别在 EVM / BTC / Cosmos / Solana 网络下打开「账户」Tab | 每个账户展示的地址格式与当前链一致（EVM 为 0x 开头、BTC 为 bc1/3/1 开头等） |
| P1 | 当前发送账户自身 | 从账户 A 发起转账，打开「账户」Tab | 账户 A 自身也在列表中（允许自转） |

## 2. 钱包分组折叠与展开

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| P1 | 默认展示状态 | 打开「账户」Tab | 所有钱包分组默认展开（平铺），可见所有账户 |
| P1 | 折叠钱包分组 | 点击某个钱包分组的折叠箭头 | 1. 该分组收起，仅显示分组标题<br>2. 其他分组不受影响 |
| P1 | 展开已折叠分组 | 点击已折叠分组的展开箭头 | 该分组展开，账户列表恢复可见 |
| P1 | 切换 Tab 后折叠状态保持 | 1. 折叠某个分组<br>2. 切换到「最近」Tab<br>3. 切回「账户」Tab | 之前折叠的分组**仍保持折叠** |
| P1 | 退出弹窗后折叠状态重置 | 1. 折叠某个分组<br>2. 关闭「选择地址」弹窗<br>3. 重新打开弹窗 → 切到「账户」Tab | 所有分组**恢复默认展开**状态 |

## 3. BTC / LTC 派生路径切换

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | BTC 路径切换按钮展示 | BTC 网络下打开「账户」Tab（已创建多个路径地址） | 钱包分组标题旁显示路径切换下拉按钮（如 "Nested SegWit ▽"） |
| ❗️❗️P0❗️❗️ | BTC 切换到 Native SegWit | 点击路径下拉 → 选择 Native SegWit | 1. 账户列表刷新为 Native SegWit 路径下的地址（bc1q 开头）<br>2. 下拉按钮文案更新为 "Native SegWit" |
| ❗️❗️P0❗️❗️ | BTC 切换到 Legacy | 点击路径下拉 → 选择 Legacy | 账户列表刷新为 Legacy 路径下的地址（1 开头） |
| P1 | BTC 切换到 Taproot | 点击路径下拉 → 选择 Taproot | 账户列表刷新为 Taproot 路径下的地址（bc1p 开头） |
| ❗️❗️P0❗️❗️ | LTC 路径切换 | LTC 网络下打开「账户」Tab（已创建多个路径地址） | 1. 显示路径切换下拉按钮<br>2. 可切换 Nested SegWit / Native SegWit / Legacy<br>3. 切换后地址格式对应正确 |
| P1 | 仅创建单路径时无切换按钮 | BTC 网络下仅创建了一个路径（如仅 Native SegWit） | 不显示路径切换下拉按钮 |
| P1 | EVM 链无快捷路径切换 | EVM 网络下打开「账户」Tab（系统设置中已切换过派生路径） | 1. **不显示**路径切换下拉按钮<br>2. 展示的是系统设置中**当前选定路径**对应的地址 |
| P1 | Solana 等其他链无快捷路径切换 | Solana 网络下打开「账户」Tab（系统设置中已切换过派生路径） | 1. **不显示**路径切换下拉按钮<br>2. 展示的是系统设置中**当前选定路径**对应的地址 |
| P1 | 路径切换后选择代入 | BTC 网络下切换到 Taproot → 选择某账户 | 代入的地址为 Taproot 格式地址（bc1p 开头） |

## 4. BTC 新鲜地址

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 开启新鲜地址时列表展示 | BTC 开启新鲜地址 → 打开「账户」Tab | 账户展示的是**当前新鲜地址**（未使用过的地址） |
| ❗️❗️P0❗️❗️ | 选中新鲜地址代入 | 直接点击账户列表中展示新鲜地址的账户 | 代入发送表单的是**新鲜地址** |
| ❗️❗️P0❗️❗️ | 搜索旧地址能匹配 | 在搜索框粘贴该账户**已使用过的旧地址** | 1. 联想匹配到该账户<br>2. 搜索结果中展示的是**旧地址**（与搜索内容一致） |
| ❗️❗️P0❗️❗️ | 选中搜索到的旧地址代入 | 搜索旧地址 → 点击匹配结果 | 代入的是搜索结果展示的**旧地址**，而非新鲜地址 |
| P1 | 关闭新鲜地址时列表展示 | BTC 关闭新鲜地址 → 打开「账户」Tab | 账户展示的是固定的首个接收地址 |

## 5. 搜索与联想

**搜索范围**：钱包名称、账户名称、地址（含新鲜地址和历史旧地址）。搜索不区分大小写。

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 按账户名称搜索 | 在搜索框输入账户名称关键字（如 "piggy"） | 列表实时过滤，仅显示名称匹配的账户 |
| ❗️❗️P0❗️❗️ | 按钱包名称搜索 | 输入钱包名称关键字（如 "ran"） | 列表实时过滤，显示该钱包下所有账户 |
| ❗️❗️P0❗️❗️ | 按地址搜索 | 粘贴某账户地址的部分内容 | 列表过滤出地址匹配的账户 |
| P1 | 大小写不敏感搜索 | 输入 "PIGGY" 或 "Piggy" | 与小写搜索结果一致 |
| P1 | 搜索无匹配结果 | 输入不存在的关键字 | 显示空状态，不显示无关结果 |
| P1 | 清空搜索恢复列表 | 1. 输入关键字搜索<br>2. 清空搜索框 | 恢复完整账户列表，派生路径过滤恢复生效 |
| P1 | BTC/LTC 搜索时跨路径匹配 | BTC 当前选中 Native SegWit → 搜索一个 Legacy 路径的地址 | 1. 搜索结果**跨所有派生路径**，能找到 Legacy 地址<br>2. 结果中显示路径标签（如 "Legacy"） |
| P1 | 搜索结果 Tab 计数 | 输入关键字，「账户」Tab 有匹配结果 | Tab 标签显示匹配数量，如 "账户 (3)" |
| P2 | Tab 自动切换 | 在「最近」Tab 搜索关键字 → 最近 Tab 无匹配但账户 Tab 有匹配 | 自动切换到「账户」Tab |

## 6. 地址输入框联想（跨 Tab）

> 地址输入框的联想提示是跨「最近」「账户」「地址簿」三个数据源的全局行为。

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 输入我的账户地址触发联想 | 在地址输入框粘贴一个属于「我的账户」的地址 | 下方联想提示该地址属于我的账户，显示账户名称 |
| P1 | 输入的地址同时存在于多个来源 | 粘贴的地址同时在「最近转账」和「我的账户」中 | 联想提示同时标注来源（如既是最近转账又是我的账户） |
| P1 | 输入非本人账户地址 | 粘贴一个外部地址（不在三个来源中） | 不展示联想提示，直接进入金额输入流程 |

## 7. 选择代入

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 选择账户代入地址 | 点击「账户」Tab 中任一账户 | 1. 地址正确带入发送表单的收款地址框<br>2. 「选择地址」弹窗关闭<br>3. 进入金额输入流程 |
| ❗️❗️P0❗️❗️ | 不同链代入地址格式正确 | 分别在 EVM / BTC / Solana / Cosmos 下选择账户 | 代入的地址格式与当前链一致 |
| P1 | 代入后修改地址 | 选择账户代入后，手动修改收款地址 | 允许修改，不报错 |
| P1 | 多次选择替换 | 1. 选择账户 A 代入<br>2. 重新打开弹窗选择账户 B | 地址替换为账户 B 的地址 |

## 8. BTC/LTC 默认 Tab 与交互细节

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| P1 | BTC/LTC 默认 Tab | BTC 或 LTC 网络下打开「选择地址」弹窗 | 默认选中「账户」Tab（而非「最近」Tab） |
| P1 | 非 BTC/LTC 链默认 Tab | EVM / Solana / Cosmos 网络下打开弹窗 | 默认选中「最近」Tab |
| P1 | Lightning Network 不支持账户与地址簿选择 | Lightning 网络下打开「选择地址」弹窗 | 仅显示「最近」Tab；**「账户」和「地址簿」Tab 不显示**，不支持从我的账户或地址簿选择收款地址 |
| P2 | 钱包名称过长截断 | 钱包名称超长时 | 分组标题中钱包名称截断显示，路径切换按钮不被挤出可视区域 |
