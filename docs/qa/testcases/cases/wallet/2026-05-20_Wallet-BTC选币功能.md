## 前置条件与测试数据

1. 已导入 BTC 助记词钱包，账户**含 ≥ 5 个不同金额的 UTXO**（覆盖小额合并场景；推荐 1 个 dust UTXO + 几个常规金额）
2. 已在 `设置 - 钱包 - BTC 多地址模式` 开启多地址（确保 More 菜单可见 Coins、且 Coin Control 入口正常）
3. 当前账户余额足以支付一笔最小转账 + 网络费用
4. 至少 1 笔未确认（pending）+ 1 个找零（Change）UTXO，用于覆盖 Pending 标识与 Change 徽标
5. 区块浏览器（mempool.space / blockstream.info）用于交易后对账

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 列表字段 | 查看每行 | Checkbox + 序号 + 金额 + 缩略地址 + 时间 |

| ❗️❗️P0❗️❗️ | Change 徽标 | 找零方向 UTXO | 该行展示「Change」徽标 |
| ❗️❗️P0❗️❗️ | 点击行勾选 / 取消 | 点击某行（Checkbox 或其他区域）| Checkbox 状态翻转；底部统计同步 |
| ❗️❗️P0❗️❗️ | 底部已选择统计 | 查看底部 | 显示「已选择 N」+ 总额；全未勾选时为 0 |
| ❗️❗️P0❗️❗️ | 全选 / 反选 | 点击全选 Checkbox | 全部勾选 / 全部取消；统计同步 |
| ❗️❗️P0❗️❗️ | 已选金额 < 转账金额 | 取消勾选至总额 < 转账金额 | 「完成」按钮置灰 |

| P1 | 与 Coins (UTXO) 数据一致 | 选币功能勾选 → 关闭 → 进入 Coins (UTXO) | Coins (UTXO) 无 Checkbox，但 UTXO 集合与选币功能一致 |
| P2 | 空列表 | 账户无 UTXO | 列表空态；底部 `0 BTC`；「完成」置灰 |

## 5. 完成与代入发送

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 完成代入 | 勾选满足金额的 UTXO → 点击「完成」 | 模态关闭；所选 UTXO 作为转账输入；网络费按 UTXO 数量重算 |
| ❗️❗️P0❗️❗️ | 完成后再次进入保留状态 | 完成代入 → 再次点击选币入口 | 勾选状态、策略、排序全部保留 |
| ❗️❗️P0❗️❗️ | 完整发送链路 | 选币 → 完成 → 发送页提交 → 区块浏览器查 | 交易输入 = 所勾选 UTXO（txid:vout 一一对应）|
| P1 | 修改金额后重新进入 | 完成代入 → 修改转账金额 → 再次进入选币 | 已选金额不够覆盖新金额时自动重选；够则保留 |
| P1 | 修改接收地址不重置 | 修改接收地址（金额不变） | 选币状态保留 |
| P2 | 手续费随 UTXO 数变化 | 对比选 1 个 vs 选 5 个 UTXO | 选 5 个时网络费显著大于选 1 个 |

## 6. 边界与异常

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| P1 | 仅 1 个 UTXO | 账户只有 1 个 UTXO 且金额够 | 默认勾选；取消勾选则「完成」置灰 |
| P1 | UTXO 都 < 转账金额需合并 | 5 个小额 UTXO，单独不够 | 默认勾选累计金额 ≥ 转账金额的 UTXO；可完成代入 |
| P1 | UTXO 总额 = 转账金额（找零 = 0）| 勾选 UTXO 总额 = 转账金额 | 「完成」可点；发送时若网络费导致不足则提示余额不足 |
| P1 | Pending UTXO 可勾选 | 勾选 Pending UTXO | 允许勾选；按 PR 实际实现是否提示风险 |
| P2 | 加载中 | 首次打开模态 | 显示 Spinner |
| P2 | 网络断开 | 模态打开时断网 | 列表显示缓存或空态；不崩溃；恢复网络后可重新拉取 |
| P2 | 多语言 | 切换 App 语言 | 界面文案按语言本地化 |
