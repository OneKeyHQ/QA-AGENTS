# Wallet - Stellar (XLM) 硬件钱包测试

## 测试概览

**测试目标**：验证 Stellar 链在硬件钱包环境下的核心转账功能，重点验证硬件设备上签名详情的显示准确性。

**测试范围**：
- XLM 原生转账（主币转账）
- Asset Token 转账

**不支持功能**：
- Contract Token 转账：不支持
- Swap：不支持
- Soroban Token：不支持
- DApp 交易：不支持（Stellar 硬件钱包不支持 DApp 交易）

**测试环境**：iOS / Android / Extension / Desktop / Web，Mainnet，硬件钱包（所有型号）

---

## 1. XLM 原生转账

| 优先级 | 自动化层级 | 输入数据 | 操作步骤 | 预期结果 (含自动化断言) |
|--------|-----------|---------|---------|----------------------|
| P0 | E2E | 金额 = 最小值，Memo = 数字字符组合（28 字节） | 1. 进入转账页面<br>2. 连接硬件钱包<br>3. 输入收款地址（老账户）<br>4. 输入金额 = 链规则最小转账金额（向老账户转账无最小限制，但需预留 Gas，建议使用 0.0000001 XLM）<br>5. 输入 Memo = 数字字符组合，长度达到 28 字节（如 `1234567890123456789012345678`）<br>6. 确认转账<br>7. 在硬件设备上查看签名详情 | 1. 硬件设备显示：From 地址正确<br>2. 硬件设备显示：To 地址正确<br>3. 硬件设备显示：转账金额正确（最小值）<br>4. 硬件设备显示：链名称 "Stellar" 或 "XLM"<br>5. 硬件设备显示：Gas 费用相关信息<br>6. 硬件设备显示：Memo 内容（28 字节，完整显示或截断提示）<br>7. 硬件设备显示：Memo 类型为 MEMO_TEXT（数字字符组合）<br>8. 在硬件设备上确认交易<br>9. 交易成功<br>10. 发送方余额扣除正确<br>11. 接收方余额增加<br>12. 交易记录生成 |
| P0 | E2E | 金额 = 最大值，Memo = 空（不填写） | 1. 进入转账页面<br>2. 连接硬件钱包<br>3. 输入收款地址<br>4. 输入金额 = 账户最大可用余额（账户余额 - 1 XLM 最低余额 - Gas 费用）<br>5. 不填写 Memo<br>6. 确认转账<br>7. 在硬件设备上查看签名详情 | 1. 硬件设备显示：From 地址正确<br>2. 硬件设备显示：To 地址正确<br>3. 硬件设备显示：转账金额正确（最大值）<br>4. 硬件设备显示：链名称 "Stellar" 或 "XLM"<br>5. 硬件设备显示：Gas 费用相关信息<br>6. 硬件设备不显示 Memo 或显示 "无 Memo"<br>7. 在硬件设备上确认交易<br>8. 交易成功<br>9. 发送方余额保留 1 XLM（账户最低余额）<br>10. 接收方余额增加<br>11. 交易记录生成 |
| P0 | E2E | 金额 = 中间值，Memo = 纯数字 | 1. 进入转账页面<br>2. 连接硬件钱包<br>3. 输入收款地址<br>4. 输入金额 = 账户余额的 50%<br>5. 输入 Memo = 纯数字（如 `123456`）<br>6. 确认转账<br>7. 在硬件设备上查看签名详情 | 1. 硬件设备显示：From 地址正确<br>2. 硬件设备显示：To 地址正确<br>3. 硬件设备显示：转账金额正确<br>4. 硬件设备显示：链名称 "Stellar" 或 "XLM"<br>5. 硬件设备显示：Gas 费用相关信息<br>6. 硬件设备显示：Memo 内容（纯数字）<br>7. 硬件设备显示：Memo 类型为 MEMO_ID（纯数字自动推断）<br>8. 在硬件设备上确认交易<br>9. 交易成功<br>10. 余额扣除正确<br>11. 交易记录生成 |
