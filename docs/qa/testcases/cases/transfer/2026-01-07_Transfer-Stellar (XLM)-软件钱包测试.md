# Wallet - Stellar (XLM) 软件钱包测试

## 测试概览

**测试目标**：验证 Stellar 链在软件钱包环境下的完整功能，包括 Token 管理、转账、账户激活、dApp 连接等核心功能。

**测试范围**：
- Token 管理与类型支持（Stellar Asset Token、Contract Token、Stellar Asset Contract）
- XLM 原生转账（新账户创建、老账户转账、Memo 处理）
- Token 转账（Asset Token、Contract Token）
- 账户激活与保证金机制
- dApp 连接与 Swap
- 私钥导出与验证

**测试环境**：iOS / Android / Extension / Desktop / Web，Mainnet

## 8. 自动化实施方案

### 8.1 单元测试（Unit）重点

1. **Memo 类型推断逻辑**：
   - 纯数字 → MEMO_ID
   - 包含字符 → MEMO_TEXT
   - 边界值测试（空字符串、特殊字符）

2. **账户最低余额计算**：
   - 基础保留 1 XLM
   - Token 占用计算（每个 Asset Token 0.5 XLM）
   - 可用余额 = 总余额 - 锁定金额

3. **Token 类型识别逻辑**：
   - Asset Code:Issuer 格式识别
   - Contract Address（C 开头）识别
   - Asset 包装判断

### 8.2 端到端测试（E2E/API）重点

1. **Mock 数据**：
   - Horizon API 响应（余额、交易记录）
   - RPC 响应（Contract Token 余额、精度）
   - WebSocket 消息（余额更新）

2. **关键拦截点**：
   - 转账交易 Request Payload
   - Token 激活交易 Request Payload
   - dApp 连接请求

3. **断言验证**：
   - 交易 Hash 生成
   - 余额变化一致性（列表页、详情页、链上查询）
   - Memo 类型和内容正确性
