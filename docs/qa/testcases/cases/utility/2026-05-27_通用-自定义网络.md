# 通用 - 自定义网络

> 需求文档：`docs/qa/requirements/Utility-自定义网络.md`
> 规则文档：`docs/qa/rules/utility-rules.md`
> 测试端：Desktop / Extension / Web
> 变更说明：v1.0（2026-05-27）新增自定义网络列表页与网络详情流程用例

---

## 前置条件

1. 已创建钱包账户并完成解锁
2. 网络环境可访问 `/wallet/v1/network/chainlist`
3. 准备一组可用 EVM 网络参数（name / rpc / chainId / symbol / explorer）用于手动添加
4. 准备 dApp 场景（如 chainlist 或 `wallet_addEthereumChain`）覆盖“已存在网络”与“未存在网络”两种分支
5. 准备服务端下发同 chainId 覆盖测试数据（自定义网络与内置网络冲突场景）

## 1. 入口与页面结构

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 网络选择器入口 | 点击 `网络选择器 -> 添加自定义网络` | 1. 进入「自定义网络列表页」<br>2. 不直接进入详情表单 |
| ❗️❗️P0❗️❗️ | 设置入口 | 点击 `设置 -> 网络 -> 添加自定义网络` | 1. 进入「自定义网络列表页」<br>2. 不直接进入详情表单 |
| ❗️❗️P0❗️❗️ | 列表页基础结构 | 打开列表页后观察页面 | 1. 页面存在「手动添加」按钮（右上角）<br>2. 网络条目列表可见 |
| P1 | 页面命名一致性 | 从列表页选择网络进入详情页 | 详情页名称为「网络详情」（非“添加自定义网络”） |

## 2. 列表数据加载与异常

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 每次进入重新请求 | 首次进入列表页后退出，再次进入列表页 | 每次进入都重新请求 `/wallet/v1/network/chainlist` |
| ❗️❗️P0❗️❗️ | 网络异常提示 | 断网或 mock 接口失败后进入列表页 | 1. 显示网络异常/请求失败提示<br>2. 不提供页面内重试按钮 |
| P1 | 重试恢复 | 异常提示出现后恢复网络，退出页面并再次进入 | 列表重新加载并展示网络条目 |
| P1 | 空数据容错 | mock 返回空数组 | 显示空态，不崩溃，不白屏 |
