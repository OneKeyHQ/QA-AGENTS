## 自动化实施方案（P0/P1）

### 1. 单元测试（Unit）重点
- **收藏状态管理逻辑**：验证收藏/取消收藏的状态切换、状态持久化
- **数据同步逻辑**：验证多入口收藏状态的一致性检查机制
- **防抖/节流逻辑**：验证快速连续点击的防抖处理
- **数据校验**：验证 Token 标识（symbol/address/chainId）的格式校验

### 2. 端到端（E2E/API）重点
- **需要 Mock**：
  - 收藏/取消收藏 API（成功/失败/超时场景）
  - WebSocket 实时数据更新（如适用）
  - 多端数据同步接口
- **关键拦截点**：
  - 收藏 API 请求参数（Token 标识、用户 ID、时间戳）
  - 收藏 API 响应数据（成功标识、收藏列表更新）
  - 观察列表数据查询接口
- **UI 断言点**：
  - 收藏按钮状态（已收藏/未收藏图标变化）
  - 观察列表数据数量变化（+1/-1）
  - 多入口收藏状态一致性（观察列表/分类列表/详情页/搜索列表）
  - 网络异常时的错误提示和状态回滚
