# Prime - OneKey ID 信息编辑测试用例

> 需求文档：`docs/qa/requirements/Prime-OneKeyID信息.md`  
> 规则文档：`docs/qa/rules/prime-rules.md` 第 `4.4` 节  
> 测试端：iOS / Android / Desktop / Extension / Web  
> 更新日期：2026-05-26

## 前置条件

1. 已准备已登录 OneKey ID 账号（含 Prime 与非 Prime 两类），并可进入 OneKey ID 页面。
2. 已准备至少两台可同时登录的设备用于跨端同步校验。
3. iOS 设备可分别覆盖照片权限：首次未授权 / 已授权 / 拒绝授权。

---

## 1. 入口与页面结构

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 1. 小屏模式<br>2. 已登录 OneKey ID | 1. 进入 OneKey ID 页面<br>2. 点击个人信息栏 | 进入 OneKey ID 信息编辑页面 |
| ❗️❗️P0❗️❗️ | 1. 大屏模式<br>2. 已登录 OneKey ID | 1. 进入 OneKey ID 页面<br>2. 点击「编辑资料」按钮 | 进入 OneKey ID 信息编辑页面 |
| ❗️❗️P0❗️❗️ | 已登录 OneKey ID | 进入 OneKey ID 页面查看功能区 | 页面直接展示 Prime / 订阅管理（如有）/ 推荐奖励 / 已登录设备 / 我的订单 |
| P1 | 已登录 OneKey ID | 进入 OneKey ID 页面查看结构 | 页面不展示「更多」按钮 |
| P1 | 未登录状态 | 进入对应页面 | 菜单顶部显示 OneKey ID 注册/登录入口与默认头像，不显示编辑资料入口 |

---

## 2. 昵称编辑

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 当前已进入信息编辑页 | 1. 输入新昵称（1~20 字符）<br>2. 点击保存 | 1. 返回上一页<br>2. 昵称与输入一致 |
| ❗️❗️P0❗️❗️ | 当前已进入信息编辑页 | 1. 清空昵称<br>2. 观察保存按钮与错误提示 | 1. 触发不能为空校验<br>2. 保存不可提交 |
| ❗️❗️P0❗️❗️ | 当前已进入信息编辑页 | 1. 输入 21 字符昵称 | 第 21 字符不可继续输入 |
| P1 | 当前已进入信息编辑页 | 输入中英文/数字/符号/Emoji 混合昵称并保存 | 校验通过时昵称展示与输入一致 |
| P1 | 当前昵称超长（历史数据） | 返回 OneKey ID 页面观察昵称区域 | 按 UI 规则省略显示，不破坏布局 |

---

## 3. 头像编辑

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | iOS 首次未授权照片权限 | 1. 进入信息编辑页<br>2. 点击头像编辑 | 触发系统照片权限申请 |
| ❗️❗️P0❗️❗️ | 非 iOS（Android/Desktop/Extension/Web） | 1. 进入信息编辑页<br>2. 点击头像编辑 | 打开系统图片选择器 |
| ❗️❗️P0❗️❗️ | 已选中图片资源 | 1. 进入裁剪流程<br>2. 保存头像 | 1. 头像预览更新<br>2. 保存后 OneKey ID 页面头像更新 |
| P1 | iOS 权限被拒绝 | 再次点击头像编辑 | 出现可理解的权限提示与引导（按端上实现） |

## 4. 图标提示层（Prime 状态图标）

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 1. 桌面/插件端<br>2. Prime 状态图标可见 | 鼠标指向 Prime 状态图标 | 触发提示层展示逻辑 |
| ❗️❗️P0❗️❗️ | 1. 移动端<br>2. Prime 状态图标可见 | 点击 Prime 状态图标 | 触发提示层展示逻辑 |
| ❗️❗️P0❗️❗️ | 当前状态存在到期时间（赠送或中断订阅） | 触发提示层 | 提示层展示 Prime 到期时间 |
| ❗️❗️P0❗️❗️ | 当前设备超限（黄标） | 触发提示层 | 提示层展示当前设备超限信息，并包含「设备数量已达上限」 |
| P1 | 当前无到期时间且设备未超限 | 触发提示层交互 | 不展示提示层 |

## 5. 异常与跨端同步

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 编辑保存时网络断开 | 1. 修改昵称或头像<br>2. 点击保存 | 1. 显示保存失败提示<br>2. 不显示保存完成态 |
| ❗️❗️P0❗️❗️ | 多端已登录同一 OneKey ID | 1. 设备 A 修改昵称/头像并保存<br>2. 设备 B 进入 OneKey ID 页面 | 设备 B 同步展示最新昵称与头像 |
| P1 | 设备 B 已在页面停留 | 设备 A 修改后观察设备 B | 设备 B 在 ≤30s 内刷新为最新信息（或重进后一致） |

---

## 6. 测试数据建议

| 数据类型 | 说明 |
| --- | --- |
| 账号 A | Prime 订阅中（无到期时间）且设备未超限 |
| 账号 B | Prime 赠送或中断订阅（有到期时间） |
| 账号 C | Prime 设备超限（黄标） |
| 昵称样本 | 1 字符、20 字符、21 字符、中英混合、Emoji |
| 图片样本 | 正常图片、超大尺寸图片（按端上实现校验） |
