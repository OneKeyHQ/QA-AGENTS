# Prime - 批量转账

> 规则文档：`docs/qa/rules/prime-rules.md` - 第 4.11 章
> 测试端：iOS / Android / Extension / Desktop
> 变更说明：按「一对多 / 多对一 / 多对多」重构为完整流程；网络改为变量；回归最小集固定为 Polygon / Solana / Tron
> 关联用例：Prime 权益校验
> 代码核对：`app-monorepo` branch `x`，2026-04-14 核对

---

## 前置条件

1. 已登录 Prime 用户（非 Prime 用于入口拦截校验）
2. 钱包有可转账资产（主币/代币）
3. 可切换网络；回归最小集网络为 Polygon / Solana / Tron

---

## 网络变量与回归策略

| 模式 | 网络变量（全量） | 回归最小集 |
| --- | --- | --- |
| 一对多 | ETH / BSC / Arbitrum / Polygon / Base / Optimism / Avalanche / Linea / zkSync Era / Tron / Solana / Bitcoin | **Polygon** |
| 多对一 | 除 Lightning Network / Nostr 外 | **Solana** |
| 多对多 | 除 Lightning Network / Nostr 外 | **Tron** |

## 2. 输入与地址校验（通用，集中）

> 适用于一对多/多对一/多对多；格式与错误提示在本节统一校验。

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 地址输入来源 | 手输/粘贴/地址簿多选/CSV 导入 | 地址按行展示；每行一个地址；导入/选择内容写入输入框 |
| ❗️❗️P0❗️❗️ | 仅地址格式 | 输入 `addr1` / `addr2` / `addr3` → 下一步 | 识别为仅地址模式；进入金额设置；显示指定金额/范围金额 |
| ❗️❗️P0❗️❗️ | 地址+金额格式 | 输入 `addr1,100` / `addr2,200` → 下一步 | 识别为地址+金额模式；进入金额设置；显示指定金额/范围金额/自定义金额 |
| ❗️❗️P0❗️❗️ | 混用格式 | 输入 `addr1` / `addr2,200` → 下一步 | 拦截并提示格式混用；错误行高亮；不可进入下一步 |
| ❗️❗️P0❗️❗️ | 无效地址 | 输入无效地址（如 `0x123`）→ 下一步 | 显示地址无效提示；错误行高亮；不可进入下一步 |
| ❗️❗️P0❗️❗️ | 重复地址（模式差异） | 输入重复地址 → 下一步 | 一对多/多对多：黄色警告+确认后可继续；多对一：红色错误+不可继续 |
| P1 | 空行/空格行 | 输入包含空行/空格行 → 下一步 | 自动过滤空行/空格行 |

## 3. 一对多（完整流程，网络变量）

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 一对多启动（网络=变量） | 选择一对多 → 网络选择 `{network}` → 资产选择 `{native/token}` | 进入地址输入页；网络/资产切换生效；BTC 显示地址类型切换器（Taproot/SegWit/Legacy） |
| ❗️❗️P0❗️❗️ | 一对多金额设置 | 仅地址输入后，使用指定金额/范围金额（参数：最小/中间/Max） | 金额可输入与校验；总金额实时更新；余额不足/低于最小转账金额时拦截 |
| ❗️❗️P0❗️❗️ | 一对多预览 | 进入预览页，检查发送总额/网络费/地址概览 | 显示发送总额（含法币）/网络费/地址概览；一对多不显示间隔设置 |
| ❗️❗️P0❗️❗️ | 一对多执行（网络差异） | 点击确认发送 | EVM/Tron 代币：先授权后批量调用；BTC：>200 地址自动拆分；SOL：按指令大小拆分并处理 ATA 租金；完成后提示并返回 |
| ❗️❗️P0❗️❗️ | 一对多失败/取消 | 模拟失败/签名弹窗取消 | 失败显示错误原因且可重试；取消后返回预览页且未发起交易 |

## 4. 多对一（完整流程，网络变量）

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 多对一启动（网络=变量） | 选择多对一 → 网络选择 `{network}` → 输入发送方/接收方数据 → 下一步 | 进入金额设置；网络保留当前选择；地址与金额校验沿用通用规则 |
| ❗️❗️P0❗️❗️ | 多对一间隔设置 | 进入间隔设置，选择指定范围/无间隔；输入 `{min}/{max}` | 默认 5s/30s；`min>=max` 或 `>600s` 拦截；无间隔时输入框隐藏 |
| ❗️❗️P0❗️❗️ | 多对一预览 | 进入预览页检查总额/网络费/间隔/地址概览 | 显示总额、网络费、间隔值（指定范围时）、地址概览 |
| ❗️❗️P0❗️❗️ | 多对一执行进度 | 点击确认进入批量发送状态页，观察待处理→处理中→终态 | 列表逐笔更新；每笔显示 hash/费用；失败显示原因；支持暂停/恢复/中止 |

## 5. 多对多（完整流程，网络变量）

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 多对多启动（网络=变量） | 选择多对多 → 网络选择 `{network}` → 输入多组地址/金额 → 下一步 | 进入金额设置；地址+金额可在自定义金额页逐行查看/编辑 |
| ❗️❗️P0❗️❗️ | 多对多间隔与预览 | 设置间隔（指定范围/无间隔）→ 进入预览 | 间隔规则与多对一一致；预览显示总额/网络费/间隔/地址概览 |
| ❗️❗️P0❗️❗️ | 多对多执行进度 | 点击确认，观察逐笔发送与状态变化 | 状态按待处理→处理中→成功/失败/跳过；可暂停/恢复/中止；跳过显示原因 |
| ❗️❗️P0❗️❗️ | Tron 特殊执行 | 网络=`Tron`，执行多对多 | 逐笔发送间隔约 300ms；资源租赁参数传递正确 |
