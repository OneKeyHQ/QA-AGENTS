# Wallet 模块测试规则文档

> 本文档记录 Wallet 模块的核心测试规则。
> 生成 Wallet 模块测试用例时，必须参考本文档。

---

## 1. 模块概述

Wallet 模块包含以下核心功能：
- 钱包首页（资产展示、余额刷新）
- 转账（主币、代币、NFT）
- Token 管理（添加、隐藏、搜索）
- 历史记录（交易列表、详情）
- NFT（展示、发送）
- 法币出入金
- 授权管理

---

## 2. 钱包首页规则

### 2.1 资产展示

| 规则项 | 规则描述 |
|-------|---------|
| 余额计算 | 显示所有 Token 余额的法币总价值 |
| 隐藏资产 | 支持隐藏小额/归零资产，不影响总余额计算 |
| 刷新策略 | 下拉刷新、自动刷新（间隔可配置） |
| 多链展示 | 按链分组或统一展示（可切换） |

### 2.2 Token 列表

| 规则项 | 规则描述 |
|-------|---------|
| 排序 | 按余额法币价值降序（或自定义排序） |
| 搜索 | 支持按名称、符号搜索 |
| 筛选 | 支持按链筛选 |
| 空状态 | 无 Token 时显示引导添加 |

## 3. 转账规则

### 3.1 核心功能覆盖

**软件钱包测试用例应包含**：
- (a) 主币转账
- (b) 代币转账（含普通代币、免 Gas 代币）
- (c) 私钥转账（如适用）
- (d) 私钥导入导出
- (e) DApp 交易（连接、签名、合约交互）

**硬件钱包测试用例应包含**（仅以下三项）：
- (a) 主币转账
- (b) 代币转账（含普通代币、免 Gas 代币）
- (c) DApp 交易（连接、签名、合约交互）

### 3.2 钱包类型分组规则（强制）

**核心原则**：生成转账测试用例时，必须将软件钱包和硬件钱包分开，输出为两篇独立文档。

**分组方式**：
1. **软件钱包文档**：`YYYY-MM-DD_Wallet-<链名>-软件钱包测试.md`
   - 保存目录：`docs/qa/testcases/cases/wallet/`
2. **硬件钱包文档**：`YYYY-MM-DD_Hardware-<链名>-硬件钱包测试.md`
   - 保存目录：`docs/qa/testcases/cases/hardware/`

### 3.3 金额转账测试规则

所有涉及金额转账的用例必须覆盖：

| 金额类型 | 定义 | 注意事项 |
|---------|-----|---------|
| 最大值 | 钱包余额（扣除账户最低余额和 Gas） | 考虑锁定金额 |
| 最小值 | 链规则允许的最小转账金额 | 参考各链规则 |
| 中间值 | 余额的中间值 | 覆盖正常场景 |

### 3.4 转账状态机

```
构建 → 签名 → 广播 → Pending → Success/Fail → 加速/取消/替换
```

### 3.5 各链转账规则

**详细的各链转账规则请参考**：`docs/qa/rules/transfer-chain-rules.md`

包含内容：
- 各链最小转账金额
- 账户最低余额限制
- Gas 规则
- Memo/Tag 规则
- 硬件钱包支持情况

## 4. Token 管理规则

### 4.1 添加 Token

| 场景 | 规则 |
|-----|------|
| 搜索添加 | 支持按名称、符号、合约地址搜索 |
| 手动添加 | 输入合约地址，自动获取 Token 信息 |
| 推荐 Token | 显示热门/常用 Token 列表 |
| 重复添加 | 已添加的 Token 显示"已添加"状态 |

### 4.2 隐藏 Token

| 场景 | 规则 |
|-----|------|
| 手动隐藏 | 支持隐藏任意 Token |
| 自动隐藏 | 可配置自动隐藏小额/归零 Token |
| 恢复显示 | 支持从隐藏列表恢复 |

### 4.3 Token 详情

| 字段 | 规则 |
|-----|------|
| 余额 | 显示 Token 数量和法币价值 |
| 合约地址 | 显示并支持复制 |
| 精度 | 显示 Token decimals |
| 价格 | 显示当前价格和涨跌幅 |

## 5. 历史记录规则

### 5.1 交易列表

| 规则项 | 规则描述 |
|-------|---------|
| 排序 | 按时间倒序 |
| 分页 | 支持下拉加载更多 |
| 筛选 | 支持按类型（发送/接收/Swap 等）筛选 |
| 状态 | 显示 Pending/Success/Failed 状态 |

### 5.2 交易详情

| 字段 | 规则 |
|-----|------|
| 交易哈希 | 显示并支持复制、跳转区块浏览器 |
| 时间 | 显示交易时间 |
| From/To | 显示发送方和接收方地址 |
| 金额 | 显示转账金额和法币价值 |
| Gas 费 | 显示实际消耗的 Gas 费 |
| 状态 | 显示交易状态 |

### 5.3 状态更新

| 场景 | 规则 |
|-----|------|
| Pending 状态 | 定时轮询更新状态 |
| 确认数 | 显示当前确认数/所需确认数 |
| 失败处理 | 显示失败原因 |

## 7. 通用规则

### 状态机
- 钱包首页加载 → 资产刷新 → Token 管理 → 历史记录

### 资金一致性
- 资产页/详情页/交易记录三处一致
- 隐藏资产不影响总余额

### 地址/规则
- 合约地址拦截/提示
- 自转自
- Memo/Tag 校验
- ENS/域名解析

### 链上不确定性
- Gas 波动
- RPC 不一致
- 链重组导致状态回滚
- 广播失败重试

### 风控
- 黑名单地址
- 风险合约提示
- 大额二次确认
- 频率限制

### 可观测
- 失败原因必须落到可验证字段
- error code + 用户文案 + hash/nonce

## 8. 规则维护指南

### 如何更新规则

1. **发现规则变更**：
   - 在测试过程中发现规则与文档不一致
   - 收到产品/开发通知规则变更

2. **更新文档**：
   - 通用规则更新本文档
   - 各链具体规则更新 `transfer-chain-rules.md`
   - 在变更记录中记录更新时间和原因
