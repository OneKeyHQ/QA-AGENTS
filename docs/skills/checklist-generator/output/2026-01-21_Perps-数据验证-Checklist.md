# Perps 数据验证 - 冒烟测试 Checklist

> 生成时间：2026-01-21  
> 测试 URL：https://app.onekeytest.com/  
> 数据源：Hyperliquid (HL) WebSocket  
> 验证方式：对比 WebSocket 数据与 UI 显示是否一致  
> 执行模式：交互式（用户点击，系统验证）
>
> **执行模式说明**：
> - 本测试为交互式验证，用户执行操作，系统验证数据
> - 系统会注入 WebSocket 监控脚本，捕获实时数据
> - 每个验证点完成后，系统报告验证结果

## 操作步骤

### 场景 1：Token 列表数据验证

**数据源说明**：数据来自 Hyperliquid，订阅了 crypto 和 hip3 xyz 的数据

- [P0] (a) 进入 Perps 页面，等待页面加载完成。**系统验证**：捕获 WebSocket 订阅消息，确认 crypto 和 hip3 xyz 数据源已连接
- [P0] (b) 查看 Token 列表，观察各 Token 数据显示。**系统验证**：对比以下字段 WS 数据与 UI 显示是否一致
  - 资产名称 (coin)
  - 最新价格 (lastPrice)
  - 24小时涨跌 (change24h)
  - 资金费率 (fundingRate)
  - 成交量 (volume24h)
  - 合约持仓量 (openInterest)
- [P1] (c) 点击切换不同 Token（如 BTC → ETH）。**系统验证**：数据更新延迟 ≤ 1 秒，新 Token 数据正确加载

---

### 场景 2：订单簿数据验证（10 档）

**数据传输方式**：WebSocket 实时推送

- [P0] (a) 查看订单簿买单区域。**系统验证**：
  - 买单按价格从高到低排序
  - 共显示 10 档数据
- [P0] (b) 查看订单簿卖单区域。**系统验证**：
  - 卖单按价格从低到高排序
  - 共显示 10 档数据
- [P0] (c) 验证订单簿「总计」列计算。**系统验证**：
  - 第 1 行总计 = 第 1 行数量
  - 第 N 行总计 = 第 1 行到第 N 行数量之和
  - 计算公式：`cumulative[n] = Σ(size[1..n])`
- [P1] (d) 验证订单簿背景色百分比。**系统验证**：
  - 第 x 行百分比 = (第 1 行到第 x 行累计量) / (第 1 行到第 N 行总量) × 100%
  - 计算公式：`percent[x] = cumulative[x] / cumulative[N] × 100%`
  - 当背景色百分比过低时，显示默认背景色
- [P0] (e) 点击订单簿中任意一档价格。**系统验证**：
  - 市价单自动切换为限价单模式
  - 点击的价格自动填入「价格」输入框
- [P1] (f) 切换到其他币种（如 ETH）。**系统验证**：
  - 订单簿数据更新及时（延迟 ≤ 1 秒）
  - 新币种的买卖单数据正确显示

---

### 场景 3：K 线图数据验证

**数据传输方式**：Hyperliquid WebSocket 推送

- [P0] (a) 查看 K 线图上方价格信息。**系统验证**：对比 WS 数据与 UI 显示
  - 标记价格 (markPrice)
  - 预言机价格 (oraclePrice)
  - 24小时成交量 (volume24h)
  - 资金费率 (fundingRate)
- [P1] (b) 观察 K 线图实时更新。**系统验证**：
  - K 线数据能正常返回
  - 买卖点数据（如有）能正常显示
- [P1] (c) 切换 K 线时长（1m / 5m / 15m / 1h / 4h / 1d）。**系统验证**：
  - 切换后数据能正常返回
  - K 线周期与选择一致
- [P2] (d) 添加多个技术指标（如 MA、MACD、RSI）。**系统验证**：
  - 指标参数正确显示
  - 指标数据计算正确
- [P2] (e) 使用鼠标滚轮或按钮放大/缩小/还原图表。**系统验证**：
  - 功能正常响应
  - 图表比例调整正确

---

### 场景 4：限价单最优价 (BBO) 验证

**数据说明**：点击下单时由 Hyperliquid 返回当前最优价格

- [P0] (a) 切换到限价单模式。**系统验证**：
  - UI 显示限价单输入界面
  - 价格输入框可编辑
- [P0] (b) 启用「最优价」选项（如有 BBO 开关）。**系统验证**：
  - 捕获 BBO 订阅数据
  - 价格输入框显示 BBO 价格
- [P0] (c) 查看当前 BBO 数据。**系统验证**：
  - buy1 (买1价) = 订单簿买单第一档价格
  - sell1 (卖1价) = 订单簿卖单第一档价格
  - BBO 数据与订单簿数据一致
- [P1] (d) 点击「做多」/「做空」按钮准备下单。**系统验证**：
  - 订单确认页显示的价格为 HL 返回的最新 BBO 价格
  - 价格可能与展示时有偏差（使用最新值提交）
