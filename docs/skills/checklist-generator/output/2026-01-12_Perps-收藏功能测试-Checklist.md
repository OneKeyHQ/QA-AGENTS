# Perps - 收藏功能测试 Checklist

> 生成时间：2026-01-12  
> 来源用例：`docs/testcases/cases/perps/2026-01-12_Perps-收藏功能测试.md`
> 
> **执行模式说明**：
> - `smoke` / `smoke-fast`：只执行 [P0] 标记的核心场景（约 20%）
> - `smoke-full`：执行所有场景（P0/P1/P2）
> 
> **备注**：收藏功能不需要登录，可以忽略云备份相关的报错。此功能不支持移动端，仅测试 Desktop/Extension 端。

## 前置条件
- [ ] 浏览器已打开，可访问 Perps 模块
- [ ] Perps 模块已加载完成，用户无需登录
- [ ] 网络连接正常
- [ ] 测试环境为 Desktop 或 Extension 端（不支持移动端）

## 操作步骤

### 场景1：默认推荐代币收藏（6个代币推荐列表）
- [P0] (a) 进入 Perps 模块，首次进入或未收藏任何代币，查看默认推荐代币列表，验证显示默认推荐的 6 个代币（BTCUSDC、ETHUSDC、BNBUSDC、SOLUSDC、HYPEUSDC、XRPUSDC），每个代币右侧显示复选框且默认全部勾选，推荐列表底部显示「添加到自选」按钮，推荐列表以两列网格布局展示
- [P0] (b) 在默认推荐列表中取消勾选部分代币（如取消 BTCUSDC、ETHUSDC），点击「添加到自选」按钮，验证取消勾选的代币复选框状态变为未勾选，点击「添加到自选」后仅已勾选的代币添加到自选，自选列表显示已勾选的代币（数量=4），行情页面顶部显示已勾选的代币（数量=4），未勾选的代币（BTCUSDC、ETHUSDC）不出现在自选列表和行情页顶部
- [P0] (c) 在默认推荐列表中取消所有代币的勾选，点击「添加到自选」按钮，验证所有复选框变为未勾选状态，点击「添加到自选」后无代币添加到自选，自选列表为空或保持原有状态，行情页面顶部不显示新添加的代币
- [P0] (d) 在推荐列表中勾选已收藏的代币，点击「添加到自选」按钮，验证已收藏的代币在推荐列表中仍可勾选，点击「添加到自选」后不产生重复添加，自选列表中的代币不重复显示，自选列表数量正确（不增加重复项）
- [P1] (e) 在推荐列表中勾选部分代币，切换其他分类标签（如切换到「PERPS」或「HIP3」），返回「自选」标签，验证切换分类后推荐列表的勾选状态保持不变，返回「自选」标签时推荐列表显示之前的勾选状态，未点击「添加到自选」前勾选状态仅保存在当前会话
- [P1] (f) 已通过推荐列表添加过代币，刷新页面，验证刷新后推荐列表仍显示默认的 6 个代币，所有复选框默认勾选状态，已添加到自选的代币在推荐列表中仍可勾选（用于重新添加，如之前取消收藏）

### 场景2：搜索列表收藏/取消收藏
- [P0] (a) 点击「搜索资产」输入框，输入代币名称（如"BTC"），在搜索结果列表中点击某个代币的收藏按钮（星形图标），验证搜索结果列表显示匹配的代币，每个代币右侧显示收藏按钮（未收藏状态：空星形图标），点击收藏按钮后按钮状态变为已收藏（实心星形图标），自选列表立即显示该代币（数量+1），行情页面顶部立即显示该代币，搜索结果列表中该代币显示已收藏状态
- [P0] (b) 搜索已收藏的代币（如"XRP"），在搜索结果列表中点击已收藏代币的收藏按钮（实心星形图标），验证搜索结果列表中该代币显示已收藏状态（实心星形图标），点击收藏按钮后按钮状态变为未收藏（空星形图标），自选列表立即移除该代币（数量-1），行情页面顶部立即移除该代币，搜索结果列表中该代币显示未收藏状态
- [P0] (c) 搜索"USDC"，在搜索结果中分别点击未收藏代币的收藏按钮，点击已收藏代币的取消收藏按钮，验证未收藏代币点击后变为已收藏状态，已收藏代币点击后变为未收藏状态，自选列表正确更新（新增的显示，取消的移除），自选列表数量与操作一致，行情页面顶部同步更新
- [P0] (d) 搜索主币（如"BTC"、"ETH"），在搜索结果中点击收藏按钮，验证主币收藏成功（按钮状态变化），自选列表显示主币（数量+1），行情页面顶部显示主币，主币在分类列表中显示已收藏标识
- [P1] (e) 搜索不存在的代币名称（如"ABCDEFG123"），验证显示"未找到相关结果"或空状态提示，无收藏按钮或收藏按钮不可用，搜索框仍可继续输入
- [P1] (f) 在搜索结果中快速连续点击同一代币的收藏按钮 3 次，等待操作完成，验证防抖生效（仅执行最后一次操作），最终状态与最后一次点击一致，自选列表数据与最终状态一致，无重复请求或状态错乱，按钮状态正确（已收藏/未收藏）
- [P1] (g) 输入代币名称的部分字符（如"BT"），在搜索结果中收藏代币，验证搜索结果包含匹配的代币，收藏操作不受搜索关键词影响，收藏后搜索结果中该代币显示已收藏状态，自选列表正确显示已收藏代币

### 场景3：自选列表展示与管理
- [P0] (a) 已收藏多个代币，点击「自选」标签，验证自选列表显示所有已收藏的代币，列表按收藏时间或默认顺序排列，每个代币显示完整信息（名称、价格、涨跌幅等），每个代币右侧显示已收藏标识（实心星形图标）
- [P0] (b) 自选列表有多个代币，在自选列表中点击某个代币的取消收藏按钮，验证点击取消收藏后该代币从自选列表移除，自选列表数量-1，行情页面顶部同步移除该代币，该代币在其他分类列表中显示未收藏状态
- [P1] (c) 自选列表为空（无收藏），点击「自选」标签，验证显示空状态文案（如"暂无收藏"或"添加自选代币"），显示引导添加收藏的提示或按钮，推荐列表显示默认的 6 个代币
- [P1] (d) 自选列表有大量收藏代币（>50），点击「自选」标签，滚动列表到底部，验证列表支持滚动加载，所有收藏代币正确展示，滚动流畅无卡顿（FPS ≥ 30），列表项加载无延迟或闪烁
- [P1] (e) 自选列表有多个代币，在自选列表中点击某个代币名称或行，验证点击代币后跳转到该代币的交易页面，跳转后页面显示该代币的详细信息，交易页面显示该代币的已收藏状态

### 场景4：行情页面顶部展示与切换显示模式
- [P0] (a) 已收藏多个代币（至少 6 个），进入行情页面（交易页面），查看页面顶部的代币列表，验证页面顶部显示已收藏的代币列表，列表横向排列显示代币图标、名称、价格变化，默认显示百分比模式（如"-2.80%"），列表支持横向滚动（如收藏代币数量 > 6）
- [P0] (b) 行情页面顶部显示已收藏代币，当前显示百分比模式，点击页面顶部的「$」按钮（切换为涨跌幅模式），验证点击「$」后所有代币的价格变化显示切换为涨跌幅（绝对数值，如"-0.0584"），「$」按钮变为激活状态（高亮或选中样式），「%」按钮变为未激活状态，所有代币的价格变化数值正确显示
- [P0] (c) 行情页面顶部显示已收藏代币，当前显示涨跌幅模式（「$」激活），点击页面顶部的「%」按钮（切换为百分比模式），验证点击「%」后所有代币的价格变化显示切换为百分比（如"-2.80%"），「%」按钮变为激活状态，「$」按钮变为未激活状态，所有代币的价格变化百分比正确显示
- [P0] (d) 行情页面顶部显示已收藏代币，已切换到百分比模式，在行情页面顶部点击某个代币（如"BTC"），验证点击代币后页面切换到该代币的交易视图，交易视图显示该代币的详细信息（价格、K线图等），页面顶部的代币列表保持不变，当前选中的代币高亮显示
- [P0] (e) 已收藏代币，行情页面顶部显示代币列表，在自选列表中添加新代币，验证添加新代币后行情页面顶部立即显示新代币（延迟 ≤ 1 秒），新代币出现在列表正确位置，列表数量+1，新代币的价格变化正确显示（百分比或涨跌幅，取决于当前模式）
- [P0] (f) 已收藏代币，行情页面顶部显示代币列表，在自选列表中取消收藏某个代币，验证取消收藏后行情页面顶部立即移除该代币（延迟 ≤ 1 秒），列表数量-1，该代币从列表中消失，其他代币位置和显示正常
- [P1] (g) 行情页面顶部显示代币列表，当前显示百分比模式，切换到涨跌幅模式（「$」），刷新页面，验证刷新后显示模式保持为涨跌幅模式（「$」激活），显示模式持久化保存（localStorage 或用户偏好），所有代币的价格变化以涨跌幅显示
- [P1] (h) 行情页面顶部显示大量收藏代币（>10），横向滚动代币列表，验证列表支持平滑横向滚动，滚动过程中所有代币正确显示，滚动流畅无卡顿（FPS ≥ 30），滚动到最右侧时显示所有收藏代币
- [P1] (i) 行情页面顶部显示代币列表，快速连续点击「$」和「%」按钮 5 次，验证防抖生效（仅执行最后一次点击），最终显示模式与最后一次点击一致，无状态错乱或闪烁，按钮状态正确（激活/未激活）

### 场景5：数据一致性与状态同步
- [P0] (a) 在推荐列表勾选代币并点击「添加到自选」，切换到「PERPS」分类列表，核对该代币的收藏状态，在搜索列表中搜索该代币，验证推荐列表添加成功，「PERPS」分类列表中该代币显示已收藏标识，搜索列表中该代币显示已收藏状态，三个入口的收藏状态完全一致
- [P0] (b) 在搜索列表收藏代币 A，切换到「自选」标签，核对代币 A 是否出现在自选列表，查看行情页面顶部，验证自选列表立即显示代币 A（或刷新后显示，延迟 ≤ 1 秒），代币 A 出现在自选列表正确位置，行情页面顶部立即显示代币 A，所有入口状态同步（延迟 ≤ 1 秒）
- [P0] (c) 在自选列表取消收藏代币 B，切换到「PERPS」分类列表，核对代币 B 的收藏状态，查看行情页面顶部，验证「PERPS」分类列表中代币 B 显示未收藏状态，行情页面顶部立即移除代币 B，搜索列表中代币 B 显示未收藏状态，所有入口状态同步（延迟 ≤ 1 秒）
- [P0] (d) 在任意入口收藏代币，抓取收藏 API 请求，验证 API 请求包含正确的代币标识（symbol/pairId），响应状态码=200，返回收藏成功标识，响应数据包含收藏时间戳，请求幂等性：重复收藏返回相同结果
- [P0] (e) 在任意入口取消收藏代币，抓取取消收藏 API 请求，验证 API 请求包含正确的代币标识（symbol/pairId），响应状态码=200，返回取消收藏成功标识，请求幂等性：重复取消收藏返回相同结果
- [P1] (f) 多标签页打开同一应用，在标签页 A 收藏代币，在标签页 B 刷新 Perps 页面，验证标签页 B 的自选列表显示标签页 A 收藏的代币，标签页 B 的分类列表显示已收藏标识，标签页 B 的行情页面顶部显示已收藏代币，跨标签页数据同步（延迟 ≤ 5 秒）
- [P1] (g) 收藏列表有大量代币（>100），进入自选列表，滚动到底部，执行收藏/取消收藏操作，验证列表滚动流畅（FPS ≥ 30），收藏/取消收藏操作响应时间 ≤ 500ms，操作后列表更新正确，无闪烁或跳动，行情页面顶部列表更新正确

### 场景6：网络异常与容错
- [P1] (a) 点击收藏按钮后立即断开网络，观察 UI 状态，验证显示网络错误提示（如"网络异常，请稍后重试"），收藏状态回滚到操作前状态，提供重试按钮或自动重试机制，按钮状态恢复为操作前状态

- [P1] (c) Mock 收藏 API 返回 500 错误，执行收藏操作，验证显示错误提示（如"服务异常，请稍后重试"），收藏状态不改变，提供重试入口，按钮状态恢复为操作前状态
- [P1] (d) 模拟弱网环境（延迟 > 3s），执行收藏操作，观察加载状态，验证显示加载状态（按钮显示 loading 或禁用），操作完成后状态正确更新，无重复请求，自选列表和行情页顶部正确更新

### 场景7：边界与异常场景
- [P1] (a) 收藏数量达到上限（如 100），尝试收藏第 101 个代币，验证显示上限提示（如"收藏数量已达上限"），收藏操作不执行，已收藏的代币仍可正常取消收藏，取消收藏后可继续添加新代币

- [P2] (e) 执行收藏操作，立即切换应用到后台，5 秒后返回应用，验证收藏操作在后台完成，返回应用后状态正确，自选列表数据已更新，行情页面顶部列表已更新
- [P2] (f) 调整浏览器窗口大小，核对推荐列表、自选列表、行情页顶部布局，验证推荐列表布局自适应窗口大小（两列网格保持），自选列表布局自适应，行情页面顶部列表支持横向滚动（如窗口缩小），所有元素不重叠或溢出
