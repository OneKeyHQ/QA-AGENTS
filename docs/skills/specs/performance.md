# 性能指标采集规范

> 本节定义性能指标的定义、阈值和报告格式。
> 
> **依赖**：采集实现方式见 `chrome-mcp.md` 第6节。

---

## 1）性能指标定义

### Core Web Vitals（必须采集）
- **LCP (Largest Contentful Paint)**：最大内容绘制时间，目标 < 2.5s
- **FID (First Input Delay)**：首次输入延迟，目标 < 100ms
- **CLS (Cumulative Layout Shift)**：累积布局偏移，目标 < 0.1

### 加载性能指标
- **FCP (First Contentful Paint)**：首次内容绘制，目标 < 1.8s
- **TTI (Time to Interactive)**：可交互时间，目标 < 3.8s
- **TBT (Total Blocking Time)**：总阻塞时间，目标 < 200ms

### 资源性能指标
- **资源加载时间**：JS/CSS/图片等资源加载耗时
- **资源大小**：各类型资源体积统计
- **资源缓存命中率**：304/200 比例

### 网络性能指标
- **请求总数**：页面加载期间的所有网络请求
- **请求耗时分布**：P50/P90/P95/P99 分位数
- **失败率**：4xx/5xx 错误占比
- **重定向次数**：301/302 跳转次数

### 运行时性能指标
- **FPS (Frames Per Second)**：页面滚动/交互时的帧率，目标 ≥ 60fps
- **内存占用**：页面内存使用量（通过 Performance API）
- **CPU 使用率**：长时间运行时的 CPU 占用

---

## 2）性能阈值规则

- **P0 级别**：Core Web Vitals 必须全部达标
- **P1 级别**：加载性能指标至少 80% 达标
- **P2 级别**：资源优化建议（非阻塞）

### 步骤耗时阈值

| 状态 | 阈值 | 说明 |
| --- | --- | --- |
| ✅ 正常 | < 500ms | 响应流畅 |
| ⚠️ 偏慢 | 500ms ~ 2000ms | 可感知延迟，需关注 |
| ❌ 过慢 | > 2000ms | 严重影响体验，需优化 |

---

## 3）报告保存规则

- **目录**：`docs/testcases/performance/`
- **文件名**：`YYYY-MM-DD_<模块>-冒烟测试.md`
- **内容格式**：标准 Markdown，可直接渲染
- **报告类型**：一体化报告（功能验证 + 性能指标）
