## 🚀 导入步骤

1. 打开 Apifox
2. 选择目标项目
3. 点击左侧菜单 **「设置」** → **「导入数据」**
4. 选择 **「Postman Collection v2」** 格式
5. 上传 `5.19.0-Borrow-Apifox-TestCases.json` 文件
6. 在导入选项中选择：
   - **「覆盖」**：会更新参数值，但不会覆盖接口字段说明（因为生成的用例只包含参数值，不包含参数说明等元数据）
   - **「新增」**：创建新的接口（如果接口已存在，会创建重复的接口）
7. 点击 **「确定导入」**

> 💡 **重要说明**：生成的用例不会覆盖接口的响应定义（返回响应、响应示例、响应字段说明等），这些是给前端看的参数定义，会完整保留。

## 🔧 参数传递规则

### ✅ 直接传参（硬编码）

固定参数直接写在请求中，无需配置：

| 参数 | 值 | 说明 |
|------|-----|------|
| `baseUrl` | `https://api.onekey.so` | API 地址 |
| `networkId` | `sol--101` | Solana 主网 |
| `provider` | `kamino` | Kamino 协议 |
| `accountAddress` | `AaNEipfe...` | 测试账户 |
| `amount` | `0.01` | 测试金额 |
| 请求头 | `5.19.0` 等 | 版本等信息 |

### 🔄 动态传递（环境变量）

接口间需要传递的动态数据，通过 Test 脚本自动写入环境变量：

| 变量 | 提取接口 | 使用接口 | 提取代码 |
|------|---------|---------|---------|
| `marketAddress` | 01-获取 Market 列表 | 后续所有接口 | `pm.environment.set('marketAddress', ...)` |
| `reserveAddress` | 03-获取 Reserve 列表 | Reserve 相关接口 | `pm.environment.set('reserveAddress', ...)` |
| `orderId` | 14-构建 Supply 交易 | 19-交易确认页 | `pm.environment.set('orderId', ...)` |

## ⚡ 运行顺序

**必须按顺序执行前置接口**，后续接口依赖其返回的环境变量：

```
1️⃣ 01-获取 Market 列表
   ↓ 提取 marketAddress
2️⃣ 03-获取 Reserve 列表
   ↓ 提取 reserveAddress
3️⃣ 14-构建 Supply 交易（或其他交易构建接口）
   ↓ 提取 orderId
4️⃣ 19-交易确认页（使用 orderId）
5️⃣ 其他接口（可按需执行）
```

### 批量运行

1. 右键点击集合 → **「运行」**
2. 确保按顺序执行
3. 查看测试报告
