## 📥 导入步骤

### 1. 打开 Apifox

### 2. 导入测试用例
1. 点击左侧菜单「设置」→「导入数据」
2. 选择「Postman Collection v2.1」格式
3. 上传 `Prime-用户权益-Apifox-TestCases.json`
4. 确认导入

### 3. 环境变量（自动获取）

以下变量会在测试执行过程中**自动提取**，无需手动配置：

| 变量名 | 说明 | 来源 |
|--------|------|------|
| `X-Onekey-Request-Token` | 登录 Token | 登录接口自动获取 |
| `uuid` | 随机 ID | 获取随机ID接口自动获取 |
| `deviceInstanceId` | 设备实例 ID | 获取设备列表接口自动获取 |

> **说明**：URL 使用相对路径，会自动使用项目配置的 Prime 服务前置 URL（`https://prime.onekeycn.com`）

## ⚠️ 注意事项

1. **Token 依赖**：大部分接口需要 `X-Onekey-Request-Token`，请先执行登录流程
2. **环境变量自动提取**：用例执行时会自动提取响应数据并写入环境变量
3. **危险操作**：`注销账户` 接口为危险操作，测试时请使用测试账号
4. **文件上传**：`上传头像` 接口需要在 Apifox 中手动选择文件

## 📊 断言规则

每个用例包含以下标准断言：

| 断言类型 | 说明 |
|----------|------|
| HTTP 状态码 | 验证响应状态为 200 |
| 响应时间 | 验证响应时间 < 3000ms |
| 业务状态码 | 验证 `code` 字段为 0 |
| 数据结构 | 验证响应数据包含必要字段 |
